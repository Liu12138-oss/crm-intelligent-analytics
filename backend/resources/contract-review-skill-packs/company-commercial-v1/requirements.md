# 公司合同审查要求

本文件适用于公司内部商务人员审查商业合同，基于公司销售风控检视表制定。

---

## 一、通用商务条款审查

### 1.1 主体资格
- [ ] **甲方信息完整性**: 单位全称、地址、联系人、电话、工商注册地、纳税人识别号、开户银行、银行账号、法定代表人完整，不得为空、不得保留模板下划线、空白单元格或占位文本
- [ ] **乙方信息完整性**: 乙方为深圳市联软科技股份有限公司，名称、地址、联系方式、税务及开户信息需与购销合同标准模板一致，不得被改写
- [ ] **最终用户明确**: 合同中需明确最终用户法定注册全称
- [ ] **合同编号规范**: 格式为"LS年月日-邮箱地址-区号-合同编号"

### 1.2 合同金额
- [ ] **金额大小写**: 合同金额需同时包含大写和小写，且两者一致
- [ ] **未税价与含税价计算正确**:
  - 产品含税价 = 未税价 × 1.13（四舍五入保留两位小数）
  - 服务含税价 = 未税价 × 1.06（四舍五入保留两位小数）

### 1.3 付款条件
- [ ] **账期符合标准**:
  - 首付款：最长30天（自然日）
  - 到货款：最长60天
  - 验收款：最长30天
- [ ] **分期比例符合要求**:
  - 合同金额 < 5万元：一次性付款
  - 合同金额 5-20万元：两次付款
  - 合同金额 > 20万元：不超过三次付款，尾款比例5%-10%
- [ ] **付款分支已收敛**: 如合同基于购销合同标准模板生成，模板中的“付款方式（二选一）”应只保留一个有效分支，另一分支不得继续保留造成歧义
- [ ] **首付款比例**: 新客户最低30%（直签客户可另行处理）
- [ ] **老客户付款条件**: 与历史合同相比，付款条件只可优化不可变差

### 1.4 违约罚则
- [ ] **违约金上限**: 违约上限为合同总金额的20%-100%，不得超出合同本身金额
- [ ] **违约金计算**: 每推迟一天按合同总金额的千分之五计算，但最高不超过合同总金额的百分之五（或服务合同百分之十）
- [ ] **与历史合同对比**: 违约条款不能比历史合同变差

## 二、软件/技术服务特定条款审查

### 2.1 产品清单（适用于购销合同、扩容合同）
- [ ] **模块描述**: 参照最新报价单版本，一字不差
- [ ] **模块数量**: 与售前方案匹配，需售前确认
- [ ] **许可名称**: 与报价单一致
- [ ] **产品型号**: 使用标准产品型号（如LV-7050、LV-7200等）

### 2.2 税率
- [ ] **产品税率**: 13%（增值税专用发票或普通发票）
- [ ] **服务税率**: 6%（维保、技术服务、定制开发、租赁订阅）
- [ ] **发票类型**: 明确开具增值税专用发票或普通发票

### 2.3 折扣率（硬性指标）
- [ ] **销售权限**: 折扣不低于3折
- [ ] **区总权限**: 折扣不低于2折
- [ ] **扩容合同**: 折扣参照历史成交价（原则上不低于原合同折扣）
- [ ] **两折以上新客户**: 如明显低于同行业报价，需评估合理性
- [ ] **两折以上老客户**: 如低于历史成交价，需按阶梯报价评估合理性
- [ ] **两折以下**: 必须提请特价审批，说明原因

### 2.4 报价金额门槛（硬性指标）
- [ ] **产品合同**: 报价总价 ≥ 15,000元
- [ ] **甲方实施的产品合同**: 报价总价 ≥ 5,000元
- [ ] **维保服务合同**: 报价总价 ≥ 6,000元
- [ ] **EMM/魔方产品线**: 有效收入低于6万原则上不进行签约

### 2.5 费用比例（硬性指标）
- [ ] **实施费**: 标准实施&非标准实施费用为报价总价的12%
- [ ] **服务费（两折以上）**: 服务费用为报价总价的12%-15%
- [ ] **服务费（两折以下）**: 服务费用为报价总价的15%及以上
- [ ] **定制开发**: 只有已报备的大客户才可申请，需经区总审批

### 2.6 实施范围
- [ ] **标准实施**: 不超过合同点数15%的推广工作，且推广不超过500点及1个物理地点
- [ ] **非标准实施**: 仅限于合同所购产品的安装、调试工作
- [ ] **不限点实施**: 买断式实施服务，规模未知（需特别注意风险）
- [ ] **甲方实施**: 需核实是否为有交付能力的核心渠道
- [ ] **应急演练**: 应急演练需明确演练次数，不超过1年1次

### 2.7 知识产权（高风险条款）
- [ ] **软件归属**: 软件及技术资料知识产权归乙方所有
- [ ] **甲方权限**: 甲方仅享有合同系统范围内的非独占、非排他的一般使用权（转售权）
- [ ] **源代码**: 定制开发接口部分源代码可共享，但不可归客户独有
- [ ] **通用需求**: 通用性需求定制开发项源代码归乙方所有
- [ ] **禁止行为**: 甲方不得逆向工程、反汇编、反编译、删除版权信息、制作注册机或破解程序

### 2.8 质保期（适用于购销合同）
- [ ] **质保期含义**: 购买产品时含一年质保期
- [ ] **质保期时间**:
  - 到货款：项目到货之日起13个月
  - 验收款：标准实施验收之日起或验收之日起12个月
- [ ] **描述标准**: 根据合同类型选择正确描述（到货日/验收日）
- [ ] **扩容合同**: 根据是否购买新后台选择正确描述

### 2.9 维保期（适用于服务合同）
- [ ] **时间描述精确**: 必须精确到某年某月某日（如：2021年11月1日-2022年11月1日）
- [ ] **错误示例**: "合同签订之日起一年"为错误描述
- [ ] **服务内容**: 按照服务报价单填写标准服务内容
- [ ] **服务对象**: 明确产品合同编号、产品合同金额、点数

### 2.10 产品许可交付（高风险条款）
- [ ] **临时许可**: 签订合同且收到首付款后可交付临时许可
- [ ] **永久许可**: 签订合同且收到全款后7个工作日内交付永久许可
- [ ] **禁止提前交付**: 未收到全款不得提前交付永久许可
- [ ] **交付物**: 产品配置清单、维保服务内容

### 2.11 开票（高风险条款）
- [ ] **开票名称**: 所有产品名称前必须加"*软件*"前缀（如：*软件*联软ESPP企业安全监测保护平台软件）
- [ ] **开票名称不可更改**: 必须参考合同模板中的报价清单，否则无法退税
- [ ] **开票规则**: 软件、硬件、技术服务费分开单独开票
- [ ] **开票信息**: 使用最新版开票信息

### 2.12 合同模板使用
- [ ] **代理商合同**: 必须按照我方模板签约
- [ ] **直签客户合同**: 优先引导使用我方模板（购销合同模板、扩容合同模板、服务合同模板）
- [ ] **标准模板固定结构完整**: 如合同基于我方购销合同标准模板生成，合同标题、主体信息区、合同范围、价款与支付、交货、知识产权和保密、不可抗力、违约责任、争议和仲裁、报价清单、附则及附件结构应完整，不得随意删节
- [ ] **标准模板固定信息不得擅改**: 乙方主体信息、争议解决基线、违约责任基线、知识产权归属基线不得被改成明显不利于乙方的版本
- [ ] **标准模板必填槽位已补齐**: 合同编号、最终用户、甲方信息、金额大写/小写、报价清单、签署页信息不得保留模板空白、下划线或占位文本
- [ ] **标准附件完整**: 购销合同模板中的产品配置清单、廉洁诚信合作协议等标准附件不得缺失、删减或弱化
- [ ] **标准模板基线维护**: 当前购销合同模板基线详见 `购销合同标准模板基线.md`，如公司模板更新，应先同步该文件

### 2.13 关键字一致性检查
- [ ] **客户名称一致性**: 最终客户名称唯一，不出现多个客户名称
- [ ] **合同编号一致性**: 前后文合同编号一致，页眉合同编号一致
- [ ] **合同金额一致性**: 不出现其他合同金额
- [ ] **联系人一致性**: 不出现其他不相关的联系人名字

## 三、一票否决项（发现以下问题合同不得签署）

| 序号 | 否决项 | 风险等级 |
|------|--------|----------|
| 1 | 知识产权、源代码归客户独有 | 🔴 高风险 |
| 2 | 未收到全款提前交付永久许可 | 🔴 高风险 |
| 3 | 开票名称不含"*软件*"前缀 | 🔴 高风险 |
| 4 | 违约上限超过100%或超出合同本身金额 | 🔴 高风险 |
| 5 | 折扣低于2折未经特价审批 | 🔴 高风险 |
| 6 | EMM/魔方产品线有效收入低于6万签约 | 🔴 高风险 |
| 7 | 甲方有权对软件进行逆向工程、反编译 | 🔴 高风险 |
| 8 | 定制开发通用性需求所有权归客户 | 🔴 高风险 |

## 四、风险评级标准

### 🔴 高优先级（Critical）- 必须修改
- 违反一票否决项
- 知识产权约定不明或归客户独有
- 付款条件严重不利（如全款预付且无保障）
- 违约金比例过高或不对等
- 折扣率低于权限未经审批

### 🟡 中优先级（Warning）- 建议修改
- 付款条件不符合公司标准（账期过长、分期比例不当）
- 实施范围不明确
- 质保/维保期描述不精确
- 合同金额计算有误（含税/未税）
- 关键字不一致

### 🔵 建议优化（Suggestion）- 可以优化
- 合同条款表述可更严谨
- 缺少某些保护性条款
- 格式问题
- 建议补充的说明

---

## 五、审查结论建议

### 合同可以通过的情况
- 无高优先级问题
- 中优先级问题不超过3项且有明确解决方案

### 合同修改后签署的情况
- 存在高优先级问题但可修改
- 中优先级问题较多

### 合同不建议签署的情况
- 存在任何一票否决项且无法修改
- 客户拒绝修改关键风险条款
- 多个高优先级问题

## 六、合同类型说明

### 购销合同
用于客户购买软硬件产品，包含产品交付、质保期等条款。

### 服务合同（维保合同）
用于客户购买运维服务、技术支持，包含服务期限、服务内容、维保期等条款。

### 扩容合同
用于客户在原有合同基础上增加点数、模块或新后台，需明确与原有合同的关系。

---

**注意**: 本文件基于公司销售风控检视表制定，适用于公司内部商务人员审查合同使用。具体数值标准（如金额门槛、折扣率等）为硬性指标，必须严格遵守。
