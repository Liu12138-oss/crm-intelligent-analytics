🔍问题三："最近2个月商机情况"返回空结果（需排查）

现象：AI 将列表查询误判为实体名称查找 需深入排查

项目内容
用户提问@渠通CRM系统机器人 最近2个月商机情况，所有商机条目列出来看看
实际回复当前未查到可继续跟进的商机，请补充更完整的名称后再试。
回复来源内容由AI生成（非硬编码模板）
期望行为进入 ANALYZE 分支，执行常规分析，返回最近2个月的商机列表/趋势
实际行为被 AI 意图分类器判定为 OPPORTUNITY_LOOKUP（按名搜索特定商机），找不到匹配项后提示"补充名称"

根因分析

回复文本 "请补充更完整的名称后再试" 是 OPPORTUNITY_LOOKUP / ENTITY_LOOKUP 路径的特征响应模式。说明：

  1. AI 意图分类器将 "最近2个月商机情况" 识别为「按名称查找某个具体商机」，而非「列出所有商机的分析」
  2. 路由到了 wecomEntityLookupService 或类似实体查找路径，而非 ANALYZE 分析路径
  3. 实体查找在 CRM API 中按名称匹配，自然找不到名为 "最近2个月商机情况" 的商机

可能的方向（待验证）

  • 检查 classifyWecomIdleConversationIntent 对此问题的分类结果
  • 检查 hasExplicitWecomAnalysisOrBlockSignal 是否正确放行（应包含 "商机" 关键词 → 返回 true）
  • 如果信号层正确放行但仍然被路由到 LOOKUP，则需要在 orchestration 层增加强制 ANALYZE 规则
  • 或优化 AI 分类器的 system prompt / few-shot examples
