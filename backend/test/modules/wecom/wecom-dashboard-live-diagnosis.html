✅ 修复1：排除性过滤 — 防止筛选类问题误入看板桥接
在 isWecomDashboardAnalysisRequest（wecom-bot.service.ts）和 isDashboardAnalysisQuestion（orchestration.service.ts）新增排除性正则：
const isExclusionaryQuery = /没有.{0,8}(客户|商机|订单|报价).{0,6}(情况|的)
  | 无.{0,4}(进展|维护|跟进|商机|订单)
  | 未.{0,4}(维护|跟进|进展)/u.test(text);
if (isExclusionaryQuery) return false;
wecom-bot.service.ts:1465 ~1473 | wecom-ai-conversation-orchestration.service.ts:994~1002
覆盖场景："近3个月没有维护进度的客户情况"、"最近3个月没有商机和订单的渠道商情况"
✅ 修复2：detectProfile 检测顺序优化 — 区域优先于渠道商
调整 detectProfile() 中正则匹配顺序，将 各区域.*看板|各区域.*数据 提前到 渠道商.*发展|渠道商.*运营 之前：

// 4. 订单.*分析 → channel-order-summary
