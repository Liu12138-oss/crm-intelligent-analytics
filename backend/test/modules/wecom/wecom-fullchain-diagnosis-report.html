🐛 Bug #1：模板卡片字段名错误（已修复）

根因：horizontal_content_list 字段名使用了 label/key 而非 keyname 已修复

企微 text_notice 模板卡片的 horizontal_content_list 数组要求每项字段名为 keyname（非 label、非 key）。使用错误字段名会导致企微 API 拒绝卡片或静默丢弃 KPI 列表，用户看不到模板卡片。

修复前（错误）

// wecom-bot.service.ts convertDashboardResultToWecomFormat() const kpiItems = kpiMetrics.map((m) => ({ label: m.label, // ❌ 错误：应为 keyname value: m.value, })); // wecom-dashboard-card-builder.service.ts buildDashboardCard() const horizontalContentList = params.kpiItems.map((item) => ({ key: item.label, // ❌ 错误：应为 keyname value: item.value, }));

修复后（正确）

// wecom-bot.service.ts convertDashboardResultToWecomFormat() const kpiItems = kpiMetrics.map((m) => ({ keyname: m.label, // ✅ 正确：企微 API 要求 keyname value: m.value, })); // 同时增加了字段长度截断，符合企微 API 限制： // keyname <= 5 字符, value <= 26 字符 // main_title.title <= 26, desc <= 30 // emphasis_content.title <= 12, desc <= 30 horizontal_content_list: kpiItems.map((item) => ({ keyname: truncateCard(item.keyname, 5), value: truncateCard(String(item.value), 26), })),

影响文件：

  • wecom-bot.service.ts — convertDashboardResultToWecomFormat() 方法，kpiItems 构建 + 卡片结构增强
  • wecom-dashboard-card-builder.service.ts — buildDashboardCard() + buildTimeoutCard() 方法

🐛 Bug #2：场景③排除过滤未在信号层拦截（已修复）

根因：hasExplicitWecomAnalysisOrBlockSignal 匹配"客户"关键词放行到 ANALYZE 分支 已修复

"近3个月没有维护进度的客户情况" 包含"客户"关键词，被 hasExplicitWecomAnalysisOrBlockSignal 第 2342 行 /商机|...|客户|.../ 匹配，返回 true → 进入 ANALYZE 分支。虽然 isWecomDashboardAnalysisRequest 有排除过滤，但信号层未拦截导致问题定位困难。

修复策略：4 层排除过滤防线

① hasExplicitWecomAnalysisOrBlockSignal
→
② isWecomDashboardAnalysisRequest
→
③ 安全网 isExclusionarySafetyNet
→
④ 最终拦截（compose 前）


排除匹配 → 强制走 executeRegularAnalysisPipeline

统一排除函数 isExclusionaryFilterQuery

private isExclusionaryFilterQuery(normalizedText: string): boolean { // 四组排除模式，覆盖所有筛选/明细类提问 return ( // 1. 没有 + (客户|商机|订单|报价|渠道商|代理商) + 情况/的/列表/明细 /没有.{0,8}(客户|商机|订单|报价|渠道商|代理商).{0,6}(情况|的|列表|明细)/u.test(normalizedText) || // 2. 无 + (进展|维护|跟进|商机|订单|更新|联系) /无.{0,4}(进展|维护|跟进|商机|订单|更新|联系)/u.test(normalizedText) || // 3. 未 + (维护|跟进|进展|更新|联系) /未.{0,4}(维护|跟进|进展|更新|联系)/u.test(normalizedText) || // 4. 没有 + (维护进度|跟进记录|商机和订单|维护记录) — 直接匹配 /没有.{0,6}(维护进度|跟进记录|商机和订单|维护记录)/u.test(normalizedText) ); }
