# 前端 UI 图标使用规范

本目录用于集中管理项目级 UI 能力。当前 `icons.ts` 统一封装 `@element-plus/icons-vue` 中的常用图标，页面和组件应优先从 `UiIcons` 读取图标，不应在页面内随意选择无语义图标、文字首字或临时字符。

## 使用原则

- 导航入口必须使用稳定业务图标，并同时保留中文名称。
- 请求型操作必须保留中文加载文案；图标只辅助识别，不替代文字。
- 成功、失败、警告、风险、阻断和加载状态必须同时使用图标与中文说明，不能只靠颜色区分。
- 上传、导出、刷新、筛选、搜索、返回、关闭、展开收起等通用操作必须优先复用 `UiIcons` 中的映射。
- 若现有映射不足，应先在 `icons.ts` 补充语义化名称，再在页面使用。

## 当前覆盖范围

- 业务入口：智能分析、合同审核、权限中心、查询模板、连接策略、审计中心。
- 通用操作：上传、下载、刷新、筛选、搜索、返回、关闭、展开、收起。
- 状态反馈：成功、失败、警告、风险、加载、信息提示。
- 表单与登录：用户、密码、登录、连接。
