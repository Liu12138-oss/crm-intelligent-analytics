# 快速开始：CRM智能分析系统（一期：企业微信 AI 问数、智能分析工作台与智能合同审核）

## 1. 目标

本文档用于指导开发与测试人员在实现完成后，快速启动本地环境并验证一期核心能力，包括账号密码登录、企业微信扫码登录、企业微信问数、Web 智能分析工作台、智能合同审核、常用查询、最近查询、导出限制、业务事件审计与 SQL 审计治理。

## 2. 前置准备

- Node.js 20 LTS
- pnpm 9.x
- MySQL 8.x
- 一份可访问的 CRM 数据源连接信息
- 一份独立应用库连接信息，用于治理配置、查询模板、最近查询、审计日志和导出记录
- 一份可写的本地或挂载目录，用于保存合同审核原始文件与产物文件
- 一套可用的内部 AI 网关地址与凭证，并确认其接入约定与 `配置/codex配置/config.toml` 保持一致
- 一套可用的企业微信机器人测试配置、回调地址和用户身份映射测试数据
- 一套可用的 CRM Open API 登录配置，包括企业标识 `corpId`
- CRM 官方 API 文档地址：`https://apidoc.weiwenjia.com/`
- 一组可对照的参考报表样例，用于验证关键指标准确性

### 2.1 配置映射约定

| 本地资料来源 | 推荐环境变量 | 用途 |
| --- | --- | --- |
| `配置/codex配置/config.toml` | `ANALYSIS_AI_BASE_URL`、`ANALYSIS_AI_MODEL_PROVIDER`、`ANALYSIS_AI_MODEL`、`ANALYSIS_AI_REASONING_EFFORT` | 驱动智能分析与合同审核的后端 AI 编排和模型调用 |
| `配置/codex配置/auth.json` | `OPENAI_API_KEY` | 作为兼容 Responses 协议网关的访问凭证 |
| 后台 AI 模型治理 | `AI_PROFILE_MASTER_KEY` | 用于加密后台保存的 AI Profile 密钥；当后台存在已激活 Profile 时，全系统 AI 调用优先读取该 Profile |
| `配置/企业微信/ID和密钥.md` | `WECOM_BOT_ID`、`WECOM_BOT_SECRET` | 驱动企业微信机器人接入与消息回传 |
| 企业微信主动通知配置 | `WECOM_NOTIFY_REAL_MESSAGE_ENABLED`、`WECOM_NOTIFY_TEST_USER_ID` | 驱动受控主动通知底座的机器人真实发送开关与测试收件人覆盖 |
| 企业微信官方通讯录同步 | `WECOM_WEB_LOGIN_APP_ID`、`WECOM_WEB_LOGIN_SECRET`、`WECOM_DIRECTORY_ROOT_DEPARTMENT_NAME`、`WECOM_DIRECTORY_PAGE_SIZE` | 驱动企业微信官方部门与成员目录同步 |
| CRM Open API 登录配置 | `CRM_OPEN_API_BASE_URL`、`CRM_OPEN_API_CORP_ID`、`CRM_OPEN_API_VERSION_CODE`、`CRM_OPEN_API_DEVICE` | 驱动 Web 账号密码登录与本地会话建立 |
| CRM 机器人受控写入内置账号配置 | `CRM_OPEN_API_WRITEBACK_LOGIN`、`CRM_OPEN_API_WRITEBACK_PASSWORD` | 驱动企业微信机器人在受控跟进写回、受控新增客户与受控新增商机时换取系统账号 token，不依赖发送人的 Web 登录态 |
| CRM 新增客户/商机接口字段映射配置 | `CRM_CUSTOMER_CREATE_*`、`CRM_OPPORTUNITY_CREATE_*` | 驱动后端新增客户/新增商机接口补齐默认值，并把截图必填业务字段映射到 CRM 标准字段或自定义字段 |
| CRM 数据源 | `CRM_READONLY_DB_*`、`CRM_WRITEBACK_DB_*` | 仅用于 CRM 官方 API 未覆盖的受控路径；查询链路、补查链路和写回链路按对象与动作选择对应连接，企业微信目录同步按受控边界写回 CRM 原生 `wx_*` 表 |
| 应用存储库 | `APP_STORAGE_DB_*` | 存储治理配置、最近查询、业务审计事件、SQL 审计记录、导出记录和会话快照 |
| 合同审核本地文件存储 | `CONTRACT_REVIEW_STORAGE_DIR`、`CONTRACT_REVIEW_MAX_FILE_SIZE_MB` | 保存合同审核原始文件、审核报告、批注版合同与结构化结果文件 |
| 合同审核 skill pack 根目录 | `CONTRACT_REVIEW_SKILL_PACK_ROOT_DIR` | 指向合同审核审核标准包根目录，运行时从该目录加载 `profile.yaml`、检查项和提示词 |
| 合同审核激活标准 | `CONTRACT_REVIEW_SKILL_PACK_CODE` | 指定当前生效的合同审核标准包编码，对应 `CONTRACT_REVIEW_SKILL_PACK_ROOT_DIR` 下的子目录 |
| 合同审核扩展查看与下载角色 | `CONTRACT_REVIEW_REVIEWER_ROLE_IDS`、`CONTRACT_REVIEW_DOWNLOADER_ROLE_IDS` | 控制哪些角色可以查看他人审核任务详情，以及哪些角色可以下载他人产物 |

- 启动本地环境时，代码统一从环境变量取值，不直接读取上述文档中的明文内容。
- 真实密钥不得写入 `.env.example`、测试夹具、截图说明或接口示例。
- AI配置中心当前以 `OpenAI 兼容 HTTP` Profile 作为主运行时；当后台激活任一 HTTP Profile 后，企业微信 AI 理解、Web 问数和合同审核 AI 审查统一读取该 Profile。
- 后台 AI配置页会自动把 `backend/.env.local` 中识别到的可用 OpenAI 兼容 HTTP 配置引导为“环境默认 OpenAI 兼容 HTTP”档案。
- 后台 AI配置页中的 `推理等级` 支持编辑；OpenAI 兼容 HTTP 未显式填写时默认补齐为 `low`，优先保障响应速度。
- 后台 AI配置页中的上下文策略支持编辑，至少包括上一轮问题保留上限、上一轮结果摘要保留上限、问答上下文保留轮次上限以及普通分析 / 任务态会话失活时长。
- 后台 AI Profile 切换依赖 `AI_PROFILE_MASTER_KEY` 加密保存密钥；如果生产环境未配置该变量，不应开启后台密钥治理能力。
- 当前项目继续复用同一个企业微信 corpId，但 Web 登录与组织架构同步必须使用两套不同的应用 secret。
- `WECOM_WEB_LOGIN_CALLBACK_URL` 可以显式配置为后端回调地址；如果未配置，扫码后会默认回到前端 `/login` 页面，由前端调用换票接口完成登录。
- 企业微信机器人当前不再提供 `WECOM_BOT_TRANSPORT_MODE` 切换项；开发环境与生产环境统一按 SDK 方式接入企业微信，但当本地或测试环境启动阶段建连失败时，系统只记录降级告警，不再打崩后端启动。
- 受控主动通知底座默认推荐使用 `WECOM_NOTIFY_REAL_MESSAGE_ENABLED=false`，并通过 `WECOM_NOTIFY_TEST_USER_ID` 指定联调测试接收人；只有在明确开启真实发送开关后，才允许向真实业务对象发消息。
- 若生产环境发现企业微信组织架构未同步，必须立即执行一键同步脚本或内部同步入口，不能继续依赖“未绑定 CRM 身份”的兜底提示作为长期状态。
- 当前生产环境现行条件是：只同步“联软科技集团”授权范围，不同步 ROOT 全量组织树。
- 合同审核主流程改为 CRM 待 1 级审批合同驱动；本地 `.docx` 上传仅作为兼容补录入口保留。若兼容上传文件大小超限、结构损坏或正文提取失败，系统应直接阻断正式审核。
- 合同审核当前默认先返回 `DETERMINISTIC_ONLY` 规则快审结果，仅覆盖已配置的明确判定项；当 AI 可用且存在未被确定性规则覆盖的检查项时，系统会继续触发 AI 补充审核并在完成后刷新结果。
- 当 AI 补充审核不可用、超时或返回非法结构时，系统继续保留 `DETERMINISTIC_ONLY` 结果，并在页面、摘要和产物中明确标识为“降级快审”及其原因；若没有降级原因，则应明确标识为“规则快审”。
- `CONTRACT_REVIEW_SKILL_PACK_CODE` 必须与 `CONTRACT_REVIEW_SKILL_PACK_ROOT_DIR` 下的目录名一致；审核标准版本、校验值和提示词指纹以运行时加载的 `profile.yaml` 和 prompts 快照为准。

### 2.2 CRM 接入优先级约束

- 涉及 CRM 的新增功能和既有非认证能力改造前，先核对 CRM 官方 API 是否已覆盖目标对象、动作、字段和权限语义。
- CRM 官方 API 的统一评估入口固定为 `https://apidoc.weiwenjia.com/`，联调或方案评审时先查该地址，再决定是否进入兜底路径。
- 只有官方 API 无法满足已确认的字段、权限、时效或稳定性要求时，才允许进入自建接口或数据库路径，并在实现说明中记录 API 缺口、白名单边界、审计留痕和回退方案。
- 当前稳定的账号密码登录、企业微信扫码登录、本地会话与统一鉴权链路先不调整；本轮联调与验收只验证其现状稳定性。

## 3. 规划中的本地启动方式

以下命令以计划中的目录结构为准：

```bash
pnpm install
pnpm --dir backend install
pnpm --dir frontend install
pnpm --dir backend dev
# 后端 dev 默认开启热更新，修改 backend/src 后会自动重启
pnpm --dir frontend dev
```

前端默认访问内部开发环境地址，后端默认连接本地应用库与 CRM 数据源。企业微信机器人默认连接测试企业微信应用与受控开发环境。

补充说明：

- 当前后端会优先读取环境变量；若未配置，将自动尝试从 `配置/` 目录装载 AI 网关与 CRM 数据源配置。
- Web 端默认先进入登录页；若未配置真实 CRM Open API，可通过测试环境使用样例账号走模拟登录。
- 企业微信机器人第一阶段已支持正式消息归一化、幂等去重、真实结果分发记录和会话串行；第二阶段已支持最小可用 AI 对话编排与解释型追问。
- 受控主动通知底座已调整为用户可见通知统一走企业微信机器人，并在真实发送开关关闭时统一改投测试收件人。

## 4. 核心验证场景

### 场景 1：Web 智能分析工作台直接问数

1. 打开 Web 登录页，使用已授权的销售总监账号通过账号密码或企业微信扫码登录。
2. 输入问题：“本季度各区域新增商机金额、赢单率和环比变化”。
3. 验证页面返回：
   - 业务化标题
   - 结论摘要
   - 关键指标卡
   - 主图表和排行表
   - 趋势预测、异常与风险、经营建议和建议下一步
   - 时间范围、筛选条件和数据更新时间
4. 继续停留在当前查询页，验证 AI 分析报告区会直接展示完整 richer report，而不是要求必须跳转详情页：
   - 趋势分析
   - 趋势预测
   - 异常与风险
   - 经营建议
   - 结果依据
   - 完整 Markdown 阅读稿折叠入口
5. 如需兼容回归，再打开结果详情页，验证页面仍同时展示执行依据摘要、命中的语义资产标签，以及统一结果包中的口径脚注或来源说明。

### 场景 2：企业微信即时问数与追问

1. 使用已授权的销售总监企业微信账号与机器人建立会话。
2. 输入问题：“本月各销售负责人新增商机金额排名”。
3. 验证机器人先返回处理中提示，再逐步返回摘要块、关键指标块和必要的结果块，而不是一次性长消息。
4. 继续追问“近三个月趋势也给我看看”，验证系统承接上下文并返回新的结果。
5. 再追问“这说明什么”，验证系统优先基于上一轮真实结果生成解释，而不是再次重新查数。

### 场景 3：必要条件补问与拒答

1. 使用已授权的销售管理账号在 Web 或企业微信中输入问题：“看一下商机转化怎么样”。
2. 验证系统先提示补充缺失的时间范围、比较对象或统计口径，而不是直接返回不完整结果。
3. 输入超出支持范围或涉及敏感字段的问题，验证系统明确拒答，并说明当前支持范围。

### 场景 4：常用查询执行

1. 使用销售总监账号进入 Web 智能分析页面。
2. 查看左侧常驻常用查询资产区。
3. 在“我的模板 / 其它模板”之间切换，使用关键词和标签筛选，验证可见列表只包含当前用户可见模板，并按历史累计点击优先展示。
4. 点击某个常用查询，例如“季度商机健康度总览”，验证系统直接生成结构化分析结果，并记录模板执行行为。
5. 验证模板执行后的查询页同样直接展示完整 richer report，且不再出现“请基于返回数据给出中文经营分析”这类占位式文案。
6. 使用山东区负责人这类部门负责人账号执行默认全量模板，验证结果、图表、转置表、矩阵表和导出内容只包含其默认业务部门、递归子部门或数据范围白名单覆盖的数据；若企业微信直属链包含其它区或技术服务线成员，审计快照中的 `ownerIds` 不得包含这些范围外成员。
7. 在治理后台校验一条按 `opportunities.department_id` 收口但通过 `customers.department_id -> departments.name` 展示团队名称的 SQL，确认页面展示中文风险和修复建议，不直接要求模板作者记忆内部范围参数。
8. 在“其它模板”点击“添加到我的模板”，确认系统创建个人副本，并且复制动作不增加模板历史点击数。

### 场景 4.1：自由问数保存为模板

1. 在 Web 工作台提交一个自然语言问题并等待结果成功返回。
2. 点击结果区“保存为模板”。
3. 输入模板名称、说明，并选择已有标签或输入新标签。
4. 点击保存，确认模板进入“我的模板”，其它用户可在“其它模板”搜索到。
5. 使用缺少可复现执行快照的历史结果保存时，系统必须返回中文不可保存原因，不得要求或允许前端传入 SQL。

### 场景 4.2：固定部门模板跨部门复用

1. 准备一个 SQL、默认条件或问题文本中写死“山东”范围的模板。
2. 使用未授权山东范围的北京用户执行该模板，系统必须阻断原范围执行，并提示可“改为当前可见范围”。
3. 用户确认改写后，系统将固定范围替换为当前用户可见范围，并重新执行 SQL 白名单、AST、范围治理和预检。
4. 对无法可靠替换的固定范围模板，系统必须提示联系管理员治理为通用模板。

### 场景 4.3：查询模板 SQL 编写权限

1. 在权限中心为某角色授予 `template.sql.write`，不授予 `template.manage`。
2. 使用该角色访问治理模板新增、SQL 校验和预览入口，确认可以创建和校验受控只读 SQL 模板。
3. 使用该角色访问完整模板治理列表或删除模板，确认仍被拒绝。

### 场景 5：最近查询重跑

1. 使用同一账号执行一次标准问题查询。
2. 进入最近查询列表，确认可以看到刚才的问题、执行时间和状态。
3. 点击“再次运行”，并把时间范围改为“近三个月”。
4. 验证系统以新的条件重新生成结果，并新增一条最近查询记录。

### 场景 6：权限范围隔离与结果一致性

1. 使用仅拥有部门级权限的区域经理账号在 Web 工作台或企业微信中输入“我负责部门近 3 个月商机阶段分布和转合同率”。
2. 验证摘要、图表、表格和明细只包含授权部门范围内的数据。
3. 再尝试查询全公司数据，验证系统返回越权阻止提示。
4. 对照参考报表，验证关键指标值与标准口径一致。

### 场景 7：受控导出

1. 使用具备导出权限的用户完成一次有效查询。
2. 发起不超过 1000 行的导出，验证导出成功并生成审计记录。
3. 使用超过 1000 行的结果再次导出，验证系统阻止。
4. 连续触发超过 3 次导出，验证当日次数限制生效。
5. 对历史常用查询结果发起导出前，先把对应模板改为未通过范围治理的复杂模板，并设置 `QUERY_TEMPLATE_SCOPE_GOVERNANCE_MODE=enforce`，验证导出被阻断并提示重新执行查询或联系管理员确认模板口径。

### 场景 8：治理配置与审计检索

1. 使用系统管理员账号登录治理界面。
2. 在 `权限中心` 中调整某个 CRM 角色的菜单权限包、风险子权限或企业微信机器人入口包；或在 `查询模板管理` 中调整常用查询的适用角色与展示顺序。
3. 重新以普通用户访问 Web 工作台或企业微信机器人，验证权限、导航显隐、合同审核权限与模板可见性立即生效。
4. 进入审计页面，按用户和时间范围检索刚才的查询、模板执行、历史重跑、导出与连接事件。
5. 切换到 `SQL 审计` Tab，按模块、数据库角色、执行阶段或表名检索刚才触发的 CRM SQL，确认能看到脱敏 SQL 摘要、参数摘要、请求 ID、会话 ID、执行阶段与命中表。
6. 使用具备 `audit.sql.view_sensitive` 的管理员点击“查看完整 SQL / 参数”，验证系统先提示二次确认，再返回完整 SQL 与参数，同时在普通审计列表中新增一条 `SQL_AUDIT_RAW_VIEWED` 事件。
7. 在 `查询模板` 页面确认默认先展示模板列表，且可直接新增、编辑、删除；点击编辑后，所有配置项应收敛在单一区域中维护。

补充验证：

- 撤销 `analysis.follow_up` 后，结果详情页继续追问必须被阻断。

### 场景 12：企业微信查询后受控写回 Opportunity 跟进

4. 验证团队汇总正文按负责人展示“日报汇总”和“日报明细”，并单独列出无源数据名单，而不是继续使用旧的“未确认草稿提醒”口径。
5. 在测试模式下验证团队汇总通过 `ProactiveNotificationService` 改投 `WECOM_NOTIFY_TEST_USER_ID`；开启真实发送后，验证汇总仍固定走企业微信机器人通道而不是企业微信应用消息。

### 场景 20A：企业微信显式查看指定小组今日日报预览

1. 准备某销售负责人小组当天的 CRM 源数据，至少覆盖一名成员当天跟进记录、新增客户或新增商机之一，并确保该负责人可从企业微信当前组织事实、直属上级链和 CRM 身份映射中解析。
2. 使用该负责人本人、其直属上级负责人或管理员身份，在企业微信中发送“请把王文定小组今天的日报发给我”这类显式口令。
3. 验证系统直接返回该小组当天的日报预览，正文包含“日报汇总”和“日报明细”，且不会创建新的正式汇总批次或主动通知任务。
4. 切换为无权限成员再次发送相同口令，验证系统返回明确无权限提示，不泄露小组日报正文。
5. 将负责人姓名改为未命中值或让目标小组当天没有任何可汇总内容，验证系统分别返回“补充更完整负责人姓名”的澄清提示和明确的空状态预览。

### 场景 20B：后台按部门启用日报并预览发送目标

1. 使用系统管理员账号登录治理后台，进入 `权限中心`，切换到“日报治理”标签页。
2. 在“日报策略”分区中确认目标销售小组已启用，例如 `山东销售`；同时确认 `山东技术团队`、`山东价值客户运营团队`、`山东职能` 等非销售部门已停用。
3. 为父级区域节点配置默认收件人，例如将 `山东区` 的覆盖收件人设置为 `牛劲`。
4. 在“收件规则”分区执行发送预览，传入目标业务日期，并确认表格展示中文规则来源、中文发送状态和成员名单。
5. 若自动识别多出或漏掉销售小组，使用“新增日报小组 / 编辑 / 删除”维护后台确认后的小组配置；若组长或成员不准确，覆盖最终收件人列表和成员名单，允许同一小组选择多个组长接收汇总。
6. 使用收件规则表格中的开关快速启用或停用小组，刷新预览后验证状态、成员数和成员名单同步变化。
7. 验证预览结果中存在 `山东销售 -> 牛劲`，且被停用的非销售部门不会作为可发送目标进入真实发送。
8. 若某销售小组缺少负责人或负责人缺少企微映射，验证预览返回明确阻断原因，而不是静默跳过或错误扩大到其他负责人。

### 场景 21：通过内部接口新增商机

1. 使用已登录的 Web 会话获取有效 Cookie。
2. 调用 `POST /api/v1/crm/opportunities`，至少传入截图标注必填的 `title`、`customerId`、`leadCode`、`productAssets`、`expectAmount`、`expectSignDate`、`renewalContractCode`、`agentFullName`、`projectStatusSummary`、`preSalesName`。
3. 验证系统能够使用当前会话用户补齐负责人和所属部门，并返回标准化的商机创建结果。
4. 清空 `productAssets` 后重试，验证接口返回 `400` 且明确提示“关联产品不能为空”。

### 场景 22：企业微信受控新增客户

1. 使用已授权企业微信账号发送“新增客户”。
2. 按机器人提示补齐 `名称`、`电话`、`IT决策权所在地`、`统一社会信用代码` 四项必填字段。
3. 验证机器人返回客户创建摘要，并要求回复“确认”或“取消”。
4. 回复“确认”，验证系统返回客户 ID、客户名称和创建时间，并记录 `WECOM_CRM_CREATE_SUCCEEDED` 审计事件。

### 场景 23：企业微信受控新增商机

1. 使用已授权企业微信账号发送“新增商机”。
2. 按机器人提示补齐 `项目名称`、`最终客户`、`线索编号`、`关联产品`、`预计有效收入`、`预计签单日期`、`被续签合同号`、`代理商全称`、`项目现状及关键点`、`售前`。
3. 若只提供客户名称，验证系统能够自动查询并在唯一命中时补齐客户 ID；若关联产品未命中受控别名映射，验证系统继续追问而不进入确认。
4. 回复“确认”，验证系统返回商机 ID、商机标题和创建时间，并记录 `WECOM_CRM_CREATE_SUCCEEDED` 审计事件。

### 场景 24：受控主动通知底座通道路由与测试收件人覆盖

1. 在本地配置中设置 `WECOM_NOTIFY_REAL_MESSAGE_ENABLED=false`，并将 `WECOM_NOTIFY_TEST_USER_ID` 设为测试接收人。
2. 触发一条管理汇总类正式通知任务，验证系统默认选择企业微信机器人主动推送通道，而不是企业微信应用消息。
3. 验证该次任务不会向真实业务对象发送，而是通过机器人通道改投测试接收人，并在发送记录中标记测试收件人覆盖状态。
4. 再触发一条机器人上下文强相关的异步结果通知，验证系统仍然沿用机器人主动推送通道，但消息标签与处理动作应与普通问答区分开。
5. 使用相同幂等键重复触发同一条通知任务，验证系统命中幂等保护并记录 `DEDUPED` 或等效状态，而不是重复发送。
## 5. 非功能验证

- 标准高频问题在 5 秒内返回首条有效反馈。
- 标准问题在 20 秒内完成结构化结果展示，复杂问题在 20 秒内返回明确补问、拒答或超时指引。
- 所有查询、模板执行、历史重跑、拦截和导出事件都能在审计页面检索到。
- SQL 审计页必须能检索到本系统后端发出的 CRM SQL，并区分只读查询、预检 SQL、执行失败和执行前阻断；该能力范围仅限本系统后端发出的 CRM SQL，不等同于数据库级全局审计。
- 敏感字段默认不出现在结果与导出内容中。
- 多用户并发连接时，会话上下文保持隔离，不出现串话、乱序或重复推送。
- 同一结果中的摘要、图表、表格和导出数据保持一致。
- 最近查询重跑会套用当前最新权限，而不是复用旧结果。
- 企业微信无效签名、来源异常和结构错误消息不会进入正式分析链路。
- 企业微信维护期降级不会回退样例业务结果，也不会误报为无权限或无数据。
- 后台 AI配置页中的密钥字段不得明文回显；日志、审计和接口响应中也不得出现真实密钥。
- 后台 AI配置页的列表只能单选启用一个当前生效档案；切换后摘要卡、列表与运行时读取结果必须一致。
- 后台 AI配置页中的推理等级字段必须支持编辑、展示和持久化；OpenAI 兼容 HTTP 未显式填写时，新建、复制、环境引导和运行时兜底都必须补齐到 `low`。
- 后台 AI配置页中的上下文策略必须支持编辑、展示和持久化；企业微信追问、Web 解释追问和任务态上下文恢复必须统一消费这些阈值。
- OpenAI 兼容 HTTP Profile 必须能通过后台真实 HTTP smoke test。
- 合同审核详情与产物下载继续遵循“上传者本人可见、授权角色扩展查看”的权限边界。
- 合同审核列表、普通状态接口和普通日志不得返回完整合同正文，只能返回必要摘要。
- 合同审核任务、问题、产物和审计事件必须能够追溯到 skill pack 版本、校验摘要、模型档位和执行模式。
- 合同审核在 `DETERMINISTIC_ONLY` 模式下必须区分“规则快审”和“降级快审”：存在 `degradationReason` 时必须明确展示降级原因；不存在时必须说明当前仅覆盖已配置的明确判定项。
- 合同审核在 `BLOCKED` 模式下不得伪装成已完成审核结果。
- 真实消息发送开关关闭时，受控主动通知不得向真实业务对象发送，且应统一改投测试接收人或进入等效测试模式。

## 7. 阶段里程碑

### 里程碑 A：正式接入可上线

- 企业微信正式消息可进入统一入站模型
- 同会话串行、不同会话并发
- 重复消息幂等去重
- 结果块真实分发并可审计

### 里程碑 B：AI 会话编排可用

- 能识别新问题、补问回复、解释型追问和改条件追问
- 能基于上一轮真实结果直接解释
- AI 只能通过受控工具进入分析链路

### 里程碑 C：体验增强与维护期降级

- 长会话具备摘要压缩
- 维护期降级提示明确且可恢复
- 追问继承与补问填槽更自然

### 里程碑 D：企业微信目录同步恢复能力

- 可通过内部入口查看最近一次目录同步状态
- 可通过一键脚本触发全量资源同步
- 机器人可通过目录镜像做手机号 / 邮箱兜底识别

## 6. 回归重点

- 问题不在一期支持范围时，系统应返回明确提示而不是报错。
- 问题缺少关键限定条件时，系统应先补问而不是猜测用户意图。
- 常用查询仅对允许角色可见，最近查询只能由本人查看。
- 商机和合同关联关系缺失时，系统应说明指标不可用。
- 空数据结果必须显示“当前授权范围内无匹配数据”。
- 企业微信会话中断、心跳超时或高并发排队时，系统必须返回明确提示而不是无响应。
- 除“受控新增客户”“受控新增商机”“唯一 Opportunity 查询后受控跟进写回”之外，企业微信写入型请求和提醒创建请求必须明确拒绝，而不是进入补问或执行链路。
