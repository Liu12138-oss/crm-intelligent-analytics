# 安全与治理需求质量检查清单：CRM智能分析系统（一期：企业微信 AI 问数与智能分析工作台）

**目的**：审查企业微信问数、Web 智能分析工作台、查询复用、结果展示、权限边界、导出控制和审计治理相关需求是否完整、清晰、一致且可验证。  
**创建日期**：2026-03-24  
**对应规格**：[spec.md](../spec.md)

**说明**：本清单用于检查需求写作质量和治理覆盖面，不代表实现已完成或测试已通过。

## 需求完整性

- [x] CHK001 已分别定义企业微信问数入口、Web 智能分析入口和治理审计入口的用户边界。  
- [x] CHK002 已定义自然语言问数、必要补问、连续追问和结构化结果返回的主链路需求。  
- [x] CHK003 已定义受控执行链路的关键要求，包括结构化意图、模板约束、白名单编译、AST 安全校验、危险语句拦截和权限范围注入。  
- [x] CHK004 已定义常用查询、最近查询、历史重跑和模板可见性的治理要求。  
- [x] CHK005 已定义导出限制、越权拦截、连接异常、模板执行和安全事件的审计留痕要求。  

## 需求清晰性

- [x] CHK006 “已授权用户”已映射到 CRM 用户、角色和数据权限的关系。  
- [x] CHK007 “结果展示”已明确包含摘要、关键指标、主视图、表格、口径说明和数据更新时间。  
- [x] CHK008 “数据准确”已明确要求不得编造未查询到的结果，并要求多视图数值一致。  
- [x] CHK009 “最近查询重跑”已明确要求按当前最新权限重新执行，而不是沿用旧结果。  
- [x] CHK010 “流式返回”和“会话隔离”已明确要求不同用户之间不能串话、串数据或错发结果。

- [x] CHK012 规格、计划和任务清单均已体现常用查询、最近查询和结果展示能力。  
- [x] CHK013 数据模型、接口契约和审计事件分类均已包含模板执行、历史重跑、连接异常和流式失败事件。  
- [x] CHK014 Web 端职责已统一为“智能分析工作台 + 治理审计后台”，不再只承担治理入口。
