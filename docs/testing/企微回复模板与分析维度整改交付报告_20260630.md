### 2. 补齐参考 HTML 要求的分析维度

本次模板已覆盖：

- 渠道商总数、合作渠道数、活跃业务渠道。
- 客户报备、商机、报价、订单四段业务漏斗。
- 报备到商机、商机到报价、报价到订单的转化率。
- 月度或近三期趋势摘要。
- 渠道集中度，包含前几名金额、占比、集中度洞察。
- 区域覆盖、省份覆盖、大区或区域业务分布。
- 渠道贡献排行，包含报备、商机金额、报价金额、订单金额多口径。
- 渠道结构、合作级别、技术服务商、商机阶段、报价状态、订单状态等分布。
- 明细摘要，企微端展示前几条，避免短消息承载过多明细。
- 数据质量与治理建议，按“问题、影响、建议”表达。

## 三、修复后回复效果要求

企微机器人看板类问题必须满足以下验收点：

- 第一屏先给“核心经营判断”，不是先堆明细。
- 必须说明数据范围、数据来源、金额单位、替代口径。
- 必须展示核心指标和业务漏斗。
- 必须展示渠道集中度、区域分布、渠道排行。
- 必须展示结构分布和数据质量治理建议。
- 订单不足时必须明确说明不能把报价或商机当作真实订单。
- 企微端只展示摘要和前几条明细，完整明细由后续追问或报告入口承接。
