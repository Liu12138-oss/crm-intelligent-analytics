# 最终目标企微长连接交付报告

## 一、交付结论

交付日期：2026-06-30

交付对象：企业微信「渠道 CRM 系统机器人」对接 CRM 智能分析最终目标。

结论：核心闭环已实施到可试运行状态。当前交付不再以正式域名或外部网页报告为必需条件，而是以企业微信长连接内的模板化完整回复为主；网页 HTML 报告仅作为可选增强，在配置 `WECOM_DASHBOARD_BASE_URL` 后启用。

本轮已补齐：

1. 企微回复必须按标准展示模板输出，包含「展示模板」「回复结构」「建议追问」。
2. 看板类问题默认通过企微长连接返回完整核心结果，不再默认依赖 Web 链接。
3. 企微模板卡片仍展示核心 KPI；只有显式配置报告地址时才附加「查看完整看板」跳转。
4. 看板公开报告保存表格列配置，避免公开页只能兜底显示「字段2」「字段5」。
5. 公开报告字段词典补充 `name`、`cooperationLevel`、`techServiceType`、`percentage`、`conversionRate` 等字段中文化。
6. 最终落地方案文档已按“不用域名，直接长连接对接企微”的口径更新。

## 四、当前回复模板标准

企微常规分析回复必须包含：

```text
【展示模板】经营总览卡 / 渠道贡献排行卡 / 商机风险清单卡 等
【回复结构】问题复述 / 数据口径 / 核心指标 / 关键发现 / 建议动作
【建议追问】生成日报；按区域拆分；查看风险清单
正文分析结果
```

企微看板分析回复必须包含：

```text
【展示模板】经营看板卡
【回复结构】问题复述 / 数据口径 / 核心指标 / 关键发现 / 明细摘要 / 建议动作
【数据口径】权限范围、数据来源
【核心指标】报备、商机、报价、订单、金额、覆盖范围等
【明细摘要】Top5 明细
【建议动作】继续追问或拆分分析
【完整看板】长连接内已返回核心看板内容；HTML 报告为可选增强
```
