# 企业微信目录同步运维说明

## 组织范围过宽排查补充

当部门负责人反馈“我是某个区域负责人，但常用查询或问数结果出现其它区域数据”时，先按以下顺序排查：

1. 在企业微信同步数据中确认该用户的 `directLeaderUserids` 下级链是否跨区域、技术线或服务线。
2. 核对该用户 CRM 登录快照中的默认 `departmentIds` 是否为真实业务部门，并确认企业微信部门树能展开到该部门子部门。
3. 查看分析请求审计中的 `ownerIdsBeforeDepartmentPrune`、`prunedOwnerIds`、`defaultDepartmentIds` 和 `departmentPruneSummary`，确认系统是否已按默认业务部门裁剪直属链。
4. 若确需查看其它部门，必须通过数据范围白名单显式授权，并在审计说明中保留授权来源；不得通过修改模板 SQL、最近查询记录或前端筛选条件扩大范围。
5. 对查询模板类问题，同时在查询模板治理页执行 SQL 校验，确认是否存在按商机或合同归属过滤、但按客户归属部门展示团队名称的风险。

## 1. 适用场景

本文档用于说明生产环境中发现企业微信组织架构未同步时的标准处理方式。

典型触发信号包括：

- 企业微信机器人持续提示“未绑定 CRM 身份”
- 新增员工在企业微信可正常发消息，但机器人始终识别不到
- 组织架构调整后，机器人权限范围迟迟未更新
- 团队日报或问数范围仍按旧负责人、旧部门或旧直属上级关系返回

## 2. 现行条件

- 当前系统在生产环境中，若发现企业微信组织架构未同步，必须立即执行目录同步。
- 目录同步应优先通过系统内置的一键同步脚本或受控内部入口完成。
- 不建议在生产现场人工逐页调用企业微信官方通讯录 API。
- 当前生产环境的授权范围应限制为“联软科技集团”，不同步 ROOT 下的全部组织树。
- 若通讯录应用可见范围误放大到 ROOT，全量同步会扩大镜像范围，不符合当前现行条件。

## 3. 执行前检查

执行一键同步前，至少确认：

- 后端服务已启动
- `WECOM_WEB_LOGIN_APP_ID` 已配置
- `WECOM_DIRECTORY_SECRET` 已配置
- 应用存储库可用
- CRM `wx_users`、`wx_user_maps` 写回连接可用
- 当前没有同类同步任务正在执行
- 企业微信官方通讯录应用的可见范围仅包含“联软科技集团”
- 企业微信通讯录应用具备读取部门负责人、成员主部门、成员所属部门和直属上级字段的授权

## 4. 一键同步脚本

默认执行全部资源同步：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-wecom-directory-sync.ps1 -ResourceType all
```

若只需单独同步某一类资源，可选：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-wecom-directory-sync.ps1 -ResourceType department
powershell -ExecutionPolicy Bypass -File .\scripts\run-wecom-directory-sync.ps1 -ResourceType user
```

## 5. 内部状态查询

同步完成后，查询最近状态：

```text
GET /api/v1/internal/wecom-directory-sync/status
```

重点查看：

- `resourceSummaries`
- `checkpoints`
- `runs`

至少确认：

- 最近一次同步时间
- 最近一次同步状态
- 当前 committed cursor
- 是否存在最近失败原因

## 6. 成功判定

一键同步成功后，应至少满足：

- 运行记录中存在对应资源的同步记录
- 状态为 `SUCCEEDED` 或 `SUCCEEDED_WITHOUT_CHANGES`
- 最近同步游标已更新
- 若用户此前因“未绑定 CRM 身份”被拦截，则重新发消息后识别结果发生变化，或至少能命中 CRM 原生 `wx_user_maps -> wx_users` 认证链路
- 组织范围诊断中能看到成员所属部门、直属上级链、默认团队范围和数据范围白名单命中情况

## 7. 失败处理

若一键同步失败：

- 先查看脚本输出的失败原因
- 再查看 `/api/v1/internal/wecom-directory-sync/status`
- 若是配置缺失，先补齐 `WECOM_WEB_LOGIN_APP_ID` 与 `WECOM_DIRECTORY_SECRET`
- 若是分页失败，记录失败页和失败资源后重试
- 若是多命中或未命中问题，转人工确认 CRM 用户绑定关系
- 若是直属上级、部门负责人或成员部门字段缺失，先检查企业微信通讯录应用字段授权，再重新执行目录同步；同步前不得通过静态表格临时扩大团队问数范围

## 8. 注意事项

- 不要直接写 CRM 主库的 `wx_*` 表修数据。
- 不要在生产环境里把企业微信应用 Secret 明文写入仓库。
- 一键同步只是恢复目录镜像，不会修改 CRM 权限模型。
- 团队日报、Web 问数和企业微信问数的数据范围默认依赖当前企业微信组织事实与 CRM 映射，不再以历史静态权限表作为生产权威来源。
- 数据范围白名单只可通过权限中心维护，并必须记录授权对象、授权部门、是否包含子部门、授权原因和审计事件。
- 若后续确需扩大同步范围，必须先更新规格与运维说明，再调整企业微信通讯录应用可见范围。
