# AI 与企业微信机器人核心收敛边界说明

## 1. 文档目的

本文档用于在正式清理代码、页面、接口和规格前，先明确当前项目的收敛边界。

本轮收敛目标不是继续扩展 CRM 智能分析系统，而是把系统临时收敛为两块稳定底座：

1. 对接 AI：提供可配置、可验证、可审计的服务端 AI 调用能力。
2. 对接企业微信机器人：提供稳定的企业微信消息接入、会话维护和 AI 回复通道。

除上述两块外，CRM 问数、渠道 CRM 分析、合同评审、日报、通知、写回、治理审计和 Web 工作台等能力全部视为后续业务能力，默认不进入本轮核心范围。

## 2. 当前系统膨胀来源

当前仓库已经从最初的“企业微信 AI 问数 + Web 智能分析”扩展成统一 CRM 智能业务平台，主要能力包括：

- 企业微信机器人问数。
- Web 智能分析工作台。
- 查询模板、最近查询、导出和结果追问。
- CRM 官方 API、受控数据库路径、渠道 CRM 标准 OpenAPI、Markdown 快照和分析库。
- 企业微信新增客户、新增商机、跟进写回、日报、主动通知。
- 智能合同审核、合同审核标准包、CRM 待审批合同接入。
- 权限中心、连接策略、AI 配置治理、审计中心和语义资产治理。

这些能力中，很多不是“AI 接入”或“企业微信机器人接入”的必要组成，而是基于 AI 和企业微信入口继续长出来的业务执行能力。若不先定义边界，后续每次保留“企业微信机器人”都会自然带回 CRM 创建、日报、跟进写回和渠道 CRM 问数等能力，导致收敛失败。

## 3. 收敛后的产品定位

### 3.1 新定位

收敛后系统定位为：

> 企业微信内的 AI 助手接入底座，负责让已配置的企业微信机器人安全接入系统，并通过统一 AI 调用层生成回复。

该定位强调“通道 + AI 能力”，不默认承诺 CRM 数据查询、合同审核、业务写回或经营分析。

### 3.2 非定位

收敛后系统暂不定位为：

- CRM 智能分析系统。
- 渠道 CRM 经营看板。
- 合同评审系统。
- 销售日报系统。
- CRM 写回助手。
- 企业微信通用办公助手。
- 治理审计后台。
- 数据分析报表平台。

这些能力可以作为后续插件化或独立 OpenSpec 变更重新接入，但不属于当前核心。

## 4. 核心能力边界

### 4.1 对接 AI 的边界

本轮保留的 AI 能力包括：

| 能力 | 保留说明 |
| --- | --- |
| AI Profile 配置 | 保留 OpenAI 兼容 HTTP 的服务地址、模型、密钥、协议、结构化输出模式、超时和推理等级。 |
| 环境变量兜底 | 没有后台激活 Profile 时，允许继续从环境变量读取默认 AI 配置。 |
| 密钥保护 | 继续要求密钥只在服务端保存和解密，不回显明文，不写入日志、文档或测试快照。 |
| 健康检查 | 保留最小真实 HTTP smoke test，确认模型网关可用后再作为生效配置。 |
| 统一调用门面 | 业务层只调用统一 AI 执行服务，不直接依赖具体厂商 SDK。 |
| 文本回复 | 支持普通文本生成，用于企业微信机器人回复。 |
| 结构化理解 | 仅保留足以支撑机器人入口识别、帮助提示和安全拒答的最小结构化理解。 |
| 调用审计摘要 | 记录 provider、model、profileId、是否 fallback、耗时和脱敏错误摘要。 |

本轮不保留为核心的 AI 能力包括：

| 能力 | 暂缓原因 |
| --- | --- |
| CRM 经营问数意图解析 | 需要 CRM 数据、指标、权限和结果口径配套，不属于 AI 接入底座。 |
| 复杂 Text-to-SQL | 需要数据库白名单、SQL Guard、权限注入和审计闭环，超出本轮范围。 |
| 合同审核 AI 审查 | 依赖合同标准包、合同快照、任务状态和产物生成，属于合同评审业务。 |
| 日报摘要和团队汇总 | 依赖 CRM 当日源数据、通知调度和接收人权限，属于日报业务。 |
| 新增客户/商机字段抽取 | 依赖 CRM 写回接口、确认流和对象权限，属于 CRM 执行业务。 |
| 模型平台化扩展 | 当前只要求 OpenAI 兼容 HTTP，不建设任意请求模板、脚本适配或多模型市场。 |

### 4.2 对接企业微信机器人的边界

本轮保留的企业微信机器人能力包括：

| 能力 | 保留说明 |
| --- | --- |
| 消息接入 | 接收企业微信机器人回调或 SDK 消息。 |
| 签名和来源校验 | 在进入 AI 处理前完成签名、来源和基础结构校验。 |
| 消息幂等 | 对同一个 messageId 做去重，避免重复回调导致重复 AI 调用。 |
| 会话上下文 | 按企业微信会话和发送人维护短上下文，支持连续对话。 |
| AI 回复 | 将用户文本转交统一 AI 调用层，并将回复返回企业微信。 |
| 帮助提示 | 用户询问能力、打招呼或输入空内容时，返回简洁帮助说明。 |
| 安全拒答 | 用户提出 CRM 查询、写回、合同审核、导出、日报、通知等暂未启用能力时，返回明确中文提示。 |
| 降级提示 | AI 网关不可用、配置缺失或企业微信通道失败时，返回清晰降级提示。 |

本轮不保留为核心的企业微信机器人能力包括：

| 能力 | 暂缓原因 |
| --- | --- |
| CRM 经营分析问数 | 依赖 CRM 或渠道 CRM 数据链路，不属于机器人通道底座。 |
| 客户/商机列表和详情查询 | 依赖 CRM 权限、对象查询和展示口径。 |
| 唯一商机后的跟进写回 | 依赖对象唯一性、写回确认、CRM 官方 API 和审计。 |
| 新增客户 | 依赖字段白名单、必填校验、CRM 创建接口和确认流。 |
| 新增商机 | 依赖字段白名单、产品/客户/代理商匹配、CRM 创建接口和确认流。 |
| 个人日报和团队日报 | 依赖 CRM 源数据、日报任务、通知调度和权限。 |
| 主动通知 | 依赖发送任务、真实发送开关、接收人解析、幂等、重试和留痕。 |
| 企业微信目录同步 | 如果只是机器人通道，不需要默认同步完整组织架构；可保留为后续认证或权限能力。 |

### 4.3 最小用户可见能力

收敛后，企业微信机器人默认只承诺以下用户可见能力：

1. 用户发送普通问题，机器人通过 AI 生成自然语言回复。
2. 用户询问“你能做什么”，机器人说明当前只支持 AI 对话和基础帮助。
3. 用户提出 CRM 数据、合同审核、日报、创建客户、写跟进等请求，机器人说明当前能力未启用，并提示后续可按模块开通。
4. AI 不可用时，机器人说明“当前 AI 服务暂不可用，请稍后再试或联系管理员检查配置”。

机器人不得在没有业务数据链路时伪造 CRM 结果、合同审核结论、日报内容或客户/商机状态。

## 5. 分层边界

收敛后的推荐分层如下：

```text
企业微信用户
  |
  v
企业微信机器人通道层
  - 接收消息
  - 验签
  - 幂等
  - 会话
  - 回复发送
  |
  v
AI 对话编排层
  - 帮助意图
  - 普通对话
  - 暂未启用能力拒答
  - 上下文裁剪
  |
  v
统一 AI 调用层
  - Profile 解析
  - OpenAI 兼容 HTTP 调用
  - 健康检查
  - 错误脱敏
  |
  v
本地运行态
  - 会话快照
  - 消息幂等
  - 脱敏调用摘要
```

暂不进入主链的业务层：

```text
CRM 数据层
渠道 CRM OpenAPI
合同评审引擎
日报编排
通知中心
查询模板
导出
治理审计后台
管理报表
```

## 6. 依赖边界

### 6.1 核心必须依赖

| 依赖 | 用途 |
| --- | --- |
| 企业微信机器人凭证 | 接收与回复企业微信机器人消息。 |
| AI Profile 或 AI 环境变量 | 调用模型生成回复。 |
| 本地应用状态或轻量存储 | 保存会话、幂等和配置快照。 |
| 基础日志 | 排查消息接入和 AI 调用问题。 |

### 6.2 核心不应强依赖

| 依赖 | 收敛要求 |
| --- | --- |
| CRM 只读库 | 不应影响企业微信机器人 AI 对话可用性。 |
| CRM 写回库 | 默认不加载，不参与机器人主链。 |
| CRM 官方 API | 默认不调用；后续业务能力开启时再接入。 |
| 渠道 CRM 标准 OpenAPI | 默认不调用；只作为后续渠道分析能力。 |
| 合同审核标准包 | 默认不加载；合同评审启用时再加载。 |
| 日报调度 | 默认不启动；日报业务启用时再注册。 |
| 通知任务队列 | 默认不启动真实发送。 |
| Web 治理后台 | 不应成为机器人运行的必要条件。 |

## 8. 业务请求处理边界

### 8.1 允许处理

| 用户请求 | 处理方式 |
| --- | --- |
| “你好” | 返回简洁问候和当前能力说明。 |
| “你能做什么” | 返回当前只支持 AI 对话和基础帮助。 |
| “帮我写一段通知文案” | 可作为普通 AI 文本任务处理。 |
| “解释一下这段话” | 可作为普通 AI 文本任务处理。 |
| “总结下面这段内容” | 可作为普通 AI 文本任务处理，但不得承诺业务事实校验。 |

### 8.2 明确拒绝或提示未启用

| 用户请求 | 默认提示方向 |
| --- | --- |
| “查本月商机金额” | 当前未启用 CRM 数据分析能力。 |
| “查渠道商排名” | 当前未启用渠道 CRM 分析能力。 |
| “帮我新增客户” | 当前未启用 CRM 写回能力。 |
| “帮我写跟进到 CRM” | 当前未启用 CRM 跟进写回能力。 |
| “生成今天日报” | 当前未启用日报能力。 |
| “审核这份合同” | 当前未启用合同评审能力。 |
| “导出全部客户” | 当前未启用导出能力，且不能通过机器人导出敏感数据。 |

## 9. 收敛原则

### 9.1 入口先收敛，代码后清理

先通过导航、路由、接口注册、功能开关和意图枚举收住入口，再逐步清理不再使用的服务代码。不要第一步就物理删除大模块，否则容易误伤仍被测试、启动或其它模块依赖的公共能力。

### 9.2 通道和业务动作分离

企业微信机器人必须拆成两层：

1. 通道层：只管消息接入、校验、幂等、会话和回传。
2. 业务动作层：CRM 问数、创建客户、写跟进、日报、通知等能力。

收敛后默认只启用通道层和普通 AI 对话，业务动作层全部关闭。

### 9.3 AI 核心不绑定业务域

AI 调用层只知道如何调用模型，不应知道 CRM、合同、日报或渠道商。业务 prompt、业务结构化 schema 和业务字段抽取应作为后续能力包接入。

### 9.4 关闭能力必须可解释

用户触发已关闭能力时，不应返回 500、无权限、无数据或沉默失败。应明确提示该能力当前未启用，并说明当前可用能力。

### 9.5 不影响其它模块的默认做法

对暂缓模块采用以下策略：

- 默认隐藏入口。
- 默认关闭任务调度。
- 默认不注册主动发送。
- 默认不调用外部接口。
- 保留数据和配置文件，不清空历史记录。
- 保留回滚路径，允许后续按模块重新启用。

### 9.6 当前实现保护线

本边界文档描述的是核心收敛目标。当前默认装配已经改为 AI 配置、企业微信核心机器人、登录会话、能力快照和最小审计查询；历史源码仍保留在仓库中，作为回滚和后续模块化恢复参考，不代表当前可用能力。

收敛执行必须遵守以下保护线：

1. 不得把未装配的历史 Controller 当作当前可用接口对外承诺。
2. 不得在默认路由、导航、帮助文案或机器人回复中暗示 CRM 问数、合同评审、日报、写回、导出等能力已启用。
3. 企业微信核心机器人只使用验签、来源校验、senderId 和轻量会话完成普通 AI 对话；不得要求 CRM 数据源存在后才能回复普通问题。
4. 已关闭业务请求只能返回“能力未启用”提示，不得进入半执行链路或编造内部业务数据。
5. 若后续恢复任一历史能力，必须重新补齐 Controller 装配、权限、审计、测试和文档入口。

## 11. 验收口径

完成边界收敛后，至少应满足：

1. 不配置 CRM 数据源时，企业微信机器人仍可完成普通 AI 对话。
2. 不配置渠道 CRM OpenAPI 时，企业微信机器人启动和回复不受影响。
3. 不配置合同审核标准包时，企业微信机器人启动和回复不受影响。
4. 用户请求 CRM 问数时，机器人给出“能力未启用”的中文提示，不编造数据。
5. 用户请求合同评审时，机器人给出“能力未启用”的中文提示，不生成风险结论。
6. 用户请求新增客户、商机或写跟进时，机器人给出“能力未启用”的中文提示，不调用 CRM 写回接口。
7. 应用启动时不自动启动日报催报、团队汇总或主动通知真实发送。
8. AI Profile 配置错误时，机器人返回 AI 服务不可用提示，不暴露密钥、内部地址或堆栈。
9. 暂缓模块的历史数据、配置和文档不被删除，后续可以按独立变更重新启用。

其中第 1 条是目标验收口径。若尚未完成企业微信最小身份模式和数据源懒加载改造，当前实现可能仍会因为 CRM 身份源不可用而阻断普通 AI 对话，必须作为收敛实施前置风险处理。

## 12. 后续重新扩展规则

后续若要重新接入业务能力，必须按以下顺序推进：

1. 单独定义能力边界，例如“只读 CRM 问数”或“合同评审”。
2. 明确依赖的数据源、权限来源、审计要求和失败降级。
3. 新增能力开关，默认关闭。
4. 在企业微信机器人中新增独立业务动作处理器，不修改通道层。
