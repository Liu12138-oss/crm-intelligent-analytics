## 2. 高频问法 Top 40

说明：

1. 时间未指定时，默认不强行查询全量，应追问或使用系统默认周期并在结果中说明。
2. 问到“我/我们/本区域/本渠道”时，必须注入当前 CRM 权限。
3. 问到“活跃、有效、成交、转化、超期”等概念时，按第 3 节口径解释，不允许模型自由发挥。

| 序号 | 高频问法 | 期望对象 | 指标/维度 | 默认处理 |
|---:|---|---|---|---|
| 1 | 上个月华东/山东渠道签了多少单？ | 订单 | 订单数、订单金额，按区域/渠道 | 时间=上一自然月，区域按标准字典匹配。 |
| 2 | 最近 30 天没活跃的客户有哪些？ | 客户 | 最近活动时间、负责人、渠道 | 活跃按报备/商机/报价/订单最近更新时间。 |
| 3 | 本月新增多少报备？ | 报备 | 报备数，按区域/渠道/负责人 | 时间=本自然月。 |
| 4 | 本月新增商机金额是多少？ | 商机 | 商机数、商机金额 | 金额按 `amount` 优先。 |
| 5 | 哪些商机超过两周没有更新？ | 商机 | 商机列表、负责人、停滞天数 | 排除已赢单/已失单等终态。 |
| 6 | 本区域哪些客户还没建商机？ | 客户/报备/商机 | 客户列表、报备状态 | 有报备无商机。 |

## 3. 复杂问题期望结果

| 序号 | 复杂问题 | 查询计划期望 | 输出期望 |
|---:|---|---|---|
| 1 | 分析山东区本季度从报备到订单的转化情况，并列出流失最多的环节。 | 子任务：报备数、商机数、报价数、订单数、阶段转化率、断点客户样例。 | 摘要 + 漏斗表 + 流失环节 + 可追溯样例 ID。 |
| 2 | 对比 P001 渠道和 P002 渠道近 30 天活跃度。 | 子任务：报备、商机、报价、订单、最近活动时间；按渠道分组。 | 两渠道对比表；若当前角色无权看 P002，应提示权限不足。 |
| 3 | 找出有报价但未下单的客户，并按报价金额排序。 | 子任务：报价客户集合、有效订单客户集合、差集、金额排序。 | 客户、报价 ID、报价金额、报价时间、负责人。 |
| 4 | 哪些商机超过两周未更新，并且金额较高？ | 子任务：商机更新时间、终态排除、金额排序。 | 商机列表、停滞天数、金额、负责人、建议动作。 |
| 5 | 本月订单金额按区域、渠道分别排名。 | 子任务：订单时间过滤、按区域聚合、按渠道聚合。 | 指标卡 + 两张排行表；金额单位统一。 |
| 6 | 本区域客户从报备到报价平均需要多久？ | 子任务：同客户/链路匹配报备、商机、报价时间差。 | 平均天数、中位数、样例链路；链路不足时说明样本数。 |
| 7 | 找出本渠道可能流失的客户。 | 子任务：客户最近活动、报价未下单、商机停滞、报备无后续。 | 风险客户列表 + 风险原因 + 建议动作。 |
| 8 | 本季度复购情况如何？ | 子任务：有效订单按客户归并，识别 2 单及以上客户。 | 复购客户数、复购率、客户明细；样本不足时说明。 |
| 9 | 分析报价转订单失败的主要原因。 | 子任务：报价状态、订单关联、报价金额、时间间隔。 | 当前 CRM 无失败原因字段时，应说明“缺少失败原因字段”，只给结构性分析。 |
| 10 | 按负责人看商机推进效率。 | 子任务：负责人维度商机数、报价数、订单数、转化率。 | 负责人排行；员工角色只返回本人。 |
| 11 | 哪些渠道业务量高但订单少？ | 子任务：报备/商机高、订单低，计算差异。 | 渠道列表 + 漏斗断点 + 建议跟进。 |
| 12 | 生成本月经营简报。 | 子任务：订单、报价、商机、报备、渠道贡献、风险项、权限范围。 | 企微摘要 + Web 完整报告链接/附件。 |

低置信追问规则：

| 不明确项 | 应追问 |
|---|---|
| 时间缺失 | “请确认看本月、上月、近 30 天还是自定义时间？” |
| 区域名称模糊 | “请确认区域是山东区、华东/大东区，还是全部权限范围？” |
| 活跃口径不明确 | “活跃按跟进、更新时间、报备/商机/报价/订单动作中的哪一种？” |
| 有效订单口径不明确 | “是否排除 pending/processing，仅统计 completed/confirmed？” |
| 导出明细 | “请确认是否导出，以及是否需要脱敏字段。” |

当前 SIT 库真实订单数为 2，因此完整链路样例当前提供 2 条。两条均为直接 ID 链路：`regId -> oppId -> quoteId -> orderId`。

| 客户 | 报备 ID | 商机 ID | 报价 ID | 订单 ID | 订单状态 | 订单金额 | 匹配规则 |
|---|---|---|---|---|---|---:|---|
| 测试客户 | `REG-1779870955652` | `OPP-1779870988696` | `QT-1779871010532` | `ORD-1779871019160` | `processing / 处理中` | 10875 | `direct_id_chain` |
| 山东碧海机械科技有限公司 | `REG-2026-950281` | `OPP-1779847047385` | `QT-1779870200797` | `ORD-1779947494707` | `pending / 待处理` | 34575 | `direct_id_chain` |

链路匹配优先级：

```text
1. 订单 quoteId -> 报价 id
2. 订单 oppId 或报价 oppId/oppIds -> 商机 id
3. 订单 regId 或商机 regId -> 报备 id
4. 若直接 ID 不完整，才按 customerId / 统一信用代码 / 标准化客户名兜底
```

AI-agent 建议：

1. 漏斗统计可以用总量计算。
2. 完整链路样例只用于链路模板和报告展示回归，不建议用当前 2 条样例代表真实业务分布。
3. 后续如需更丰满的漏斗演示，建议业务侧补 5 到 10 条完整链路样例。

## 5. 敏感字段清单和明细展示边界

### 5.1 字段级边界

| 字段/类型 | 示例字段 | 是否允许分析 | 企微展示 | Web 展示 | 导出 | 处理规则 |
|---|---|---|---|---|---|---|
| 密码/密钥/Token | `password`, `appSecret`, `token`, `secret`, `privateKey` | 否 | 禁止 | 禁止 | 禁止 | 不入分析库、不出日志、不展示。 |
| 手机/电话 | `phone`, `mobile`, `contacts` | 否 | 脱敏 | 权限展示 | 脱敏导出 | `138****8000`。 |
| 联系人 | `contact`, `contacts` | 否 | 脱敏或仅姓名 | 权限展示 | 脱敏导出 | 非必要不在企微摘要展示。 |
| 邮箱 | `email` | 否 | 脱敏 | 权限展示 | 脱敏导出 | `a***@example.com`。 |
| 地址 | `address`, `deliveryAddr` | 否 | 不展示或省市级 | 权限展示 | 脱敏导出 | 企微默认不展示详细地址。 |
| 统一信用代码 | `creditCode` | 可做关联键 | 脱敏 | 权限展示 | 脱敏导出 | 仅用于匹配，不建议明文展示。 |
| 金额 | `amount`, `total`, `totalAmount`, `discountAmount` | 是 | 权限展示 | 权限展示 | 按角色控制 | 管理角色可看汇总；明细按权限。 |
| 客户名 | `customer`, `customerName`, `name` | 是 | 权限展示 | 权限展示 | 权限导出 | 对有权限角色可展示。 |
| 服务商名 | `partnerName` | 是 | 权限展示 | 权限展示 | 权限导出 | 对有权限角色可展示。 |
| 产品模块 | `products`, `workloadSnapshot`, `matchedFeatureNames` | 是 | 摘要展示 | 可展开 | 可选导出 | 企微避免长列表刷屏。 |
| 审计字段 | `createdBy`, `updatedBy`, `statusHistory` | 是 | 默认不展示 | 可展示 | 管理员导出 | 只做追溯和排障。 |

## 9. P5 验收问题清单

建议每类角色至少验收以下问题：

| 角色 | 验收问题 | 期望报告形态 |
|---|---|---|
| 超管 | 本月整体经营情况如何？ | 企微摘要 + Web 完整报告，含全量指标和排行。 |
| 超管 | 报备到订单转化漏斗是多少？ | 漏斗图/表 + 两条完整链路样例。 |
| 超管 | 哪些渠道贡献最高？ | 渠道排行 + 订单金额 + 报备/商机/报价/订单数。 |
| 超管 | 哪些商机超过两周没更新？ | 风险列表 + 负责人 + 建议动作。 |
| 区管 | 山东区本月订单金额多少？ | 只看山东区，显示区域权限范围。 |
| 区管 | 山东区哪些渠道长期没报备？ | 本区域渠道列表，不出现外区数据。 |
| 区管 | 山东区报价转订单率是多少？ | 本区域漏斗，不跨区。 |
| 渠道账号 | 我们渠道本季度订单情况？ | 只看 P001 渠道链路。 |
| 渠道账号 | 我们渠道哪些客户有报价未下单？ | 本渠道客户列表，敏感字段脱敏。 |
| 渠道账号 | 我们渠道商机转报价率？ | 渠道内商机/报价转化。 |
| 销售员工 | 我负责的商机有哪些超两周没更新？ | 只看本人 S022 相关商机。 |
| 销售员工 | 我本月报价金额多少？ | 当前若无报价，返回 0 并说明权限范围。 |
| 销售员工 | 我负责的客户哪些还没有订单？ | 本人客户列表，不越权。 |

验收通过标准：

1. 结果里明确显示“权限范围、时间范围、数据截至时间”。
2. 企微摘要能看懂主要结论，复杂明细进入 Web 或附件。
3. 越权数据不出现在摘要、表格、图表、附件和导出里。
4. 敏感字段按本文第 5 节处理。
5. 与 CRM 页面/导出结果对账差异可解释。
