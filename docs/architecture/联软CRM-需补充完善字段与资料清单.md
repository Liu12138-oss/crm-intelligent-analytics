# 联软CRM-需补充完善字段与资料清单

## 1. 文档目的

本文档用于发给联软渠道 CRM 对接方，说明为了让 AI-agent 完整支持“渠道 CRM 系统接入后的智能分析”，联软侧还需要确认或补充的标准 OpenAPI 字段、权限口径、样例数据、错误码和企微身份映射资料。

本清单不要求 AI-agent 直接读取联软 SQLite。联软 CRM 可以继续以 SQLite 作为内部存储，但对外需要通过稳定标准 OpenAPI 暴露对象、字段、权限和字典口径。

## 2. 当前对接目标

当前双方目标不是只完成接口连通，而是形成以下完整闭环：

```text
真实登录 / 企业微信身份
  -> CRM 用户身份落位
  -> 当前用户权限范围
  -> 标准 OpenAPI 只读取数
  -> AI-agent 本地受控聚合
  -> AI 经营分析报告
  -> Web / 企业微信展示
  -> 审计与诊断
```

因此联软侧需要重点保证三类信息完整：

1. 六类对象字段完整。
2. 不同角色权限裁剪口径明确。
3. 字典、样例、错误码和关联关系稳定。

## 3. 标准 OpenAPI 总体要求

| 项目 | 需要联软确认或补充 | 用途 |
|---|---|---|
| Base URL | 正式联调地址、生产地址、是否区分内外网 | AI-agent 标准 OpenAPI 配置 |
| 鉴权模式 | `appKey/appSecret`、用户态 `user_token`、多 client 映射三者最终采用哪种 | 决定多角色权限实现方式 |
| Token 有效期 | `expiresIn`、提前刷新建议时间 | 避免联调中途 token 失效 |
| 分页规范 | `pageNo`、`pageSize`、`total` 是否所有列表一致 | 本地安全聚合和分页拉取 |
| 排序规范 | 支持哪些 `sortBy` 字段，默认排序是什么 | 服务商加入时间、更新时间排序 |
| 筛选规范 | 每类对象支持哪些筛选参数 | 时间、区域、状态、服务商筛选 |
| 字段命名 | 是否长期稳定使用当前字段名 | 防止后续字段改名导致分析失效 |
| 字典接口 | 角色、服务商等级、状态、阶段、报价状态、订单状态 | 用户可读中文展示和 AI 语义归一 |
| 错误码 | 鉴权失败、权限不足、参数错误、限流、服务异常、字段不支持 | 页面和企微友好提示 |
| `requestId` | 每个响应是否返回请求 ID | 联调排障和审计追踪 |

## 4. 六类对象字段完整性清单

### 4.1 用户 `users`

|---|---|---|---|
| `id` | 必需 | CRM 用户唯一 ID | 登录落位、权限、审计 |
| `username` | 必需 | 登录名或账号名 | 诊断展示 |
| `name` | 必需 | 用户姓名 | 报告展示 |
| `role` | 必需 | 角色编码 | 权限视角 |
| `roleName` | 建议 | 角色中文名 | 页面和企微展示 |

| `bigRegion` | 建议 | 所属大区 | 大区分析 |

| `partnerName` | 角色相关必需 | 绑定服务商名称 | 展示和诊断 |
| `status` | 必需 | 启用、禁用、待审批等 | 登录和权限阻断 |
| `wecomUserId` | 企微必需 | 企业微信用户 ID | 企微用户映射 |

## 5. 统计分析接口补充建议

为提升性能和口径一致性，建议联软侧在对象列表之外补充统计接口，至少覆盖以下能力。

| 接口能力 | 建议路径 | 说明 |
|---|---|---|
| 服务商贡献统计 | `GET /analytics/partners/contribution` | 按服务商返回报备、商机、报价、订单数量和金额 |
| 服务商画像统计 | `GET /analytics/partners/profile` | 按等级、状态、是否技术服务商、区域返回分布 |
| 转化漏斗统计 | `GET /analytics/funnel` | 报备转商机、商机转报价、报价转订单 |
| 区域经营统计 | `GET /analytics/regions/contribution` | 按区域 / 大区聚合金额和数量 |
| 负责人经营统计 | `GET /analytics/owners/contribution` | 按负责人聚合金额和数量 |

如果短期不能提供统计接口，AI-agent 可以继续用列表接口分页拉取后本地安全聚合，但联软侧需要保证分页、筛选、字段和权限裁剪稳定。

## 6. 权限口径与角色资料

请联软侧确认以下角色权限矩阵，并提供每类角色的测试账号和预期结果。

| 角色视角 | 应返回的数据范围 | 需要确认 |
|---|---|---|
| 超管 | 全部区域、全部服务商、全部对象 | 是否完全不裁剪 |
| 区域负责人 | 所属区域 / 大区内数据 | 区域字段以哪个字段为准 |
| 渠道管理员 | 绑定服务商及允许的下级服务商 | 是否包含下级服务商 |
| 普通员工 | 自己负责、创建、分配或授权的数据 | 负责人字段优先级 |
| 禁用 / 待审批账号 | 不允许登录或不允许查询 | 错误码和提示 |

每类测试账号建议提供：

| 信息 | 说明 |
|---|---|
| CRM 用户 ID | 用于身份落位 |
| 登录账号 | 用于真实登录 |
| 角色编码和角色名 | 用于权限判断 |
| 区域 / 大区 | 用于区域视角验证 |
| 绑定服务商 | 用于渠道视角验证 |
| 可见样例对象 | 至少各提供 3 条可见样例 |
| 不可见样例对象 | 至少各提供 3 条不可见样例 |
| 预期统计结果 | 同一问题在不同角色下的预期数量和金额 |

## 8. 样例数据要求

请联软侧提供一批覆盖真实业务关系的样例数据，不需要提供敏感客户资料，可使用脱敏名称，但关系必须真实。

| 场景 | 样例要求 |
|---|---|
| 服务商画像 | 至少 20 家服务商，覆盖不同等级、状态、是否技术服务商 |
| 山东区域商机 | 至少 10 条商机，覆盖多个服务商、金额、阶段 |
| 报备转商机 | 至少 5 条可通过 `regId` 关联的链路 |
| 商机转报价 | 至少 5 条可通过 `oppId` 关联的链路 |
| 报价转订单 | 至少 5 条可通过 `quoteId` 关联的链路 |
| 多角色权限 | 同一批样例在超管、区域、渠道、员工视角下返回不同范围 |

## 9. 错误码与异常响应

请联软侧确认所有标准 OpenAPI 的错误响应格式一致，建议包含：

```json
{
  "code": 40101,
  "message": "token 已过期",
  "requestId": "req_xxx"
}
```

需要明确的错误类型：

| 场景 | 期望 |
|---|---|
| appKey 无效 | 明确返回 client 无效 |
| appSecret 错误 | 明确返回鉴权失败 |
| token 过期 | 明确返回 token 过期 |
| 无权限访问资源 | 明确返回无权限，不伪装成空数据 |
| 字段不支持 | 明确返回字段不支持 |
| 筛选参数不支持 | 明确返回参数错误 |
| 服务异常 | 返回可追踪 `requestId` |
| 接口限流 | 返回限流错误和建议重试时间 |

如果要保证企业微信小助手与 Web 同账号同权限，需要联软侧补充以下资料：

| 信息 | 说明 |
|---|---|
| CRM 用户 ID 与企微 `userid` 映射 | 企微消息进入后定位 CRM 用户 |
| CRM 部门与企微部门映射 | 后续组织权限和通知 |

| 离职、禁用、转岗同步规则 | 避免权限滞后 |

## 11. 优先级建议

### P0：必须补齐

1. `partners.isTechnicalServiceProvider` 或等价字段。
2. 六类对象的金额、时间、状态、区域、服务商关联字段。
3. 四类角色测试账号和权限预期结果。
4. 字典接口中的服务商等级、状态、商机阶段、报价状态、订单状态。
5. 错误码和 `requestId`。

### P1：建议补齐

1. 服务商简称。
2. 大区字段。
3. 报备、商机、报价、订单的完整转化链路关联字段。
4. 服务商树字段。
5. 统计接口。

### P2：后续增强

1. 企微组织映射。
2. 变更增量同步字段。
3. 复杂统计预聚合接口。
4. 数据质量诊断接口。

## 12. 请联软侧优先回复的问题

1. `partners` 是否可以补充 `isTechnicalServiceProvider`？
2. “服务商加入时间”最终以 `createdAt` 还是其它字段为准？
3. 标准 OpenAPI 是否支持真实登录 `user_token` 访问六类对象？
4. 当前 4 组 client 是否分别绑定超管、区域、渠道、员工视角？
5. 标准 OpenAPI 的无权限结果是返回空数组，还是返回无权限错误？
6. 服务商下级关系是否需要在渠道管理员视角中自动包含？
7. 金额字段单位是元、万元，还是其它单位？
8. 区域字段是否统一为 `region`，大区是否统一为 `bigRegion`？
9. 是否可以提供服务商画像和转化漏斗统计接口？
10. 是否可以提供 CRM 用户与企微 `userid` 映射？
