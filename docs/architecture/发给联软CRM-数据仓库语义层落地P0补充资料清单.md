# 发给联软 CRM：数据仓库语义层落地 P0 补充资料清单

> 日期：2026-06-09  
> 用途：用于 AI-agent 正式建设 MySQL 分析库、ODS 落库、标准数据模型、语义层字段目录和受控 Text-to-SQL 执行器前，向联软 CRM 补齐 P0 必需资料。  
> 说明：本文只列资料项和确认口径，不需要提供明文 `appSecret`、Token、数据库密码、机器人 Secret 或生产库连接信息。

## 1. 我方当前落地计划

我方下一步将按以下链路建设：

```text
联软 CRM 标准 OpenAPI
  -> AI-agent 同步任务
  -> AI-agent MySQL 分析库
  -> ODS 原始层
  -> DWD 标准明细层
  -> DWS 汇总服务层
  -> 语义层字段 / 指标目录
  -> AI 查询计划
  -> 程序受控 SQL 校验、权限注入、审计
  -> 企微 / Web 智能分析报告
```

边界确认：

- 第一阶段仍以联软标准 OpenAPI 作为正式同步源和权限来源。
- 不要求联软 CRM 将当前 SQLite 改为 MySQL。
- 不直接读取联软生产运行中的 SQLite 文件。
- Text-to-SQL 只在 AI-agent 分析库或受控只读语义视图执行。
- SQL 安全、权限注入、脱敏、超时、LIMIT 和审计由 AI-agent 侧完成。
