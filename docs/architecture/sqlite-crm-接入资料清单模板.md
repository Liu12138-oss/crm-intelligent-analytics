# SQLite CRM 接入资料清单模板

## 1. 文档用途

本文档用于向第三方 CRM 厂商、实施方或客户技术团队收集 SQLite CRM 对接所需资料，便于我方先完成以下工作：

- 评估现有 SQLite CRM 是否适合直接接入本项目
- 判断应采用“SQLite 直连适配”“SQLite 同步到 MySQL”“补充中间兼容层”中的哪一种方案
- 先按对方现有数据结构调整本项目的只读查询、组织权限、企业微信身份映射和后续写回方案

请对方尽量一次性提供完整资料，避免后续因字段口径、关联关系或权限规则不明确而反复返工。

## 2. 提交说明

### 2.1 提交方式

建议对方按“文档说明 + 数据样例 + 数据库文件”三部分打包提交：

- 一份说明文档
- 一份或多份脱敏后的 SQLite 测试库文件
- 一份字段字典或表结构导出文件

### 2.2 脱敏要求

请对以下内容做脱敏处理后再提供：

- 客户名称、联系人姓名、手机号、邮箱
- 合同编号、统一社会信用代码、开户地址等敏感标识
- 员工手机号、身份证号、邮箱、工号等敏感信息
- 账号密码、Token、密钥、证书、内部地址

脱敏后仍需保留以下特征，方便我方做真实结构评估：

- 字段名、字段类型、字段长度
- 主键、外键、唯一键、索引结构
- 真实的状态值、枚举值、空值分布
- 合理的数据量级和关联关系

### 2.3 提交范围

本轮至少覆盖以下能力相关数据：

- 用户与部门
- 客户
- 商机
- 合同
- 跟进记录
- 审批记录
- 企业微信身份映射基础
