## 3. 请对方补齐的正式真实登录最小项

### 3.1 正式真实登录入口参数

请对方明确以下参数：

| 项目 | 是否必需 | 说明 |
|---|---|---|
| 正式真实登录 `Base URL` | 是 | 例如 `http://10.18.16.114:3000`，请明确正式环境实际地址 |
| 正式真实登录 `loginPath` | 是 | 请明确是否就是 `/api/v2/auth/login`，若不是请给正式路径 |
| 是否必须传 `corp_id` | 是 | 若必须，请给固定值；若不需要，也请明确说明“不需要” |
| `device` 固定值 | 建议 | 若有固定要求，请提供；否则我方默认使用 `open_api` |
| `version_code` 固定值 | 建议 | 若有固定要求，请提供；否则我方默认使用 `9.9.9` |
| `timeout` 建议值 | 建议 | 请提供毫秒值，便于我方统一超时配置 |

---

### 3.2 正式真实登录响应契约

请对方补齐正式真实登录接口的成功/失败样例，至少覆盖以下内容：

| 项目 | 是否必需 | 说明 |
|---|---|---|
| 登录成功响应样例 | 是 | 至少包含 `code`、`message`、`data.user_id`、`data.user_token` |
| 登录失败响应样例 | 是 | 至少覆盖“账号或密码错误”“账号不存在”“账号禁用/冻结” |
| 成功判定规则 | 是 | 例如 `code = 0` 代表成功 |
| 错误码语义说明 | 建议 | 便于我方把错误提示转成业务可读中文 |
| `user_token` 有效期 | 建议 | 便于后续评估会话续期策略 |

---

### 3.3 登录成功后的身份落位数据源

我方正式真实登录成功后，还必须根据 `user_id` 落位出用户身份、角色、组织、部门和权限范围，否则不能安全建立系统会话。

请对方二选一明确提供：

#### 方案 A：继续提供只读库

| 项目 | 是否必需 | 说明 |
|---|---|---|
| 只读库主机 | 是 | 仅用于身份和权限只读解析 |
| 只读库端口 | 是 | 例如 `3306` |
| 只读库库名 | 是 | 仅提供目标库名 |
| 只读库账号 | 是 | 需为最小只读权限 |
| 只读库密码 | 是 | 请通过安全渠道单独提供 |

#### 方案 B：提供正式身份查询 API

| 项目 | 是否必需 | 说明 |
|---|---|---|
| 身份查询 API `Base URL` | 是 | 可与登录服务同域，也可独立 |
| 身份查询路径 | 是 | 需支持按 `user_id` 查询 |
| 请求方式 | 是 | 例如 `GET` 或 `POST` |
| 鉴权方式 | 是 | 请明确是 `crm-token` 还是其他方式 |
| 成功响应样例 | 是 | 至少覆盖身份与权限字段 |
| 失败响应样例 | 是 | 便于我方做错误处理 |

如果走身份 API，请至少返回以下字段：

| 字段 | 是否必需 | 用途 |
|---|---|---|
| `id` 或 `user_id` | 是 | CRM 用户唯一标识 |
| `name` | 是 | 页面和会话展示 |
| `roleIds` 或 `roleNames` | 是 | 角色与权限装载 |
| `organizationIds` | 是 | 组织范围判断 |
| `departmentIds` | 是 | 部门范围判断 |
| `isAdmin` | 建议 | 便于快速识别管理权限 |
| `wecomSenderId` | 建议 | 便于后续企业微信身份复用 |
