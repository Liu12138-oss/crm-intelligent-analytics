# 联软CRM-第二阶段真实登录参数单

## 1. 文档目的

本文档用于沉淀联软 CRM 第二阶段“真实账号密码登录”所需参数、我方当前已完成改造、以及接下来应如何落地。

本文档聚焦两个问题：

1. 真实登录要补什么参数
2. 我方现在已经做到哪里、后续怎么切

---

## 2. 当前状态结论

截至 2026-06-04，当前状态如下：

1. 第一阶段标准 OpenAPI 已联调通过，可继续作为只读查询稳定基线。
2. 我方当前页面登录入口仍可用本地 mock 账号 `admin / admin123` 登录。
3. 这不代表“真实 CRM 登录已正式切换”，只代表本地后台登录入口可正常使用。
4. 我方后端已补入第二阶段真实登录兼容改造，并新增“身份查询 API 兜底”能力。
5. 在对方最终补齐真实登录参数前，不建议直接关闭本地 mock 回退。

---

## 3. 我方当前已完成的真实登录兼容改造

### 3.1 已具备能力

1. 登录成功后，可按 `user_id` 继续解析 CRM 实时身份。
2. 优先走现有只读库身份解析，不破坏现有链路。
3. 若只读库未命中，且已配置身份查询 API，可自动回退到身份 API。
4. 会话刷新时仅对 `database` / `crm-api` 身份继续回源，不影响 mock 身份链路。
5. 已补对应单元测试与构建验证，当前编译通过。

### 3.2 本次新增的第二阶段配置项

| 环境变量 | 用途 | 是否必须 |
|---|---|---|
| `CRM_AUTH_IDENTITY_API_BASE_URL` | 身份查询 API 基址 | 条件必填 |
| `CRM_AUTH_IDENTITY_API_USER_PATH` | 按 `user_id` 查身份的路径模板 | 条件必填 |
| `CRM_AUTH_IDENTITY_API_TIMEOUT_MS` | 身份查询 API 超时 | 建议填写 |
| `CRM_AUTH_IDENTITY_API_AUTH_MODE` | 身份 API 鉴权方式，当前支持 `crm-token` / `none` | 建议填写 |

说明：

1. 如果第二阶段继续使用 CRM 只读库做身份落位，则这 4 项可以不启用。
2. 如果对方不提供只读库，只提供身份查询 API，则这 4 项需要补齐。

## 4. 第二阶段最小参数清单

## 4.1 真实登录主链路参数

| 对方提供项 | 我方环境变量 | 是否必须 | 说明 |
|---|---|---|---|
| 真实登录 Base URL | `CRM_OPEN_API_BASE_URL` | 是 | 我方会调用 `${CRM_OPEN_API_BASE_URL}/api/v2/auth/login` |
| 真实登录路径 | `CRM_OPEN_API_LOGIN_PATH` | 建议提供 | 当前代码已支持自定义登录路径，默认 `/api/v2/auth/login` |
| 是否必传 `corp_id` | `CRM_OPEN_API_CORP_ID` | 条件必填 | 若登录接口强制要求，则必须提供固定值 |
| 建议 `device` | `CRM_OPEN_API_DEVICE` | 建议提供 | 默认可用 `open_api` |
| 建议 `version_code` | `CRM_OPEN_API_VERSION_CODE` | 建议提供 | 默认可用 `9.9.9` |
| 超时建议 | `CRM_OPEN_API_TIMEOUT_MS` | 建议提供 | 建议明确毫秒值 |

## 4.2 登录返回契约

我方当前最少依赖以下真实登录返回字段：

1. `code`
2. `message`
3. `data.user_id`
4. `data.user_token`

建议对方明确补充：

| 项目 | 是否必须 | 说明 |
|---|---|---|
| 登录成功响应样例 | 是 | 至少含 `data.user_id`、`data.user_token` |
| 登录失败响应样例 | 是 | 至少覆盖密码错误、账号不存在、账号禁用 |
| 成功判定规则 | 是 | 例如 `code = 0` |
| `user_token` 有效期 | 建议提供 | 便于评估会话刷新策略 |
| 错误码字典 | 建议提供 | 便于我方做中文错误提示 |

## 5. 登录成功后的身份落位二选一

真实登录成功后，我方还必须把 `user_id` 落成系统可识别的 `CrmUser`，否则不能安全建立权限会话。

对方必须二选一提供：

## 5.1 方案 A：继续提供 CRM 只读库

| 对方提供项 | 我方环境变量 | 是否必须 | 说明 |
|---|---|---|---|
| 只读库主机 | `CRM_READONLY_DB_HOST` | 是 | 仅用于身份与权限只读解析 |
| 只读库端口 | `CRM_READONLY_DB_PORT` | 是 | 例如 `3306` |
| 只读库库名 | `CRM_READONLY_DB_NAME` | 是 | 真实身份来源库 |
| 只读库用户名 | `CRM_READONLY_DB_USER` | 是 | 最小只读权限 |
| 只读库密码 | `CRM_READONLY_DB_PASSWORD` | 是 | 需通过安全渠道传递 |

适用判断：

1. 对方允许我方继续访问只读库。
2. 我方希望最大化复用现有数据库身份装载逻辑。
3. 这是当前改动最少、风险最低的路径。

## 5.2 方案 B：提供身份查询 API

若对方不再提供只读库，则至少需提供一个按 `user_id` 查询身份的 API。

对应我方配置如下：

| 对方提供项 | 我方环境变量 | 是否必须 | 说明 |
|---|---|---|---|
| 身份 API Base URL | `CRM_AUTH_IDENTITY_API_BASE_URL` | 是 | 建议与登录服务同域 |
| 身份查询路径模板 | `CRM_AUTH_IDENTITY_API_USER_PATH` | 是 | 例如 `/api/open/v1/identity/users/{userId}` |
| 身份 API 超时 | `CRM_AUTH_IDENTITY_API_TIMEOUT_MS` | 建议提供 | 建议与登录超时保持同级 |
| 身份 API 鉴权方式 | `CRM_AUTH_IDENTITY_API_AUTH_MODE` | 建议提供 | 当前优先支持 `crm-token` |

身份 API 至少需要返回以下字段：

| 字段 | 是否必须 | 用途 |
|---|---|---|
| `id` 或 `user_id` | 是 | 统一 CRM 用户主键 |
| `name` | 是 | 页面与会话展示 |
| `roleIds` 或 `roleNames` | 是 | 角色与权限装载 |
| `organizationIds` | 是 | 组织范围装载 |
| `departmentIds` | 是 | 部门范围装载 |
| `ownerIds` | 建议提供 | 更细粒度归属范围 |
| `isAdmin` | 建议提供 | 管理权限快速判断 |
| `channels` | 建议提供 | 当前支持 `web-console`、`wecom-bot` |
| `wecomSenderId` | 建议提供 | 便于后续企微身份复用 |

## 7. 我方推荐落地方案

为了不影响现有功能，建议按以下顺序推进：

## 7.1 推荐技术路线

1. 第一阶段标准 OpenAPI 继续保留，作为只读查询稳定基线。
2. 第二阶段真实登录只改“登录入口和身份落位”，不改第一阶段六类对象查询适配层。
3. 真实登录成功后，只做身份装载和会话建立。
4. 六类对象查询仍继续复用第一阶段标准 OpenAPI 适配层，不把两条链路混成一套。

## 7.2 推荐切换步骤

1. 先补齐 `CRM_OPEN_API_*` 参数。
2. 再确定身份落位方式：
   - 继续走只读库
   - 或切到身份查询 API
3. 在本地或 SIT 保持 `CRM_AUTH_MOCK_ENABLED=true` 做灰度。
4. 真实账号登录通过后，再切 `CRM_AUTH_MOCK_ENABLED=false` 做严格联调。
5. 严格联调通过后，再进入 UAT 与生产灰度。

---

## 8. 第二阶段我方还要做什么

在对方参数补齐后，我方下一步动作如下：

1. 填入真实 `CRM_OPEN_API_*` 参数。
2. 若对方给身份 API，则填入 `CRM_AUTH_IDENTITY_API_*` 参数并联调。
3. 用真实账号验证：
   - 登录成功
   - 失败提示
   - 会话建立
   - 权限范围是否正确落位
4. 再验证登录后进入治理、诊断、问数页面时是否仍保持兼容。
5. 最后再决定是否关闭 mock 回退。

---

## 9. 当前建议结论

当前可以明确的结论是：

1. 我方已经支持第二阶段真实登录改造继续推进。
2. 当前缺的不是主代码框架，而是对方最终确认的真实登录参数与身份落位来源。
3. 在不影响第一阶段能力的前提下，推荐优先采用：
   - 第一阶段继续走标准 OpenAPI
   - 第二阶段只切登录与身份落位
4. 只要对方把真实登录参数和身份落位方案补齐，我方即可继续联调与灰度切换。
