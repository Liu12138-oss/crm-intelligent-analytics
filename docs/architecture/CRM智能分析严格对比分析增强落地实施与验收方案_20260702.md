# CRM 智能分析严格对比分析增强落地实施与验收方案

编写日期：2026-07-02

## 一、目标与范围

本方案面向企业微信机器人长连接问数场景，解决“用户要求对比分析，但系统只返回总量汇总”的问题。增强目标是让区域对比、大区对比、商机维度、渠道商维度、订单维度、销售维度都能输出可核对的数据差异、贡献占比、风险原因和动作建议，并通过企微卡片、Markdown 正文和图片图表同时呈现。

本阶段不引入 HTML 作为企微主交付载体，不新增写回动作，不绕过权限和白名单。所有增强均复用现有 CRM 已同步真实明细数据和 OpenAPI Markdown 快照主链。

## 二、当前问题诊断

| 问题 | 当前表现 | 影响 |
| --- | --- | --- |
| 对比意图弱 | “一季度与二季度”“区域/大区对比”已能识别，但部分问法仍以总览标题和泛化摘要输出 | 用户看不到真正的差异判断 |
| 维度聚合不足 | 已有季度、区域、渠道商汇总，但缺少销售负责人维度和大区单独对比视图 | 无法回答“谁贡献、谁下滑、哪个大区异常” |
| 占比字段不足 | 行里有数量和金额，但多数没有贡献占比、转化率、头部集中度 | 企微卡片和图表只能展示绝对值 |
| 风险原因不足 | 停滞商机有风险口径，但普通对比分析缺少原因归纳 | 对比结论难以转化为行动 |
| 动作建议不足 | 正文有通用建议，但表格行级没有可执行动作 | 销售管理者无法直接分派跟进 |

## 三、增强后用户可见标准

### 3.1 对比分析必须包含的内容

每个命中严格对比分析的问题，至少输出以下 6 类信息：

1. 对比对象：区域、大区、渠道商、销售负责人或季度。
2. 核心指标：商机数量、商机金额、订单数量、订单金额。
3. 差异变化：跨季度必须有数量变化、金额变化和变化率。
4. 贡献占比：行级必须有商机金额占比、订单金额占比或总贡献占比。
5. 风险原因：识别高商机低订单、报备多商机少、金额集中、无订单承接、阶段推进不足等原因。
6. 动作建议：给出可执行的跟进动作，例如复盘头部渠道、推进高金额商机报价、补齐低转化区域负责人动作。

### 3.2 企业微信呈现标准

| 载体 | 必须呈现 |
| --- | --- |
| 模板卡片 | 标题命中对比分析；主指标体现对比结果；关键观察不只写总量 |
| Markdown 正文 | 有问题复述、数据口径、核心判断、对比表、风险原因、动作建议 |
| 图片附件 | 至少 1 张对比图；季度用折线/趋势，区域/大区/渠道商/销售用排行或占比图 |
| 明细视图 | 保留可追溯行，避免只输出结论没有数据支撑 |

## 四、落地设计

### 4.1 统一增强点

优先改造 `LianruanCrmAnalysisExecutorService.executeBusinessChainSnapshotTask` 及其聚合函数，因为当前企微主链的真实明细、季度对比、区域对比、渠道商贡献和图片附件均从这里产生。这样可以一次覆盖企微机器人、公开报告和测试脚本，避免每个入口重复写逻辑。

### 4.2 新增对比视图

| 视图 | 数据来源 | 关键字段 |
| --- | --- | --- |
| 商机季度对比 | opportunities | 商机数、商机金额、环比变化、变化率、风险原因、动作建议 |
| 大区经营对比 | partners + registrations + opportunities + quotes + orders | 大区、渠道数、商机金额占比、订单金额占比、转化率、风险原因、动作建议 |
| 区域经营对比 | 同上 | 区域、大区、贡献占比、业务链指标、风险原因、动作建议 |
| 渠道商贡献对比 | 同上 | 渠道商、商机金额占比、订单金额占比、商机到订单转化率、风险原因、动作建议 |
| 销售负责人对比 | registrations + opportunities + quotes + orders | 负责人、商机金额占比、订单金额占比、转化率、风险原因、动作建议 |
| 订单承接对比 | orders + opportunities | 订单数、订单金额、商机金额、订单承接率、风险原因、动作建议 |

### 4.3 风险原因规则

| 风险类型 | 判断口径 | 用户可见原因 |
| --- | --- | --- |
| 高商机低订单 | 商机金额大于 0 且订单金额为 0 | 商机储备已有，但订单承接未体现，需要推进报价和签单闭环 |
| 低转化 | 商机数大于 0 且订单数/商机数低于 10% | 商机到订单转化偏低，需要复核阶段推进和客户决策阻塞 |
| 头部集中 | 单行金额占比大于 50% | 贡献集中在少数对象，存在头部依赖风险 |
| 覆盖不足 | 渠道数或负责人承接数较少 | 覆盖面不足，需要补齐区域/渠道/销售动作 |
| 数据缺口 | 区域、大区、负责人或渠道商字段缺失 | 字段缺失会影响归因，需要治理主数据 |

### 4.4 动作建议规则

| 场景 | 动作建议 |
| --- | --- |
| 商机金额高但无订单 | 锁定高金额商机，补报价、确认客户决策人和预计签约时间 |
| 订单占比高 | 复盘打法并复制到相邻区域或同类渠道 |
| 低转化 | 建立本周推进清单，按阶段拆分阻塞原因 |
| 数据字段缺失 | 补齐渠道商区域、大区、负责人和客户归属字段 |
| 头部集中 | 维护头部渠道，同时激活第二梯队渠道降低依赖 |

## 五、实施阶段

### P0：方案与主链统一

- 新增方案文档。
- 明确企微主交付仍为卡片、Markdown、PNG 图片。
- 确认增强入口为 OpenAPI Markdown 快照业务链。

验收：文档存在，目标、范围、字段、风险规则和测试问题完整。

### P1：数据行增强

- 为季度、区域、大区、渠道商、销售负责人、订单承接行补齐占比、转化率、风险原因、动作建议。
- 保留现有字段，新增字段不破坏老模板。
- 所有新增用户可见字段使用中文自然表达。

验收：专项问题返回行里包含占比、风险原因、动作建议。

### P2：展示视图增强

- 将大区对比、销售负责人对比、订单承接对比加入二级视图。
- 对比类问题优先展示季度对比和区域/大区对比。
- 图片附件自动消费新增视图，企微可看到图表而不是只看文字。

验收：企微模拟交付至少包含 1 张对比图，且图片标题命中“季度对比/区域经营对比/大区经营对比/销售负责人对比”。

### P3：验收脚本增强

- 对自定义对比问题增加严格断言：模板、图片、正文、占比、风险原因、动作建议。
- 跑原始问题和同族问题专项验收。
- 跑 300 问严格回归，确保不破坏已有能力。

验收：专项问题通过率 100%，300 问严格回归通过。

## 六、代表验收问题

1. 大北区26年一季度与二季度商机对比分析。
2. 各大区26年一季度与二季度商机对比分析。
3. 不同区域26年一季度与二季度商机对比分析。
4. 山东区与大北区26年一季度与二季度商机对比分析。
5. 26年一季度与二季度各销售负责人商机和订单对比分析。
6. 26年一季度与二季度渠道商贡献占比和风险原因分析。
7. 大北区26年一季度与二季度订单承接对比分析。

## 七、通过标准

| 验收项 | 严格通过标准 |
| --- | --- |
| 意图识别 | 对比类问题实际模板为 `REGION_COMPARISON` 或等价对比模板 |
| 时间口径 | “26年一季度与二季度”解析为 2026 年一季度至二季度闭开区间 |
| 对比结果 | 返回季度、区域/大区、渠道商或销售负责人至少一个对比表 |
| 贡献占比 | 至少一个对比表包含占比字段 |
| 风险原因 | 正文或表格出现“风险原因” |
| 动作建议 | 正文或表格出现“动作建议” |
| 图表附件 | 企微模拟交付至少 1 张 PNG 图片附件 |
| 回归 | 300 问严格验收不下降 |

## 八、风险与边界

- 如果 CRM 明细没有订单金额或订单状态，本阶段只能按已有字段输出“订单承接不足”，不能虚构订单结果。
- 如果区域、大区、负责人字段缺失，必须明确提示数据治理建议，不能把缺失字段推断成真实区域。
- 企微长连接不传 HTML；HTML 和公开报告只作为备查入口。
- 本阶段不真实发送企微消息，只用本地模拟链路验收，避免污染真实群聊。
