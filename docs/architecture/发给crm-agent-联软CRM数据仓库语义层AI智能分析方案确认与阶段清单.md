# 发给 crm-agent：联软 CRM 数据仓库语义层 AI 智能分析方案确认与阶段清单

> 日期：2026-06-09  
> 依据：贵方《联软CRM-数据仓库语义层AI智能分析总体方案与双方分工》、联软 CRM 当前 OpenAPI 与权限现状  
> 用途：确认方案可行性、阶段边界、双方分工、联软 CRM 可先提供给贵方的确定内容  
> 结论：该方案方向可行，适合作为长期智能分析架构；第一阶段仍建议以联软 CRM 标准 OpenAPI 作为正式数据源和权限源，后续逐步建设 AI-agent 侧数据仓库、语义层和受控 SQL。

---

## 1. 总体判断

贵方提出的：

```text
联软 CRM 数据源
  -> 数据同步
  -> 分析数据仓库
  -> 标准数据模型
  -> 业务语义层
  -> AI 查询计划
  -> 受控 SQL
  -> 分析报告
```

整体可行，并且比以下方案更适合长期运营：

1. AI-agent 直接连接联软生产库。
2. AI 自由生成 SQL 后直接执行。
3. 只把字段清单给 AI，让 AI 自己猜业务口径。
4. 只做单接口查询，不沉淀指标、语义和回归样例。

联软 CRM 侧建议确认的总体原则：

```text
当前：标准 OpenAPI 作为正式联调和同步通道。
后续：AI-agent 建设 MySQL/数仓语义层承接复杂问数。
边界：不直接开放生产业务 SQLite，不让 AI 绕过 CRM 权限和审计。
```

---

## 2. 推荐落地架构

建议双方按“CRM 提供可信数据源，AI-agent 承接分析数仓和语义层”的方式落地。

```text
企业微信 / Web 智能分析
        |
        v
AI-agent 会话、身份识别、自然语言理解
        |
        v
AI-agent 数据仓库与语义层
        |
        v
受控 SQL / 权限注入 / 审计
        |
        v
分析结果与报告

数据来源：
联软 CRM 标准 OpenAPI
        |
        v
AI-agent 增量同步 / 定时同步
        |
        v
ODS / DWD / DWS / ADS
```

说明：

1. 联软 CRM 不直接面向 AI 暴露生产库。
2. 联软 CRM 提供 OpenAPI、字段字典、权限口径、指标口径和样例数据。
3. AI-agent 将 CRM 数据同步到自己的分析库或语义层。
4. AI 只负责语义理解和查询计划，SQL 执行必须经过程序校验。
5. 权限条件必须由程序基于 CRM 用户权限范围注入，不能由 AI 自己拼。

## 3. 当前可确定给贵方的内容


## 4. 分阶段实施方案

### P0：方案确认与数据资产对齐

目标：

```text
先确认同步方式、对象范围、字段口径、权限口径和样例问题，避免后续 AI 按错误语义建模。
```

联软 CRM 需要提供：

| 内容 | 当前状态 | 说明 |
|---|---|---|
| OpenAPI Base URL | 已有 | SIT 当前为 `http://10.18.16.114:3000/api/open/v1` |
| 4 组 client | 已有 | 用于超管、区管、渠道管理员、员工四类权限 |
| 对象清单 | 已有初版 | 用户、渠道、客户、报备、商机、报价、订单 |
| 字段字典 | 已有初版 | 后续可继续补字段中文含义和敏感级别 |
| 指标口径 | 已有初版 | 成交、转化、活跃、生命周期等需继续业务确认 |
| 权限口径 | 已有初版 | OpenAPI 已按绑定用户裁剪 |
| 样例数据 | 有模板 | 需继续补真实样例 ID 和预期结果 |

AI-agent 需要完成：

| 内容 | 说明 |
|---|---|
| 确认数仓承载位置 | AI-agent 侧 MySQL / 分析库 / 其他存储 |
| 确认同步方式 | 优先 OpenAPI 增量同步 |
| 输出字段映射模板 | 将 CRM OpenAPI 字段映射到数仓模型 |
| 输出第一批高频问题 | 20 到 30 条回归问题 |
| 确认 SQL 安全方案 | 只读、白名单、权限注入、审计 |

P0 验收：

1. 双方确认第一阶段以 OpenAPI 为数据源。
2. 确认 P0 对象：用户、渠道、客户、报备、商机、订单、字典、权限。
3. 确认第一批样例问题和测试账号。
4. 确认不直连生产业务库。

### P1：数据同步与 ODS 原始层

目标：

```text
让 AI-agent 能稳定把 CRM 只读数据同步到分析库。
```

联软 CRM 需要提供：

| 内容 | 说明 |
|---|---|
| OpenAPI 增量字段 | `createdAt`、`updatedAt`、分页参数 |
| 分页规则 | `pageNo`、`pageSize`，当前建议单页不超过 200 |
| 鉴权方式 | `POST /auth/token` + `Bearer token` |
| IP 白名单 | 当前已支持对方出口 IP 和内网联调 IP |
| 数据范围说明 | 当前 token 绑定用户权限范围内数据 |
| 同步对象顺序 | 字典、用户、渠道、客户、报备、商机、报价、订单 |

AI-agent 需要完成：

| 内容 | 说明 |
|---|---|
| 同步调度 | 定时同步、手动同步、失败重试 |
| ODS 原始层 | 保存 OpenAPI 原始 JSON 或展开表 |
| 同步日志 | 记录对象、页码、数量、耗时、失败原因 |
| 数据批次 | 记录 `syncBatchId`、`syncedAt`、`sourceUpdatedAt` |
| 数据校验 | 源端数量、目标数量、关键字段非空率 |

P1 验收：

1. P0 对象可完整同步。
2. 每个对象有同步批次、数量、数据截至时间。
3. 同步失败可重试并有日志。
4. 不影响 CRM 正常业务访问。

### P2：标准数据模型与权限模型

目标：

```text
把 OpenAPI 原始数据清洗为客户、渠道、报备、商机、订单等标准明细模型，并建立权限桥表。
```

建议模型：

| 层级 | 模型 | 说明 |
|---|---|---|
| DWD | `dim_customer` | 客户维度 |
| DWD | `dim_partner` | 渠道/服务商维度 |
| DWD | `dim_user` | CRM 用户维度 |
| DWD | `dim_region` | 区域/大区维度 |
| DWD | `dim_dictionary` | 字典维度 |
| DWD | `fact_registration` | 报备事实 |
| DWD | `fact_opportunity` | 商机事实 |
| DWD | `fact_quote` | 报价事实 |
| DWD | `fact_order` | 订单事实 |
| DWD | `bridge_user_permission` | 用户权限桥表 |

联软 CRM 需要确认：

| 内容 | 说明 |
|---|---|
| 客户 ID 规则 | 当前输出 `customerId`、`customerIdRule` |
| 渠道对象口径 | 合作伙伴、服务商、渠道商第一阶段按同一对象 `partners` 处理 |
| 金额字段 | 商机金额、报价金额、订单金额分别使用哪个字段 |
| 时间字段 | 创建时间、更新时间、下单时间、成交时间口径 |
| 区域字段 | 按客户、渠道、商机或负责人区域的优先级 |
| 权限字段 | `ownerId`、`assignedStaffId`、`createdBy`、`partnerId`、`region`、`bigRegion` |

AI-agent 需要完成：

| 内容 | 说明 |
|---|---|
| 标准模型映射 | OpenAPI 字段到 DWD 模型 |
| 数据清洗 | 空值、枚举、金额、时间、客户归并 |
| 权限桥表 | 将 CRM 权限范围转为可 SQL 注入的条件 |
| DWS 汇总 | 高频聚合表，如渠道贡献、客户生命周期、商机风险 |

P2 验收：

1. 标准模型中的数量与 OpenAPI 汇总口径一致。
2. 四类测试账号的可见数据差异符合 CRM 权限。
3. 客户、渠道、商机、订单主链可以关联。
4. 金额、时间、状态口径有可追溯说明。

### P3：业务语义层与 AI 查询计划

目标：

```text
让 AI 先理解业务问题并生成结构化查询计划，而不是直接生成最终 SQL。
```

联软 CRM 需要提供：

| 内容 | 示例 |
|---|---|
| 业务同义词 | 合作伙伴=服务商=渠道商 |
| 高频问法 | 山东区域商机金额、未报备客户、超期未更新商机 |
| 指标解释 | 商机数、订单金额、转化率、客户创建时长 |
| 维度解释 | 区域、渠道、负责人、月份、阶段、等级 |
| 筛选口径 | 今年、最近三个月、超过两周、有效商机 |

AI-agent 需要完成：

| 内容 | 说明 |
|---|---|
| 语义目录 | 对象、指标、维度、筛选、字典、同义词 |
| 查询计划协议 | AI 输出 JSON 查询计划，不直接执行 SQL |
| 低置信追问 | 字段不明、口径不明时追问 |
| 回归样例 | 高频问题解析结果可回归 |

查询计划示例：

```json
{
  "subject": "partner_operation",
  "timeRange": "this_year",
  "filters": {
    "region": "山东"
  },
  "tasks": [
    {
      "name": "商机汇总",
      "metrics": ["opportunity_count", "opportunity_amount"],
      "dimensions": ["partner"]
    },
    {
      "name": "下单汇总",
      "metrics": ["order_count", "order_amount"],
      "dimensions": ["partner"]
    }
  ],
  "resultShape": "report_with_tables",
  "confidence": "HIGH"
}
```

P3 验收：

1. 常见问题能拆出正确对象、指标、维度和筛选。
2. 复杂问题可拆成多个子任务。
3. AI 不再把不同问题都落到同一个默认接口或默认 SQL。
4. 低置信时追问，不硬查。

### P4：受控 SQL 执行与安全审计

目标：

```text
将查询计划安全转换为 SQL，执行前完成白名单、只读、权限、脱敏、超时和审计。
```

联软 CRM 需要配合：

| 内容 | 说明 |
|---|---|
| 校验样例结果 | 对比 OpenAPI 和数仓查询结果 |
| 校验权限结果 | 四类账号同题查询 |
| 校验敏感字段 | 哪些字段可分析、可展示、需脱敏 |

AI-agent 需要完成：

| 安全能力 | 要求 |
|---|---|
| SQL 类型 | 只允许单条 `SELECT` |
| 表字段白名单 | 只允许访问语义层登记表字段 |
| 参数化 | 用户输入不能直接拼接 SQL |
| 权限注入 | 基于当前 CRM 用户权限强制追加 WHERE 条件 |
| LIMIT | 默认限制返回行数，建议默认 100、最大 1000 |
| 超时 | 建议默认 5 秒，复杂查询最大 30 秒 |
| 脱敏 | 手机、联系人、信用代码、地址等按规则处理 |
| 审计 | 记录问题、计划、SQL、参数、权限快照、耗时、行数 |

P4 验收：

1. 写入、删改、DDL、跨库、危险函数全部被阻断。
2. 未授权字段无法查询。
3. 未注入权限的 SQL 不能执行。
4. 查询结果与 OpenAPI 权限口径一致。
5. 每次查询可审计追踪。

AI-agent 需要完成：

## 5. 最小可行版本建议

为了尽快落地，建议先做 MVP，不要一开始追求全量数仓。

### 5.1 数据范围

| 范围 | 是否纳入 MVP |
|---|---|
| 用户 | 是 |
| 渠道/服务商 | 是 |
| 客户 | 是 |
| 客户报备 | 是 |
| 商机 | 是 |
| 订单 | 是 |
| 报价 | 建议纳入，但可弱化 |
| 跟进记录 | 如果当前 OpenAPI 未稳定，可放 P1/P2 |
| 审批/合同全文 | 暂不纳入 |

### 5.2 问题范围

首批建议覆盖：

1. 今年山东区域合作伙伴客户、报备、商机和下单整体情况。
2. 最近三个月山东区域有商机合作伙伴的商机数量和金额。
3. 最近三个月山东区域有下单合作伙伴的订单数量和金额。
4. 有多少客户没有报备商机，分别创建了多长时间。
5. 有哪些商机超过两个星期没有更新进展。
6. 最近一年加入的服务商按合作级别、等级、是否技术服务商分析。
7. 本月各区域商机数量和金额对比。
8. 本月各负责人商机金额排行。
9. 报备到商机、报价、订单的转化漏斗。
10. 当前账号权限范围内高风险商机清单。

### 5.3 安全范围

MVP 必须具备：

1. OpenAPI 鉴权。
2. CRM 用户权限继承。
3. 字段白名单。
4. SQL 只读校验。
5. 权限条件注入。
6. 查询超时和行数限制。
7. 查询审计。
8. 敏感字段脱敏。

## 6. 关键决策建议

| 决策点 | 建议 |
|---|---|
| 数据同步方式 | 第一阶段优先 OpenAPI 增量同步 |
| 数仓建设位置 | 建议由 AI-agent 侧建设 MySQL/分析库 |
