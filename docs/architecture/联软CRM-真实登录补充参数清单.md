## 3. 对方需补充的最小项

## 3.1 登录接口最小参数

以下是我方切换真实 CRM 登录时的最小必填项：

| 对方提供项 | 对应环境变量 | 是否必填 | 用途 | 说明 |
|---|---|---|---|---|
| CRM 登录服务 Base URL | `CRM_OPEN_API_BASE_URL` | 是 | 我方后端将请求 `${CRM_OPEN_API_BASE_URL}/api/v2/auth/login` | 请提供不带 `/api/v2/auth/login` 的基址，例如 `http://10.18.16.114:3000` |
| 登录接口是否要求 `corp_id` | `CRM_OPEN_API_CORP_ID` | 条件必填 | 若登录接口必须传 `corp_id`，我方将随请求体一并提交 | 若不需要，也请明确回复“登录接口无需 `corp_id`” |
| 登录接口推荐 `device` 值 | `CRM_OPEN_API_DEVICE` | 建议提供 | 我方当前登录请求体会携带 `device` | 若对方无特殊要求，可沿用默认值 `open_api` |
| 登录/接口调用超时建议 | `CRM_OPEN_API_TIMEOUT_MS` | 建议提供 | 便于我方设置合理超时，避免长时间等待 | 请提供毫秒值，例如 `12000` |
| 后续官方接口推荐 `version_code` | `CRM_OPEN_API_VERSION_CODE` | 建议提供 | 我方后续复用 CRM 既有官方接口时，会透传到请求头 | 若无明确要求，可回复“使用默认值 `9.9.9`” |

说明：

1. 上表是“真实 CRM 登录链路”使用的参数，不是第一阶段标准 OpenAPI 参数。
2. `CRM_STANDARD_OPEN_API_BASE_URL / APP_KEY / APP_SECRET` 已用于第一阶段只读联调，和本节不是同一套配置。
3. `真实登录联调测试与迁移手册.md` 中给出的是“本地 mock 演练默认值”，不能直接视为正式联调环境参数。

## 3.4 实时身份映射数据源

真实 CRM 登录成功后，我方还需要根据 `user_id` 读取该用户在 CRM 中的实时身份与权限上下文，否则无法安全建立我方系统会话。

对方可二选一提供：

### 方案 A：提供 CRM 只读身份库连接信息

| 对方提供项 | 对应环境变量 | 是否必填 | 说明 |
|---|---|---|---|
| 只读库主机 | `CRM_READONLY_DB_HOST` | 是 | 仅用于只读身份与权限解析 |
| 只读库端口 | `CRM_READONLY_DB_PORT` | 是 | 例如 `3306` |
| 只读库库名 | `CRM_READONLY_DB_NAME` | 是 | 仅提供只读账号对应的目标库名 |
| 只读库用户名 | `CRM_READONLY_DB_USER` | 是 | 需具备最小只读权限 |
| 只读库密码 | `CRM_READONLY_DB_PASSWORD` | 是 | 请通过安全渠道单独传递 |

说明：

1. 我方不会用该连接做写入，只用于按 `user_id` 解析实时身份。
2. 建议对方同步说明只读账号可访问的表白名单，避免后续权限误判。

### 方案 B：提供等价的身份查询 API

如果对方不方便开放只读库，请提供 1 个“按 `user_id` 查询身份与权限”的等价 API，最少返回以下信息：

| 字段 | 是否必填 | 用途 |
|---|---|---|
| `id` | 是 | CRM 用户唯一标识，对应 `user_id` |
| `name` | 是 | 用户姓名，用于会话与页面展示 |
| `roleIds` / `roleNames` | 是 | 用于装载角色与权限 |
| `organizationIds` | 是 | 用于组织范围判断 |
| `departmentIds` | 是 | 用于部门范围判断 |
| `isAdmin` | 是 | 用于快速识别管理权限 |
| `wecomSenderId` | 建议提供 | 便于后续企微身份映射与统一入口复用 |

如果对方现阶段无法直接返回上述字段，也请至少提供：

1. 当前可返回的字段清单
2. 缺失字段后续补齐计划
3. 哪些字段可以通过二次接口查询补足

补充说明：

1. 新收到的“真实登录联调测试与迁移手册”已明确：正式切真实地址前，`user_id` 对应的身份数据源仍需二选一提供。
2. 因此这一项并没有被新资料消除，仍然是第二阶段真实登录能否落地的关键前置条件。
