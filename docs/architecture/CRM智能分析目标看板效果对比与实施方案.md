## 3. 当前项目与附件目标的差距

| 对比项 | 当前项目 | 附件目标 | 差距判断 |
|---|---|---|---|
| 报告层级 | 单次问数结果为主 | 多模块经营报告 / 看板 | 需要新增看板级报告编排 |
| 数据集数量 | 通常 1 到少量任务 | KPI、趋势、排行、明细、地图、证书等多数据集 | 需要新增多数据集任务计划 |
| 图表能力 | 自研轻量条形 / 折线 / 列表 | Chart.js / ECharts 多图、地图、组合图 | 需要引入或封装图表渲染能力 |
| 地图能力 | 暂无 | 全国省份覆盖地图、双击钻取 | 需要新增地图看板组件或静态地图渲染 |
| 交互表格 | Web 有基础表格，企微有预览 | 排序、搜索、筛选、折叠、钻取 | 需要新增看板表格交互模板 |
| HTML 附件 | 暂无精美 HTML 报告导出 | 单文件 HTML 可转发 | 需要新增 HTML 报告生成与文件发送 |
| 企微展示 | 文字 + 图片海报 | 可进一步发送 HTML 文件 / 多图摘要 | 需要新增“企微文件附件 + 多图摘要” |
| 经营洞察 | 有通用 AI 富报告 | 附件洞察与指标高度绑定 | 需要新增按报告模板定制洞察 |
| 数据对象 | 服务商、商机、报备、报价、订单为主 | 渠道下单、签约额、证书、覆盖城市、团队 | 需要联软补订单/签约/证书/城市覆盖字段或接口 |
| 权限口径 | 已有本地二次裁剪基础 | 看板所有模块按当前角色裁剪 | 需要每个数据集统一套权限与审计 |

## 4. 能否直接实现附件效果

可以实现，但需要分层推进。

短期可以做到：

1. 企业微信问一句“生成山东区域渠道经营看板”，系统返回文字摘要 + 图片版核心看板。
2. Web 结果页展示 KPI、图表、表格、AI 报告。
3. 生成一个可下载 HTML 报告文件，样式接近附件。
4. 企微小助手发送 HTML 文件或链接，用户在企微里点开查看。

中期可以做到：

1. 支持固定主题看板模板，例如“渠道下单汇总分析”“全国代理商运营看板”。
2. 支持多图、多表、多数据集联动。
3. 支持地图、省份钻取、折叠表格、搜索筛选。
4. 支持按角色权限生成不同口径的看板。

不能直接一步到位的点：

1. 如果联软 OpenAPI 没有“签约 / 下单 / 合同 / 证书 / 城市覆盖”字段，无法真实生成附件里的完整指标。
2. 当前企微机器人不适合直接承载复杂交互看板，最佳方式是“企微摘要 + 图片预览 + HTML 文件/链接”。
3. 当前 Web 图表组件不是 ECharts，地图和组合图需要新增组件或引入图表库。

## 5. 推荐目标架构

```text
Web / 企业微信自然语言
  -> AI 意图识别：普通问数 or 看板报告
  -> 看板模板选择：渠道下单汇总 / 代理商运营 / 区域经营 / 服务商画像
  -> 多数据集任务编排
  -> 联软 OpenAPI 明细 / 统计接口取数
  -> 当前 CRM 用户权限裁剪
  -> 指标计算与一致性校验
  -> AI 生成经营洞察和建议
  -> 看板数据包 DashboardPayload
  -> Web 看板渲染
  -> HTML 文件生成
  -> 企微摘要 + 图片预览 + HTML 文件/链接
  -> 审计记录
```

建议新增统一数据结构：

```ts
interface DashboardReportPayload {
  reportId: string;
  title: string;
  theme: 'channel-order-summary' | 'agent-operations-dashboard' | 'region-business-review';
  scopeSummary: string;
  temporalScope: string;
  metricCards: Array<{ label: string; value: string | number; unit?: string; sub?: string }>;
  insightCards: Array<{ title: string; detail: string; severity?: 'info' | 'warning' | 'success' }>;
  chartBlocks: Array<DashboardChartBlock>;
  tableBlocks: Array<DashboardTableBlock>;
  drilldownBlocks?: Array<DashboardDrilldownBlock>;
  sourceSummary: string;
  generatedAt: string;
}
```

## 7. 实施计划

### 阶段一：看板数据包与渠道下单模板

目标：先实现最接近当前联软数据能力的“渠道下单汇总分析”。

我方工作：

1. 新增看板报告数据结构 `DashboardReportPayload`。
2. 新增看板意图识别：识别“生成看板、生成报告、汇总分析、渠道下单、代理商运营”等问法。
3. 新增 `channel-order-summary` 模板。
4. 基于 `orders` 和 `partners` 生成 KPI、年度趋势、TOP10、TOP30、全部渠道排名和明细。
5. 复用现有权限裁剪，确保当前用户只能看自己角色可见数据。
6. 生成 Web 看板区块。
7. 生成企微简版摘要和图片预览。

联软需要：

1. 确认订单 / 合同对象字段完整。
2. 确认订单金额、成交时间、合同号、渠道、负责人、部门字段。
3. 提供至少一组可验证样例数据。

验收标准：

1. 问“广州办渠道下单汇总分析”可生成 KPI + 趋势 + TOP + 明细。
2. Web 能展示完整看板。
3. 企微能收到摘要和图片预览。
4. 数据合计与明细行可追溯一致。

### 阶段二：HTML 报告导出与企微文件发送

目标：达到附件“单文件 HTML 可转发”的效果。

我方工作：

1. 新增 HTML 报告生成器。
2. 内置项目级 CSS 主题，不把密钥或敏感信息写入文件。
3. 支持图表数据内嵌，优先使用安全的本地模板。
4. 支持生成 `.html` 文件并记录导出审计。
5. 企微端支持发送“摘要 + 图片 + HTML 文件/链接”。

验收标准：

1. Web 点击“生成 HTML 报告”可下载。
2. 企业微信可收到 HTML 文件或可访问链接。
3. HTML 打开后包含 KPI、图表、表格、洞察和口径说明。

### 阶段三：全国代理商运营看板

目标：实现第二个附件级运营看板。

我方工作：

1. 新增 `agent-operations-dashboard` 模板。
2. 新增省份地图或中国地图展示组件。
3. 新增代理商覆盖、万亿城市覆盖、技术服务人员、证书认证数据模型。
4. 新增大区/团队折叠表格。
5. 新增省份钻取弹窗或详情区。

联软需要：

1. 补 `province`、`city`、`teamName`、代理商等级和类型。
2. 补技术服务人员和证书数据接口。
3. 提供城市覆盖口径和万亿城市清单。

验收标准：

1. Web 能展示全国地图和大区对比。
2. 可按大区 / 团队筛选代理商明细。
3. 可展示技术服务人员和证书认证。

### 阶段四：模板治理、权限和稳定性

目标：让看板能力长期可维护。

我方工作：

1. 看板模板纳入治理后台，支持启用 / 停用。
2. 看板字段完整性诊断。
3. 看板生成审计，包括用户、角色、权限范围、数据源、导出文件。
4. 权限回归测试：超管、区域负责人、渠道管理员、普通员工。
5. 性能保护：分页上限、聚合上限、超时、降级提示。
6. 企微端加入“重新生成、缩小范围、导出 HTML、看明细”的快捷回复。

验收标准：

1. 字段缺失时不生成假看板，而是说明缺口。
2. 不同角色看板数据口径与联软 CRM 权限一致。
3. 看板生成失败不影响普通问数。

## 8. 推荐优先级

| 优先级 | 工作项 | 原因 |
|---|---|---|
| P0 | 渠道下单汇总模板 | 最接近当前订单 / 渠道 / 金额数据能力 |

| P1 | 多数据集报告编排 | 看板级体验必须具备 |

## 9. 当前结论

当前项目已经具备“问数、取数、权限、AI 分析、Web 展示、企微摘要、图片表格”的基础能力，但附件目标属于更高一级的“专题经营看板报告”。

建议下一步不要零散优化现有单次问数页面，而是新增一条看板报告链路：

```text
普通问数：轻量结果，快速回答
专题看板：多数据集，多图表，多表格，可导出 HTML，可发企微
```

第一阶段优先做“渠道下单汇总分析”，因为它与当前联软渠道 CRM 的 `orders`、`partners`、`opportunities` 能力最接近；第二阶段再做“全国代理商发展运营数据看板”，因为它额外依赖省市、团队、技术服务人员和证书认证字段。
