# CRM 智能分析展示模板层实施记录

> 日期：2026-06-30  
> 目标：落实“问题和答案展示模板匹配”的第一步，为企微机器人后续图片卡片、模板卡片和完整报告链接打基础。  
> 范围：仅新增展示模板匹配与企微结果挂载，不改变真实查询链路、不改变权限链路、不启用写回能力。

## 一、本次已完成

1. 新增 12 套 CRM 智能分析展示模板类型，覆盖经营总览、业务漏斗、区域对比、渠道排行、渠道画像、分销层级、技术服务商、客户报备、商机风险、报价订单、数据权限口径、经营节奏。
2. 新增 `CrmAnalysisPresentationTemplateService`，根据用户问题、结果标题、摘要、结果形态和表格字段匹配展示模板。
3. 在企微渠道展示层 `AnalysisChannelPresenterService` 中附加 `presentationTemplate`，Web 控制台结果保持原样。
4. 企微 Markdown 顶部增加“展示模板”提示，便于联调期确认问题匹配是否正确。
5. 新增单元测试，覆盖经营总览、权限口径、商机风险、渠道排行和漏斗诊断等模板匹配。
6. 后端构建已通过。

## 二、当前实现边界

1. 本次只做展示模板匹配，不参与事实计算。
2. 本次不改变 CRM 开放接口调用、不改变 SQL 或分析数据来源。
3. 本次不改变企业微信真实发送链路。
4. 图片生成能力已有 `WecomAnalysisTableImageService` 可复用，但尚未把 12 套模板分别渲染成定制图片卡片。
5. 模板卡片能力已有 `WecomTemplateCardBuilder` 可复用，但尚未新增“查看完整报告、继续追问、导出申请”等经营分析动作卡片。

## 三、后续实施顺序

1. 将 `presentationTemplate.renderHints` 接入 `WecomAnalysisTableImageService`，优先实现经营总览、渠道排行、商机风险、权限口径 4 类图片卡片。
2. 将 `presentationTemplate.recommendedActions` 接入企微模板卡片，提供继续追问、完整报告、导出申请等按钮。
3. 在企微长连接流程中拆分“进度消息、摘要消息、明细消息、图片消息、卡片消息”。
4. 为 P0 模板补充端到端测试，覆盖不同用户权限下的数据口径说明。
5. 将模板匹配结果写入审计，便于后续复盘“哪些问题命中哪些模板、哪些模板需要调整”。
