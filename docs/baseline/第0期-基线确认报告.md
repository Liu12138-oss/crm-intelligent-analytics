## 3. 两个标杆案例数据口径确认

### 3.1 案例一：广州办渠道下单汇总分析

**文件**：`广州办渠道下单汇总分析.html`（Chart.js 4 单页应用）

**数据口径**：
- 统计对象：广州办渠道签约合同
- 过滤条件：合同对方不为空（即"渠道签约"）；合同阶段排除"评审"和"定稿"
- 数据维度：渠道名称、年度（2023 年以前 / 2023 / 2024 / 2025 / 2026）、所属部门、负责人、合同类型、实施方、最终客户
- 核心指标：合作渠道数（253 家）、下单总量（909 单）、下单总额（11055.99 万）、平均单笔金额（12.16 万/单）
- 集中度指标：TOP5 占比 27.0%、TOP10 占比 40.3%、TOP20 占比 55.6%、一次性渠道家数 139 家（54.9%）

**图表类型**：
- 4 张 KPI 卡片
- 集中度分析卡片网格（4 格）+ 文本洞察框（核心发现 4 条）
- 年度趋势复合图：柱状（金额，左轴）+ 折线（数量，右轴），双 Y 轴
- 集中度柱状图（TOP5 / TOP10 / TOP20 / 其余）
- TOP10 横向条形图
- TOP30 分年明细表（带排名徽章 + 金额条）
- 全部 253 家渠道排名表（支持排序）
- 合同明细清单表（支持搜索 + 筛选 + 排序）

**数据来源线索**：字段名 `customer / contract_no / impl / date / dept / amount / owner / type / channel`，数据内嵌 JSON，推测来自合同/CRM 签单系统导出。

**与联软 OpenAPI 的对接关系**：
- 合同维度 → 联软 `orders`（订单）资源，`channel` 对应 `partnerName`，`amount` 对应 `totalAmount`
- 渠道贡献排名 → `/analytics/partners/contribution` 接口
- 负责人维度 → `/analytics/owners/contribution` 接口
- 区域筛选（广州办）→ `region` 参数
