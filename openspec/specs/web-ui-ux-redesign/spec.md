## Requirements
### Requirement: Web UI 重设必须遵循 DESIGN.md

系统 MUST 以根级 `DESIGN.md` 和主规格 `ui-design-system` 为后续 Web UI/UX 重设的视觉与体验依据。重设后的页面 MUST 呈现浅色主导、Stripe 化企业工作台、统一内容骨架和响应式优先的视觉方向，并保留分析、合同审核、治理审计和权限边界的业务语义。

#### Scenario: 协作者实施 Web UI 改造

- **WHEN** 协作者开始改造 Web 页面视觉和交互体验
- **THEN** 必须先读取根级 `DESIGN.md`
- **THEN** 页面视觉不得继续以旧白底蓝灰后台风格或旧 `.pen` 原型作为强约束

### Requirement: UI/UX 改造不得修改功能代码

系统 MUST 将本次 UI/UX 改造限定在 Vue `template`、CSS class、全局样式和静态体验文案范围内。实施时 MUST NOT 修改 `<script setup>` 业务逻辑、Pinia store、service、router、types、后端 API、数据库、OpenAPI 契约或环境变量。

#### Scenario: 检查变更文件和代码区域

- **WHEN** 实施完成后审查代码 diff
- **THEN** 变更不得包含后端文件、store 文件、service 文件、router 文件或 types 文件
- **THEN** Vue 文件中不得修改 `<script setup>` 业务逻辑

### Requirement: 应用壳层必须呈现统一工作台骨架

系统 MUST 将应用壳层重设为统一工作台骨架：桌面端默认采用左侧导航、顶部吸顶栏、主内容区和右侧信息区 / 抽屉，并继续展示当前用户、角色提示、分析服务状态和退出登录入口。导航 MUST 保留现有页面入口和路由目标，不得改变路由路径。

#### Scenario: 用户在应用壳层切换页面

- **WHEN** 用户点击智能分析、合同审核、治理配置或审计检索导航项
- **THEN** 页面必须仍然跳转到现有路由
- **THEN** 当前页面必须有清晰的激活状态和业务说明
- **THEN** 顶部栏必须保持吸顶，左侧导航必须支持收起为图标轨道

### Requirement: 基础交互控件必须统一几何与标签排版

系统 MUST 将按钮、输入框、下拉框、搜索框、文本域和登录切换标签统一到同一套控件几何规则中。状态标签、顶部 chip、轻量提示胶囊和表格状态标签 MUST 默认不换行，避免出现多行标签破坏页面节奏。

#### Scenario: 协作者重构基础控件样式

- **WHEN** 协作者调整按钮、输入框、标签和 chip 的样式
- **THEN** 控件圆角必须遵循根级 `DESIGN.md` 的统一 `12px` 圆角规则
- **THEN** 标签和 chip 必须保持单行显示，不得出现两行标签

### Requirement: 分析链路必须强化 AI 问数与可信结果呈现

系统 MUST 将分析工作台和结果详情重设为以自然语言输入、阶段反馈、关键指标、AI 洞察、筛选条件、权限范围、数据更新时间、导出状态和继续追问为核心的信息架构。所有发起分析、刷新状态、导出、查看详情和提交追问的事件绑定 MUST 保持原有调用。

#### Scenario: 用户发起一次自然语言分析

- **WHEN** 用户在分析工作台输入问题并点击开始分析
- **THEN** 页面必须突出 AI 问数入口和执行中反馈
- **THEN** 按钮事件必须继续调用现有提交逻辑

#### Scenario: 用户查看分析结果详情

- **WHEN** 用户进入分析结果详情页
- **THEN** 页面必须清晰展示结果摘要、指标、图表、表格、筛选条件、可执行动作、导出入口和追问入口
- **THEN** 导出和追问必须继续触发现有函数

### Requirement: 合同审核必须突出风险优先体验

系统 MUST 将合同审核工作台和详情页重设为风险优先体验，突出上传入口、最近审核任务、总体结论、一票否决、高风险项、审核依据、降级初筛提示和产物下载状态。文件选择、拖拽上传、任务跳转、风险展开和产物下载事件 MUST 保持原有调用。

#### Scenario: 用户上传合同

- **WHEN** 用户在合同审核工作台选择或拖拽 `.docx` 文件
- **THEN** 页面必须展示清晰上传反馈和审核路径说明
- **THEN** 上传事件必须继续调用现有上传逻辑

#### Scenario: 用户查看合同审核详情

- **WHEN** 用户打开合同审核任务详情
- **THEN** 页面必须优先展示总体结论、一票否决、高风险项、审核依据和降级初筛提示
- **THEN** 风险展开和产物下载必须继续触发现有逻辑

### Requirement: 治理审计必须形成控制塔式信息层级

系统 MUST 将治理策略、模板管理、连接策略和审计检索页面重设为控制塔式体验，优先展示关键阈值、风险摘要、策略状态、模板效果、筛选条件和事件明细。保存策略、创建模板、刷新审计和筛选审计事件 MUST 保持原有调用。

#### Scenario: 管理员维护治理配置

- **WHEN** 管理员修改治理策略或连接策略并保存
- **THEN** 页面必须展示明确的配置分区、保存状态和反馈信息
- **THEN** 保存按钮必须继续触发现有保存逻辑

#### Scenario: 管理员检索审计事件

- **WHEN** 管理员刷新或筛选审计事件
- **THEN** 页面必须展示控制塔式摘要和事件流明细
- **THEN** 刷新和筛选按钮必须继续调用现有查询逻辑

### Requirement: 登录页必须保持现有文案与能力不变并提升信任感

系统 MUST 保留账号密码登录、企业微信扫码登录、绑定提示、错误反馈和帮助入口能力，并继续沿用当前 `LoginPage.vue` 的系统名称、副标题、登录标题、输入占位和扫码提示口径，仅按 `DESIGN.md` 优化视觉氛围、布局、反馈状态和响应式表现。

#### Scenario: 用户使用登录页

- **WHEN** 用户切换账号密码登录或企业微信扫码登录
- **THEN** 页面必须保持现有登录方式和反馈逻辑
- **THEN** 视觉必须与新的 AI 工作台设计系统一致

#### Scenario: 登录页增强左侧辅助区

- **WHEN** 协作者优化登录页视觉
- **THEN** 左侧只允许增加精简的系统能力说明
- **THEN** 右侧登录面板仍必须保持现有登录流程为主
- **THEN** 不得新增独立营销文案体系

#### Scenario: 登录页左右区块形成统一画布

- **WHEN** 协作者增强登录页左侧辅助区
- **THEN** 左右区块必须共享同一背景画布、对齐基线或柔和视觉桥接
- **THEN** 登录页不得表现为两个互不相关的独立漂浮卡片

### Requirement: 页面必须覆盖关键交互状态和移动端可读性

系统 MUST 为按钮、输入框、文本域、上传区、表格行、导航项、筛选项和风险项提供清晰的悬浮、点击、聚焦、禁用、加载、成功、失败、空状态和风险状态。页面 MUST 在窄屏下保持问数输入、关键指标、风险结论、表单和表格摘要可读，并满足根级 `DESIGN.md` 中的统一圆角、标签不换行和响应式断点要求。

#### Scenario: 用户遇到空状态或失败状态

- **WHEN** 页面没有数据、请求失败、导出被阻断或审核降级
- **THEN** 页面必须展示原因说明和下一步建议
- **THEN** 状态表达不得只依赖颜色

#### Scenario: 用户在窄屏浏览核心页面

- **WHEN** 页面宽度不足以展示完整桌面布局
- **THEN** 核心输入、关键指标、风险结论和主要操作必须保持可读可用
- **THEN** 页面不得出现横向溢出依赖用户缩放

#### Scenario: 登录页在小屏幕展示

- **WHEN** 登录页在中小屏或移动端展示
- **THEN** 左侧辅助区可以折叠、隐藏或下沉为简版说明
- **THEN** 右侧登录流程必须完整保留，不得因为响应式改造而丢失管理员信息、扫码区或登录入口

### Requirement: 应用壳层导航切换必须即时响应

系统 MUST 将应用壳层中的一级菜单点击设计为即时响应交互。对于同一页签内已经完成登录的用户，点击智能分析、经营报表、智能合同、治理配置或审计中心等一级菜单时，页面 MUST 先完成导航激活态更新、主内容切换和基础骨架展示；系统 MUST NOT 让用户长期停留在原页面等待重接口加载完成后才看到目标页。

#### Scenario: 用户点击一级菜单后立即看到目标页骨架

- **WHEN** 用户在应用壳层左侧导航点击任一一级菜单
- **THEN** 当前导航项激活态必须立即更新
- **THEN** 主内容区必须快速切换到目标页骨架，而不是继续停留在旧页面等待数据

### Requirement: 重页面必须优先复用最近内容并对重区块懒加载

系统 MUST 为重页面提供最近内容复用与区块级懒加载体验。再次进入页面时，若当前页签内存在最近一次成功状态，页面 SHOULD 优先展示该状态；若需要刷新，系统 MUST 通过静默刷新、局部骨架、局部失败提示和重试入口完成，而不是让整页重新空白等待。权限中心、经营报表、智能分析等重页面 MUST 明确哪些区块属于首屏必要信息，哪些区块必须延后加载。

#### Scenario: 权限中心再次进入时复用摘要与列表

- **WHEN** 用户已经在当前页签打开过权限中心并成功看到摘要或部分列表
- **AND** 用户切换到其它一级页面后再次返回
- **THEN** 页面应优先复用最近一次成功内容
- **THEN** 若需要更新重区块，必须以局部刷新方式完成

#### Scenario: 经营报表重专题不得阻塞整页展示

- **WHEN** 用户进入经营报表页面
- **THEN** 页面必须先展示标题区、筛选上下文和首屏摘要
- **THEN** 区域经营、收款明细或其它重专题可以在页面显示后再按需加载
- **THEN** 单个专题失败时必须优先局部降级，不得让整页空白

### Requirement: Web UI 改造必须按业务模块建立视觉识别

系统 MUST 基于 `DESIGN.md` 的业务辅助色、图标图形体系和页面级视觉改造基线，为每个主要 Web 业务模块建立稳定视觉识别。视觉识别 MUST 体现在模块徽标、页头状态、关键图表、空状态、表格主列、风险项或对象标签中，不得只依赖左侧导航高亮和页面标题。

#### Scenario: 用户进入任一主要业务页面

- **WHEN** 用户进入智能分析、经营报表、智能合同、权限中心、查询模板、连接策略、AI 配置或审计中心
- **THEN** 主内容区必须展示能识别当前业务模块的视觉锚点
- **THEN** 页面不得只靠导航激活态和标题说明当前模块

#### Scenario: 页面展示业务对象列表

- **WHEN** 页面展示模板、策略、合同任务、审计事件、连接配置、权限对象或 AI Profile 列表
- **THEN** 主列或状态列必须包含对象图标、等级标签、状态点或模块色标识
- **THEN** 列表不得退化为纯文字表格

### Requirement: 智能分析工作台必须升级为经营分析工作台首屏

智能分析工作台 MUST 从单一问数输入页升级为经营分析工作台首屏。页面 MUST 同时呈现 AI 问数入口、权限或数据边界、常用 / 最近资产、经营概览或推荐分析入口。未产生结果时，页面 MUST 提供有经营信息价值的概览或推荐入口，而不是大面积空白。

#### Scenario: 用户首次进入智能分析工作台

- **WHEN** 用户进入智能分析工作台且当前没有分析结果
- **THEN** 页面必须显示 AI 问数输入入口
- **THEN** 页面必须显示权限范围、数据更新时间、AI 服务状态或同等可信边界信息
- **THEN** 页面必须显示常用查询、最近查询、推荐分析或经营概览中的至少一类内容

#### Scenario: 用户查看分析结果

- **WHEN** 一次分析结果返回并展示在工作台
- **THEN** 页面必须展示结论摘要、关键发现、图表、表格、口径边界和后续动作
- **THEN** 关键发现、图表和指标必须使用图标、图表色板或状态标签强化扫描效率
- **THEN** 页面不得只堆叠普通结果白卡

### Requirement: 查询模板与模板治理必须采用方案库式结构

查询模板与模板治理页面 MUST 优先采用“左侧分类 / 中间列表 / 右侧详情或抽屉”的方案库式结构。模板列表 MUST 展示模板等级、来源、使用频率、更新时间和权限范围等可扫描信息，并通过图标、等级色或标签区分分类、模板、规则和行业包。

#### Scenario: 用户浏览查询模板列表

- **WHEN** 用户打开查询模板或模板治理页面
- **THEN** 页面必须提供分类或分组入口
- **THEN** 模板列表必须展示模板名称以外的关键元信息
- **THEN** 模板等级、来源或状态必须以中文标签和非文字视觉线索共同表达

#### Scenario: 用户查看模板详情

- **WHEN** 用户点击模板、分类、规则或行业包详情
- **THEN** 页面必须通过右侧详情抽屉、详情面板或同等结构展示详情
- **THEN** 列表上下文必须保持可返回或可继续操作

### Requirement: 审计中心必须升级为事件复核工作台

审计中心 MUST 以事件复核工作台组织，而不是纯日志表。首屏 MUST 展示聚合摘要，事件列表 MUST 展示事件类型、风险等级、行为人、对象、时间、处理状态和复核入口。事件详情 MUST 优先使用抽屉展示原因、影响范围、请求上下文、审计证据和下一步动作。

#### Scenario: 管理员进入审计中心

- **WHEN** 管理员进入审计中心
- **THEN** 首屏必须展示今日查询量、拦截量、导出量、敏感命中或待复核高风险中的多个摘要
- **THEN** 事件列表必须具备事件类型图标和风险等级标签
- **THEN** 页面不得只展示技术日志字段表

#### Scenario: 管理员复核审计事件

- **WHEN** 管理员打开审计事件详情
- **THEN** 详情必须展示事件原因、影响范围、审计证据和建议动作
- **THEN** 详情必须隐藏或降噪低频技术字段，除非当前入口为治理排障或开发诊断

### Requirement: 合同审核必须提供风险优先三层阅读结构

合同审核页面 MUST 采用风险优先体验。工作台首屏 MUST 突出上传入口、当前任务、审核模式和最近结果；详情页 MUST 形成总体结论、风险项清单、原文证据 / 审核依据三层阅读结构。风险项 MUST 同时使用等级图标、风险色、影响条款和建议动作。

#### Scenario: 用户打开合同审核工作台

- **WHEN** 用户打开合同审核工作台
- **THEN** 页面必须突出合同上传入口和当前任务状态
- **THEN** 页面必须展示审核模式、最近结果或任务列表中的至少一类上下文

#### Scenario: 用户查看合同审核详情

- **WHEN** 用户查看合同审核详情
- **THEN** 页面必须优先展示总体结论
- **THEN** 风险项必须展示等级图标、风险等级、影响条款、原因和建议动作
- **THEN** 原文证据或审核依据必须能与风险项建立对应关系

### Requirement: 经营报表必须强化数据可视化与业务口径

经营报表页面 MUST 展示周期、负责人、部门、客户范围、更新时间和核心指标口径。报表图表 MUST 优先使用趋势线、排行条形、漏斗、矩阵表或经营异常标记，并使用 `DESIGN.md` 的图表色板和文字摘要保证可解释性。

#### Scenario: 用户查看经营报表首页

- **WHEN** 用户进入经营报表页面
- **THEN** 页面必须展示当前周期、组织范围或筛选上下文
- **THEN** 核心指标必须展示单位、口径、更新时间或比较基准
- **THEN** 页面不得只展示孤立数字

#### Scenario: 用户查看报表图表

- **WHEN** 页面展示趋势、排行、漏斗或矩阵数据
- **THEN** 图表必须使用统一图表色板
- **THEN** 图表必须提供标题、时间范围、口径说明或文字摘要
- **THEN** 图表必须与明细表格或摘要说明保持语义一致

### Requirement: 配置型页面必须使用紧凑页头和抽屉详情

权限中心、连接策略和 AI 配置等配置型页面 MUST 使用紧凑页头、表格 / 列表主区和右侧抽屉详情组织。页面 MUST 突出对象类型、状态、生效范围、最近变更、健康状态和影响范围，不得以大面积说明卡长期占据首屏。

#### Scenario: 管理员维护权限或连接配置

- **WHEN** 管理员进入权限中心或连接策略页面
- **THEN** 页面必须优先展示对象列表、状态、生效范围和主操作
- **THEN** 低频说明必须收纳到 tooltip、Popover、折叠区或抽屉

#### Scenario: 管理员维护 AI 配置

- **WHEN** 管理员进入 AI 配置页面
- **THEN** 页面必须分层展示当前激活配置、模型、上下文策略、健康检查或切换历史
- **THEN** 测试失败、降级和未验证状态必须有中文原因和下一步动作

### Requirement: Web UI 改造必须通过响应式和截图验收

每批页面 UI 改造完成后，系统 MUST 对目标页面进行桌面端和窄屏响应式验收。验收 MUST 至少确认文字不重叠、按钮不溢出、标签内部不换行、表格不撑开页面、抽屉可滚动、主操作可聚焦并且关键状态有图标或文字说明。

#### Scenario: 协作者完成单页 UI 改造

- **WHEN** 协作者完成任一目标页面 UI 改造
- **THEN** 必须在桌面宽度检查页面主信息结构和视觉锚点
- **THEN** 必须在窄屏宽度检查主操作、标签、表格和抽屉的可用性
- **THEN** 不得只用桌面单截图作为验收依据

#### Scenario: 协作者完成一批页面改造

- **WHEN** 协作者完成一批页面改造
- **THEN** 必须运行对应 lint、单元测试或 E2E 验证
- **THEN** 必须确认现有路由、store、service 调用和请求反馈未回归
