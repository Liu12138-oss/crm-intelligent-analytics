## Purpose
定义自由问数共享知识层的治理边界，确保别名、时间口径、组织归一、已验证问法和高风险问法样例可以被后台维护、发布、审计和回滚，同时不突破既有受控分析安全栈。
## Requirements
### Requirement: 系统必须提供可治理的问数语义资产目录
系统 MUST 为受控问数提供可治理的语义资产目录。语义资产目录 MUST 至少覆盖指标 / 业务口径别名、时间字段与默认时间口径提示、组织 / 团队归一提示、已验证问法示例和负例 / 高风险问法示例。治理后台 MUST 允许管理员以中文业务文案维护这些资产，并区分资产类型、状态、更新时间和最近发布版本；系统 MUST 不得继续只依赖代码常量作为唯一问数知识来源。

#### Scenario: 管理员可在后台维护语义资产
- **WHEN** 管理员进入语义资产治理入口
- **THEN** 系统必须允许其查看并维护别名、时间口径、组织归一提示、已验证问法和负例样例
- **THEN** 页面不得只暴露开发内部字段名而缺少中文业务说明

#### Scenario: 语义资产目录覆盖受控问数最小知识集合
- **WHEN** 系统为自由问数准备共享知识上下文
- **THEN** 语义资产目录必须至少能提供别名、时间口径、组织归一、已验证问法和负例样例这五类知识
- **THEN** 系统不得因为某一类资产缺少独立模型而把它继续硬编码回业务服务

### Requirement: 系统必须以发布态快照向分析主链提供语义资产
系统 MUST 将语义资产的编辑态与分析主链消费态分离。治理后台中的编辑内容 MUST 先进入草稿或等价编辑态；受控分析主链、统一 AI 理解层和受控查询任务生成器 MUST 只消费已发布的语义资产快照。每次发布后的资产快照 MUST 至少保留资产 ID、资产类型、版本或发布时间和启停状态，以便执行轨迹、审计和回滚引用。

#### Scenario: 编辑中的草稿不直接进入正式问数主链
- **WHEN** 管理员刚修改一条语义资产但尚未发布
- **THEN** 正式问数主链不得直接消费这条编辑中的内容
- **THEN** 系统必须继续使用上一版已发布快照或安全兜底逻辑

#### Scenario: 发布后的资产快照可被执行轨迹引用
- **WHEN** 某次问数命中一条已发布的语义资产
- **THEN** 系统必须能够在执行轨迹或审计快照中记录该资产的标识和版本信息
- **THEN** 后续排障时必须能解释本次问数实际消费的是哪一版资产

### Requirement: 语义资产必须服从受控分析边界
语义资产 MUST 只能在既有受控分析边界内提供语义提示，不得绕过表字段白名单、权限注入、时间边界校验、AST 校验、API-first 路由和导出限制。语义资产中涉及字段、对象、时间口径或组织归一时，系统 MUST 校验其引用内容仍位于批准范围内；超出受控边界的资产内容 MUST 被阻断、拒绝发布或在消费时忽略，并留下可排障提示。

#### Scenario: 资产引用未批准字段时被阻断
- **WHEN** 管理员尝试维护一条引用未批准字段或未批准对象的语义资产
- **THEN** 系统必须阻止该资产发布或正式生效
- **THEN** 系统不得让这条资产进入统一问数知识上下文

#### Scenario: 负例样例只能驱动澄清或阻断
- **WHEN** 某条负例或高风险问法样例命中当前用户输入
- **THEN** 系统必须只允许进入补问、阻断或安全兜底路径
- **THEN** 系统不得因为命中负例样例而扩大读取范围或生成自由 schema 探测行为

### Requirement: 系统必须审计语义资产生命周期与发布变更
系统 MUST 为语义资产的新增、修改、启停、发布和回退记录审计事件。审计事件 MUST 至少包含操作者、资产类型、资产标识、变更前后摘要、发布结果和发生时间；当语义资产发布后影响正式问数行为时，系统 MUST 能从执行轨迹反查到对应资产发布记录。

#### Scenario: 发布语义资产时写入审计
- **WHEN** 管理员发布一组新的语义资产快照
- **THEN** 系统必须记录发布审计事件
- **THEN** 审计内容必须包含资产类型、数量、版本摘要和操作者信息

#### Scenario: 执行轨迹可反查发布记录
- **WHEN** 审计人员查看一条问数执行轨迹中命中的语义资产
- **THEN** 系统必须能够继续追溯到该资产的发布或最近一次生效变更记录
- **THEN** 审计链路不得停留在“命中了某条资产”而无法解释该资产从何而来
