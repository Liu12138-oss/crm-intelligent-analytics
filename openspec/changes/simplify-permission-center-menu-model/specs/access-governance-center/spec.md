## MODIFIED Requirements

### Requirement: 系统必须按 CRM 角色维护菜单与动作能力矩阵

系统 MUST 提供按 CRM `roleId` 维护的角色能力矩阵，但管理员可见编辑模型 MUST 收口为“菜单权限包 + 风险子权限”。每个角色 MUST 至少可配置角色状态、`智能分析`、`经营报表`、`智能合同审核`、`企业微信机器人`、`权限中心`、`查询模板管理`、`连接策略`、`AI配置`、`审计中心` 等菜单包，以及 `智能分析/导出数据`、`经营报表/导出数据`、`智能合同审核/查询他人合同`、`智能合同审核/下载他人合同或审核产物` 等风险子权限。系统 MAY 在内部继续持久化或返回 `visibleMenus`、`actionKeys`、`webConsoleEnabled`、`wecomBotEligible`、`exportAllowed`、`templateManageAllowed`、`contractReviewUploadAllowed`、`contractReviewCrossViewAllowed` 和 `contractReviewCrossDownloadAllowed` 等兼容字段，但这些字段不得继续作为普通管理员日常编辑入口的主要配置项。应用层能力配置 MUST 只能收紧入口和动作，不得扩大 CRM 已定义的数据范围、对象范围或字段范围。

#### Scenario: 平台管理员通过菜单权限包更新 CRM 角色
- **WHEN** 平台管理员在权限中心编辑某个 CRM 角色并勾选 `智能分析`、`经营报表` 或其它菜单包后保存
- **THEN** 系统必须持久化该角色的简化权限配置或等价映射结果
- **THEN** 后续命中该角色的用户在下一次能力快照请求或入口鉴权时必须获得该菜单包对应的常规能力

#### Scenario: 业务菜单的风险动作必须作为子权限配置
- **WHEN** 平台管理员只勾选 `智能分析` 但未勾选 `智能分析/导出数据`
- **THEN** 命中该角色的用户必须可以进入智能分析并发起常规问数与追问
- **THEN** 系统必须继续阻断该用户导出智能分析结果

#### Scenario: 系统维护菜单勾选后默认具备该页面维护能力
- **WHEN** 平台管理员为某个 CRM 角色勾选 `查询模板管理`
- **THEN** 系统必须默认授予该角色查询模板治理页面的常规维护能力
- **THEN** 管理员不得再被要求额外勾选单独的 `模板治理` 动作才能使用该菜单

#### Scenario: 菜单包授权不得扩大 CRM 数据范围
- **WHEN** 平台管理员为某个 CRM 角色开通 `智能分析`、`经营报表`、`智能合同审核` 或 `企业微信机器人`
- **THEN** 系统必须仍按该角色当前 CRM 组织、部门、负责人和字段权限生成最终 `scopeSnapshot`
- **THEN** 不得因为应用层开通菜单包就扩大其 CRM 数据范围、合同对象范围或字段范围

### Requirement: 系统必须返回统一的有效权限快照供前端消费

系统 MUST 提供统一的有效权限快照接口，返回当前登录用户的 `scopeSummary`、`roleNames`、`visibleMenus`、`actionKeys`、`wecomBotAccessState`、`wecomBotAccessReason`、合同审核权限摘要、导出资格与剩余次数、模板可见数量、数据新鲜度和服务状态。前端 MUST 基于该快照决定导航显隐、风险动作可用态和无权限提示，不得再通过散落的角色名关键字硬编码判断菜单可见性。若权限中心采用简化菜单权限包编辑，能力快照仍 MUST 返回运行时所需的菜单、动作和风险权限状态，保证旧页面与后端执行点可以继续消费。

#### Scenario: Web 工作台根据有效权限快照决定菜单显隐
- **WHEN** 登录用户请求工作台初始化能力快照
- **THEN** 系统必须返回该用户当前可见的菜单键集合和动作键集合
- **THEN** 前端必须据此决定 `智能分析`、`经营报表`、`智能合同审核`、`查询模板管理`、`权限中心`、`连接策略`、`审计中心` 和 `AI配置` 的可见性

#### Scenario: 简化菜单包仍返回运行时动作状态
- **WHEN** 某个角色通过简化权限树被授予 `经营报表` 但未授予 `经营报表/导出数据`
- **THEN** 能力快照必须包含 `management-report` 菜单和 `management.report.view` 等常规动作
- **THEN** 能力快照不得包含可放行经营报表导出的动作状态

#### Scenario: 权限变更后下一次能力快照立即生效
- **WHEN** 管理员修改某角色的菜单包或风险子权限后，命中该角色的用户再次请求能力快照
- **THEN** 系统必须返回新的有效权限快照
- **THEN** 不得继续返回旧的菜单、动作、风险子权限或企业微信入口状态

### Requirement: 系统必须提供用户级有效权限预览能力

系统 MUST 提供用户级有效权限预览接口，允许平台管理员基于 `crmUserId` 或 `wecomUserId` 预览某个用户在当前时刻命中的角色权限、菜单包、风险子权限、企业微信入口状态、合同审核权限和最终拒绝原因。预览结果 MUST 复用与正式运行相同的权限决策服务和简化权限映射，不得另写一套仅供页面展示的伪逻辑。预览结果 SHOULD 同时展示业务可读的菜单包名称和必要的内部动作摘要，帮助管理员排查“为什么页面可见但风险动作不可用”。

#### Scenario: 管理员按 CRM 用户预览最终有效权限
- **WHEN** 平台管理员输入某个 `crmUserId` 发起权限预览
- **THEN** 系统必须返回该用户命中的角色、菜单包、风险子权限、导出资格、合同审核资格和企业微信入口状态
- **THEN** 返回结果必须与该用户真实运行时会命中的权限判断保持一致

#### Scenario: 管理员按企业微信用户预览映射后的权限
- **WHEN** 平台管理员输入某个 `wecomUserId` 发起权限预览
- **THEN** 系统必须先完成企业微信到 CRM 用户的映射解析
- **THEN** 若映射成功，系统返回映射后的最终有效权限
- **THEN** 若映射失败，系统返回 `UNMAPPED_CRM_IDENTITY` 或等价状态，而不是伪造权限结果

#### Scenario: 预览结果展示菜单包与风险动作差异
- **WHEN** 某个用户拥有 `智能合同审核` 菜单包但没有 `查询他人合同` 和 `下载他人合同或审核产物`
- **THEN** 权限预览必须明确显示该用户可使用合同审核常规能力
- **THEN** 权限预览必须明确显示跨合同查看和跨产物下载不可用

## ADDED Requirements

### Requirement: 系统必须在简化权限树和内部运行时权限之间提供统一映射

系统 MUST 提供统一的简化权限映射能力，用于将历史角色权限回显为菜单权限包，并将新菜单权限包保存为当前运行时仍可消费的 `visibleMenus`、`actionKeys` 和兼容布尔字段。权限中心列表、编辑抽屉、保存接口、用户权限预览和权限覆盖测试 MUST 复用同一套映射规则，不得各自实现不同转换逻辑。

#### Scenario: 历史智能分析权限回显为智能分析菜单包
- **WHEN** 某个历史角色包含 `analysis-workbench`、`analysis.use`、`analysis.follow_up` 或 `template.view` 任一权限
- **THEN** 权限中心编辑抽屉必须回显该角色已勾选 `智能分析`
- **THEN** 不得因为历史数据缺少其中某个动作就把该角色显示为没有智能分析权限

#### Scenario: 新简化配置保存为旧运行时字段
- **WHEN** 管理员勾选 `智能分析` 并保存
- **THEN** 系统必须生成 `analysis-workbench` 菜单和智能分析常规动作
- **THEN** 现有智能分析页面、接口和后端权限校验必须能够继续使用生成后的运行时字段

#### Scenario: 风险子权限不得脱离主菜单残留
- **WHEN** 管理员取消勾选 `智能合同审核`
- **THEN** 系统必须同时清理该角色的 `查询他人合同` 和 `下载他人合同或审核产物` 风险子权限
- **THEN** 后续能力快照不得保留可放行合同跨查看或跨下载的残留动作

### Requirement: Web 入口资格必须由 Web 菜单派生

系统 MUST 不再要求管理员单独配置 `Web 入口`。角色只要拥有至少一个 Web 菜单包，系统 MUST 派生该角色具备 Web 工作台入口资格；角色没有任何 Web 菜单包时，系统 MUST 派生该角色不具备 Web 工作台入口资格。历史存在 `webConsoleEnabled=true` 但没有任何菜单的角色 MUST 被识别为异常半配置，不得在管理员未确认菜单前通过保存操作静默收缩或扩大权限。

#### Scenario: 勾选任意 Web 菜单后具备 Web 入口
- **WHEN** 管理员为某角色勾选 `经营报表`
- **THEN** 系统必须派生该角色具备 Web 工作台入口资格
- **THEN** 命中该角色的用户登录后必须可以进入其有权访问的 Web 首页

#### Scenario: 没有任何 Web 菜单时不具备 Web 入口
- **WHEN** 某个启用角色只勾选 `企业微信机器人`，未勾选任何 Web 菜单
- **THEN** 系统必须派生该角色不具备 Web 工作台入口资格
- **THEN** 该派生结果不得影响其企业微信机器人准入判断

#### Scenario: 历史半配置角色需要显式处理
- **WHEN** 某个历史角色仅有 `webConsoleEnabled=true` 但没有任何 Web 菜单或动作
- **THEN** 权限中心必须将该角色标识为需要补充菜单的异常配置
- **THEN** 系统不得在管理员未选择菜单的情况下把该角色自动保存成新的有效菜单包
