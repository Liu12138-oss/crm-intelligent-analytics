## MODIFIED Requirements

### Requirement: 系统必须为企业微信机器人入口提供独立灰度准入模式

系统 MUST 为 `wecom-bot` 提供独立的灰度准入策略，并至少支持 `DISABLED`、`PILOT_ONLY` 和 `FULL` 三种模式。`DISABLED` 表示企业微信机器人入口整体关闭；`PILOT_ONLY` 表示只有命中灰度白名单的用户可进入；`FULL` 表示所有已具备 `企业微信机器人` 角色权限包的用户都可进入灰度之后的机器人准入判断。该灰度模式 MUST 独立于 Web 入口存在，不得因为企业微信机器人关闭就同时关闭 Web 工作台。角色权限中心不再要求管理员分别配置企业微信问数、新增客户、新增商机、跟进写回和日报预览动作；这些运行时动作 MAY 由 `企业微信机器人` 权限包统一派生。

#### Scenario: 企业微信机器人处于关闭模式
- **WHEN** 企业微信机器人灰度模式被设置为 `DISABLED`
- **THEN** 系统必须拒绝所有企业微信机器人入口请求
- **THEN** 同一用户若具备 Web 工作台权限，仍可继续使用 Web 入口

#### Scenario: 企业微信机器人处于全量开放模式
- **WHEN** 企业微信机器人处于 `FULL` 模式
- **AND** 某个用户已完成企业微信到 CRM 身份映射并具备 `企业微信机器人` 权限包
- **THEN** 系统必须允许该用户进入机器人入口的后续业务编排判断
- **THEN** 不得再要求管理员额外配置单独的企业微信问数、日报预览或写回动作

#### Scenario: 未配置企业微信机器人权限包的用户仍被拒绝
- **WHEN** 企业微信机器人处于 `FULL` 模式
- **AND** 某个用户已完成企业微信到 CRM 身份映射但没有 `企业微信机器人` 权限包
- **THEN** 系统必须拒绝该用户进入企业微信机器人业务编排
- **THEN** 拒绝提示必须说明当前角色未开通企业微信机器人入口资格或等价语义

### Requirement: 系统必须在企业微信正式业务编排前完成灰度准入判断

系统 MUST 在企业微信入口完成签名校验、来源校验、消息结构校验和身份映射后、进入正式业务编排前执行角色权限包与灰度准入判断。未通过 `企业微信机器人` 权限包、灰度模式、白名单或停用名单的消息 MUST 被直接拒绝，并记录统一的准入拒绝事件；系统不得让未通过准入的消息继续进入分析、受控新增、跟进写回、日报预览或其它正式业务执行链路。通过准入后，系统仍 MUST 对具体工作流执行固定安全前置检查、CRM 数据范围、对象级权限、确认和审计约束。

#### Scenario: 灰度未命中用户在进入正式分析链路前被拒绝
- **WHEN** 企业微信发送者完成身份映射且具备 `企业微信机器人` 权限包，但未命中当前灰度白名单
- **THEN** 系统必须在进入正式分析链路前阻断该消息
- **THEN** 不得创建正式分析请求、合同任务或业务写回草稿

#### Scenario: 命中灰度的用户可继续进入工作流安全判断
- **WHEN** 企业微信发送者已具备 `企业微信机器人` 权限包并命中灰度准入
- **THEN** 系统才可以继续进入问数、受控新增、跟进写回或日报预览等工作流
- **THEN** 灰度命中和机器人权限包本身不得替代字段白名单、CRM 范围、对象级授权、用户确认或幂等判断

#### Scenario: 企业微信写回仍必须执行对象级授权
- **WHEN** 企业微信用户已通过机器人权限包和灰度准入
- **AND** 该用户对目标客户或商机不满足负责人、协作人、递归上级或管理员等对象级授权关系
- **THEN** 系统必须阻断跟进写回
- **THEN** 不得因为该用户具备企业微信机器人权限包就放行写回

## ADDED Requirements

### Requirement: 企业微信机器人角色配置必须以入口包形式展示

系统 MUST 在角色权限编辑界面中以 `企业微信机器人` 单个入口包展示企业微信角色侧能力。管理员勾选该入口包后，系统 MUST 派生当前已上线的企业微信机器人运行时动作，包括企业微信问数、受控新增客户、受控新增商机、跟进写回和日报预览。管理员取消该入口包后，系统 MUST 清理这些派生动作和 `wecomBotEligible` 兼容状态。

#### Scenario: 勾选企业微信机器人后派生现有机器人能力
- **WHEN** 管理员为某角色勾选 `企业微信机器人` 并保存
- **THEN** 系统必须派生企业微信机器人入口资格
- **THEN** 系统必须派生当前已上线企业微信机器人工作流所需的运行时动作

#### Scenario: 取消企业微信机器人后清理派生动作
- **WHEN** 管理员取消某角色的 `企业微信机器人` 权限包并保存
- **THEN** 系统必须清理该角色的企业微信入口资格
- **THEN** 系统必须清理企业微信问数、新增客户、新增商机、跟进写回和日报预览等派生动作

#### Scenario: 企业微信灰度策略仍独立于角色权限包
- **WHEN** 某个角色已勾选 `企业微信机器人`
- **AND** 企业微信灰度模式为 `PILOT_ONLY`
- **THEN** 命中该角色的用户仍必须命中用户、角色或部门白名单之一才能进入机器人入口
- **THEN** 角色权限包不得绕过企业微信灰度策略

### Requirement: 企业微信权限简化不得改变安全前置检查和受控执行边界

系统 MUST 保证企业微信角色配置简化后，签名校验、来源校验、文件类型与大小校验、登录鉴权、权限装载、白名单、危险动作阻断、候选唯一性、用户确认、对象级权限、CRM 官方 API 优先和审计留痕仍按现有规格执行。企业微信机器人权限包只决定角色侧是否具备机器人入口能力，不得把任何未批准的新增、修改、删除、提醒创建或非白名单写回变为可执行动作。

#### Scenario: 非白名单写回请求仍被拒绝
- **WHEN** 用户具备 `企业微信机器人` 权限包并发送非白名单对象写回请求
- **THEN** 系统必须按现有安全边界拒绝处理
- **THEN** 不得因为企业微信角色配置简化而创建未批准的业务写回草稿

#### Scenario: 受控新增仍必须等待用户确认
- **WHEN** 用户具备 `企业微信机器人` 权限包并通过机器人发起新增客户或新增商机
- **THEN** 系统必须先完成字段抽取、必填校验、候选确认和用户确认
- **THEN** 在用户确认前不得调用 CRM 创建接口

#### Scenario: 日报预览仍必须复用当前权限范围
- **WHEN** 用户具备 `企业微信机器人` 权限包并请求个人或团队日报预览
- **THEN** 系统必须按当前 CRM 身份、负责人关系和日报治理策略计算可见范围
- **THEN** 不得复用旧授权快照或绕过日报预览权限边界
