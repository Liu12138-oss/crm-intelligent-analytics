## MODIFIED Requirements

### Requirement: 权限中心中的 30 个权限点必须映射到真实执行点
系统 MUST 为权限中心中所有管理员可见菜单包、风险子权限和内部运行时权限建立唯一的运行时语义和真实执行点映射。管理员可见权限点从原基础开关、菜单可见性和动作权限大列表收口为菜单包与风险子权限后，系统 MUST 继续保证每个可见配置项至少命中一个真实页面入口、后端接口或企业微信工作流门槛；由菜单包派生出的内部动作也 MUST 保持与真实执行点一致，不得继续保留“页面可配置但系统不消费”的死配置。

#### Scenario: 管理员查看简化权限覆盖清单时不存在未映射配置项
- **WHEN** 管理员基于当前简化权限矩阵核对系统权限点与执行点的对应关系
- **THEN** 系统必须能说明每个菜单包和风险子权限对应的前端入口、后端执行点或企业微信工作流
- **THEN** 不得存在长期未绑定任何真实执行点的权限配置项

#### Scenario: 菜单包派生动作与真实执行点保持一致
- **WHEN** 管理员为某角色勾选 `智能分析`
- **THEN** 系统必须派生智能分析常规动作并允许该角色访问智能分析页面、提交问数和继续追问
- **THEN** 不得出现“菜单包预览可用但真实接口仍按旧细粒度动作拒绝”的结果

#### Scenario: 风险子权限预览结果与真实执行点保持一致
- **WHEN** 管理员在权限中心预览某个用户的 `经营报表/导出数据` 风险子权限
- **THEN** 同一用户点击经营报表导出按钮或调用导出接口时必须命中相同的允许或拒绝结果
- **THEN** 不得出现“预览拒绝但真实接口仍放行”或“预览允许但真实接口仍拒绝”

### Requirement: 系统必须区分通道准入与工作流动作准入
系统 MUST 将 Web / 企业微信通道准入与具体业务工作流安全门闩分开判断。Web 通道准入 MUST 由当前角色是否拥有任一 Web 菜单包派生；企业微信通道准入 MUST 由 `企业微信机器人` 权限包、身份映射和企业微信灰度策略共同决定。企业微信角色配置收口为入口包后，系统 MAY 由该包派生现有企业微信工作流动作，但每个工作流仍 MUST 执行自己的固定安全前置检查、CRM 数据范围、对象级授权、确认和审计规则。系统 MUST 不得把“能进入 Web 或企业微信通道”错误等同于“可以绕过业务对象和数据边界”。

#### Scenario: 企业微信机器人包放行后仍需执行对象级权限
- **WHEN** 某用户已具备 `企业微信机器人` 权限包且命中企业微信灰度准入
- **AND** 该用户尝试对无负责人、协作人或上级关系授权的客户执行跟进写回
- **THEN** 系统必须继续按目标对象授权主体关系阻断写回
- **THEN** 不得因为角色具备企业微信机器人包就跳过对象级权限判断

#### Scenario: 仅企业微信机器人包不产生 Web 入口
- **WHEN** 某个角色只勾选 `企业微信机器人`，未勾选任何 Web 菜单
- **THEN** 命中该角色的用户可以在企业微信入口继续接受身份、灰度和工作流安全判断
- **THEN** 该用户不得因此获得 Web 工作台入口

#### Scenario: Web 菜单包不替代风险动作判断
- **WHEN** 某个角色已勾选 `智能分析`，但未勾选 `智能分析/导出数据`
- **THEN** 系统必须允许该角色执行智能分析常规问数和结果查看
- **THEN** 系统必须拒绝该角色导出分析结果

### Requirement: 系统必须对系统级治理、审计检索和模板可见执行后端权限校验
系统 MUST 对系统级治理、审计检索、AI配置治理、模板可见和模板治理执行后端真实校验，而不是只依赖前端菜单显隐、路由守卫或管理员硬编码。简化权限模型中，系统维护菜单包 MAY 自动派生对应内部动作；但审计中心、AI配置、治理策略、模板管理和模板可见列表必须按统一权限决策结果执行放行或拒绝。

#### Scenario: 缺少审计中心菜单包的用户直接调用审计接口被拒绝
- **WHEN** 某个已登录用户未被授予 `审计中心` 菜单包或等价派生动作，但直接调用审计中心接口
- **THEN** 系统必须返回明确的权限不足响应
- **THEN** 不得因为该用户已登录或具有其它治理菜单权限而放行审计数据读取

#### Scenario: 智能分析菜单包派生模板可见能力
- **WHEN** 某个用户被授予 `智能分析`
- **THEN** 系统必须允许该用户获取其角色可见的常用查询模板列表
- **THEN** 不得再要求管理员额外配置单独的 `模板可见` 动作

#### Scenario: 查询模板管理菜单包派生模板治理能力
- **WHEN** 某个用户被授予 `查询模板管理`
- **THEN** 系统必须允许该用户进入查询模板治理页并执行常规模板维护
- **THEN** 不得再要求管理员额外配置单独的 `模板治理` 动作

### Requirement: 系统必须对页面级基础访问资格执行前后端一致校验
对于代表页面级基础访问资格的菜单包权限，系统 MUST 同时控制前端入口和后端页面读接口。`智能分析`、`经营报表`、`智能合同审核`、`权限中心`、`查询模板管理`、`连接策略`、`AI配置` 和 `审计中心` 不得只停留在导航显隐层；对应页面的关键只读或初始化接口必须依据统一权限结果放行。

#### Scenario: 合同审核菜单被撤销后本人任务列表也被阻断
- **WHEN** 某个用户被撤销 `智能合同审核` 菜单包资格，且未保留其它合同审核基础访问兼容权限
- **THEN** 前端不得继续展示合同审核导航入口
- **THEN** 后端合同审核工作台列表或本人任务详情接口也必须拒绝该用户访问

#### Scenario: 权限中心菜单被撤销后相关接口也被阻断
- **WHEN** 某个用户失去 `权限中心` 菜单包和其派生的系统级治理动作
- **THEN** 前端不得继续展示权限中心入口
- **THEN** 后端权限中心接口必须拒绝该用户读取摘要、矩阵、灰度策略和预览结果

#### Scenario: 经营报表菜单被授予后常规查看接口可用
- **WHEN** 某个用户被授予 `经营报表`
- **THEN** 前端必须展示经营报表入口
- **THEN** 后端经营报表初始化、快照和专题详情接口必须按该用户当前 CRM 范围正常执行

### Requirement: 系统必须对 SQL 审计与敏感明细查看执行独立动作校验
系统 MUST 为 SQL 审计查看与完整 SQL / 参数 reveal 建立运行时动作权限。简化权限模型中，`审计中心` 菜单包 MAY 自动派生 `audit.view`、`audit.sql.view` 和 `audit.sql.view_sensitive`，但 SQL 审计接口和 reveal 行为仍 MUST 经由后端统一权限决策执行校验。系统 MUST 不得因为前端展示审计中心菜单就跳过后端 SQL 审计或敏感明细 reveal 校验。

#### Scenario: 审计中心菜单包派生 SQL 审计能力
- **WHEN** 某个用户被授予 `审计中心`
- **THEN** 系统必须允许其进入审计中心并查看 SQL 审计分区
- **THEN** 能力快照必须包含可支持 SQL 审计查看的运行时动作状态

#### Scenario: 缺少审计中心菜单包的用户不能查看 SQL 审计
- **WHEN** 某个用户未被授予 `审计中心`，但直接调用 SQL 审计列表或详情接口
- **THEN** 系统必须拒绝其访问 SQL 审计数据
- **THEN** 不得因为该用户拥有其它系统维护菜单而放行 SQL 审计

#### Scenario: reveal 完整 SQL 仍必须留下审计记录
- **WHEN** 具备审计中心菜单包的管理员通过受控 reveal 行为查看完整 SQL 或完整参数
- **THEN** 系统必须允许该行为通过后端权限校验
- **THEN** reveal 行为本身必须留下新的审计记录

## ADDED Requirements

### Requirement: 菜单包常规能力不得被重复动作开关阻断

系统 MUST 保证菜单包授权后的常规能力不会再被管理员不可见的重复动作开关意外阻断。对于已纳入菜单包的常规能力，保存简化配置时 MUST 派生完整运行时动作；后端在校验这些常规能力时 MUST 使用派生后的统一权限结果，而不得继续依赖未被新界面维护的旧动作残留。

#### Scenario: 智能分析菜单包允许常规问数和追问
- **WHEN** 管理员为某角色勾选 `智能分析` 并保存
- **THEN** 命中该角色的用户必须可以发起智能分析问数
- **THEN** 命中该角色的用户必须可以在结果详情继续追问

#### Scenario: 经营报表菜单包允许常规查看
- **WHEN** 管理员为某角色勾选 `经营报表` 并保存
- **THEN** 命中该角色的用户必须可以打开经营报表并查看授权范围内的报表快照
- **THEN** 不得因为缺少旧 `management.report.view` 残留动作而被后端拒绝

#### Scenario: 系统维护菜单包允许页面常规维护
- **WHEN** 管理员为某角色勾选 `连接策略` 并保存
- **THEN** 命中该角色的用户必须可以打开连接策略页面并保存连接策略
- **THEN** 不得要求管理员额外配置隐藏的系统级治理动作

### Requirement: 风险动作必须继续独立阻断和审计

系统 MUST 对导出数据、查询他人合同、下载他人合同或审核产物等风险动作保留独立运行时校验和拒绝审计。菜单包授权只放行常规能力；风险动作未被授予时，前端必须禁用或隐藏对应操作，后端必须拒绝直接接口调用并记录可排障审计事件。

#### Scenario: 智能分析导出未授权时被阻断
- **WHEN** 用户拥有 `智能分析` 但没有 `智能分析/导出数据`
- **AND** 该用户直接调用分析结果导出接口
- **THEN** 后端必须拒绝导出
- **THEN** 系统必须记录功能权限拒绝审计事件

#### Scenario: 经营报表导出未授权时被阻断
- **WHEN** 用户拥有 `经营报表` 但没有 `经营报表/导出数据`
- **AND** 该用户点击或直接调用经营报表导出
- **THEN** 系统必须拒绝导出并返回中文无权限提示
- **THEN** 系统必须记录功能权限拒绝审计事件

#### Scenario: 合同跨查看未授权时被阻断
- **WHEN** 用户拥有 `智能合同审核` 但没有 `查询他人合同`
- **AND** 该用户尝试查看他人合同审核任务详情
- **THEN** 系统必须拒绝访问
- **THEN** 系统必须记录合同查看权限拒绝或等价功能权限拒绝审计事件

#### Scenario: 合同跨下载未授权时被阻断
- **WHEN** 用户拥有 `智能合同审核` 但没有 `下载他人合同或审核产物`
- **AND** 该用户尝试下载他人合同审核产物
- **THEN** 系统必须拒绝下载
- **THEN** 系统必须记录下载权限拒绝或等价功能权限拒绝审计事件
