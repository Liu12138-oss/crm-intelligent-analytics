## ADDED Requirements

### Requirement: SQL 审计写入不得无限阻塞普通业务主链路

系统 MUST 将普通只读 SQL 审计写入设计为不随运行态记录数量线性拖慢业务接口。普通只读 SQL 审计 MAY 使用异步队列、批量持久化或独立存储实现，但用户请求不得因为低风险只读审计的同步整文件落盘而持续秒级等待。高风险审计、阻断审计、写回审计、导出审计和 reveal 审计 MUST 保留更高可靠性保障。

#### Scenario: 普通只读 SQL 审计进入异步队列

- **WHEN** 受控问数、模板列表、身份解析或治理列表产生普通只读 SQL 审计记录
- **THEN** 系统可以先将审计记录写入受控队列
- **THEN** 用户接口不得等待完整运行态文件重写完成后才返回

#### Scenario: 高风险 SQL 审计必须可靠记录

- **WHEN** SQL 被权限或白名单阻断、CRM 写回执行、导出发起或管理员 reveal 敏感 SQL 明细
- **THEN** 系统必须使用高可靠路径记录对应审计
- **THEN** 不得因为普通只读审计异步化而丢失这些高风险审计证据

#### Scenario: 审计队列超过阈值

- **WHEN** SQL 审计异步队列长度超过配置阈值
- **THEN** 系统必须记录降级告警
- **THEN** 系统可以暂停低风险只读 SQL 审计入队
- **THEN** 系统不得暂停高风险、阻断、写回、导出和 reveal 审计

### Requirement: SQL 审计必须支持运行态保留和历史归档

系统 MUST 对保存在运行态中的 SQL 审计记录提供可配置保留策略，并支持将历史记录归档到受控文件或独立存储。归档后的近期审计查询 MUST 保持可用，历史归档 MUST 保留完整审计字段以便后续迁移或排障追溯。

#### Scenario: 运行态审计超过保留条数

- **WHEN** `sqlAuditRecords` 数量超过 `SQL_AUDIT_RUNTIME_MAX_RECORDS` 或等价配置
- **THEN** 系统必须裁剪运行态中超出保留范围的旧 SQL 审计记录
- **THEN** 裁剪不得影响最近记录的列表、详情和 reveal 权限控制

#### Scenario: 历史审计被归档

- **WHEN** 运维执行历史 SQL 审计归档
- **THEN** 归档结果必须保留审计 ID、时间、用户、模块、SQL 摘要、指纹、耗时、状态、错误摘要和权限上下文
- **THEN** 归档过程不得写入或泄露明文数据库密码、Token 或其它敏感配置

### Requirement: SQL 审计必须支持独立持久化存储

系统 MUST 支持将 SQL 审计记录持久化到独立于 `app-storage.json` 的存储中。该存储 MUST 支持按时间、用户、模块、状态、请求 ID、会话 ID 和 SQL 指纹分页检索。独立存储不得写入 CRM 业务库，除非经过单独方案评审明确授权。

#### Scenario: 审计中心读取独立 SQL 审计存储

- **WHEN** 独立 SQL 审计存储启用
- **THEN** 审计中心列表、详情和统计必须优先读取独立存储
- **THEN** 返回字段必须保持与现有前端消费结构兼容

#### Scenario: 独立存储灰度双写

- **WHEN** 系统处于 SQL 审计独立存储灰度期
- **THEN** 系统可以同时写入独立存储和运行态保留区
- **THEN** 系统必须能对账两边记录数量、失败数量和关键字段完整性

#### Scenario: 独立存储不可用

- **WHEN** 独立 SQL 审计存储暂时不可用
- **THEN** 系统必须记录降级原因
- **THEN** 普通业务接口不得因为低风险只读 SQL 审计写入失败而失败
- **THEN** 高风险审计必须进入可靠降级路径或明确告警

### Requirement: SQL 审计性能必须被持续观测

系统 MUST 记录 SQL 审计生成、入队、批量持久化、独立存储写入和查询检索的性能指标。指标 MUST 能支持判断接口慢是否由审计持久化造成。

#### Scenario: 审计持久化耗时升高

- **WHEN** SQL 审计持久化耗时超过配置阈值
- **THEN** 系统必须记录持久化耗时、队列长度、记录数量和降级状态
- **THEN** 运维必须能据此判断是否需要归档、扩容或关闭异步灰度

#### Scenario: 审计查询分页性能退化

- **WHEN** SQL 审计记录达到十万级规模后用户查询审计列表
- **THEN** 系统必须通过分页和索引稳定返回结果
- **THEN** 系统不得在普通列表查询中加载完整历史审计集合
