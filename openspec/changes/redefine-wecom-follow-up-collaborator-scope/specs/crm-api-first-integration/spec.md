## ADDED Requirements

### Requirement: 企业微信跟进授权主体读取必须优先复用 CRM 官方对象与协作人接口
当企业微信跟进写回需要解析目标对象的负责人和协作人集合时，系统 MUST 先评估并优先复用 CRM 官方对象详情接口与协作人接口。对于 `Customer`，系统 MUST 优先使用客户详情接口和客户协作人接口；对于 `Opportunity`，系统 MUST 优先使用商机详情接口和商机协作人接口。只有在官方 API 无法满足字段、权限语义、稳定性或返回结构要求时，系统才 MAY 进入受控数据库兜底，并必须记录 API 缺口原因、字段白名单、权限继承方式、审计留痕和回退方案。

#### Scenario: 官方协作人接口可用时不得默认查库
- **WHEN** 企业微信跟进写回需要读取客户或商机的协作人集合，且 CRM 官方协作人接口已满足所需字段与权限语义
- **THEN** 系统必须优先调用官方协作人接口作为主读取路径
- **THEN** 不得因为数据库更熟悉或已有旧 SQL 而绕过官方接口

#### Scenario: 官方协作人接口缺口已确认后方可进入受控兜底
- **WHEN** 企业微信跟进写回所需的协作人字段或权限语义无法通过 CRM 官方协作人接口满足
- **THEN** 设计文档必须记录 API 缺口、兜底所用表与字段白名单、权限继承来源和审计方案
- **THEN** 系统才可以进入受控数据库或受控内部接口兜底

#### Scenario: 未说明 API 缺口时禁止新增协作人直查路径
- **WHEN** 实现方案准备直接访问 `entities_assist_users` 或其它协作人关系表来支持企业微信跟进判权
- **THEN** 若未提供官方协作人接口不可用证明和受控边界说明，该方案必须被视为不合规并停止推进
