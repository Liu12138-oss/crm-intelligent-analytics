## ADDED Requirements

### Requirement: 受控分析追问与解释复用必须遵守 AI 上下文治理策略
系统 MUST 在 Web 智能分析和企业微信分析追问中统一消费后台 AI 上下文治理策略。上一轮问题、上一轮结果摘要和解释复用输入 MUST 按后台配置的长度上限收敛；普通分析追问、解释型追问和结果摘要复用 MUST 按后台配置的普通分析会话失活时长判断是否允许继续复用。系统 MUST 不得继续在追问分流、解释复用和历史结果继续分析链路中直接依赖固定字符串长度或无限期复用旧结果摘要。

#### Scenario: 解释型追问在有效窗口内复用受控上下文
- **WHEN** 用户在普通分析会话失活时长内继续发起解释型追问，且上一轮结果仍可复用
- **THEN** 系统必须按当前 AI 上下文治理策略裁剪上一轮问题和上一轮结果摘要后，再送入追问分流或解释生成链路
- **THEN** 系统不得直接把未收敛的长问题或长摘要整包复用到后续 AI 调用

#### Scenario: 普通分析会话超时后不再复用旧结果上下文
- **WHEN** 用户继续追问时，上一轮分析结果的上下文已经超过后台配置的普通分析会话失活时长
- **THEN** 系统必须停止复用旧问题、旧摘要或旧结果包作为继续追问的默认上下文
- **THEN** 系统必须将该输入当作新问题处理，或返回明确的上下文失效提示

#### Scenario: Web 解释追问的上一轮摘要超过上限时先受控裁剪
- **WHEN** Web 结果页触发解释型追问，且上一轮 `groundedExplanation`、摘要或标题超过后台配置的上一轮结果摘要保留上限
- **THEN** 系统必须先按治理策略截断或压缩可复用摘要
- **THEN** 系统不得继续用超出治理上限的整段旧结果直接参与追问分流或解释生成
