## ADDED Requirements

### Requirement: 系统必须为经营报表结果提供统计来源与一致性说明
系统 MUST 为经营报表中的核心统计结果提供来源说明和一致性说明。该说明至少 MUST 覆盖来源表、来源字段、时间字段、聚合方式和口径说明。经营报表中的摘要、图表、表格和导出结果 MUST 引用同一组后端聚合结果，不得在页面层或导出层再次产生独立统计版本。

#### Scenario: 经营报表摘要与图表共享同一统计来源
- **WHEN** 同一经营报表快照同时返回摘要指标和图表数据
- **THEN** 系统必须让这两类结果引用同一组后端聚合数据
- **THEN** 不得分别通过不同查询路径重新计算同名指标

#### Scenario: 导出沿用同一统计结果而不是重新计算
- **WHEN** 用户导出当前经营报表结果
- **THEN** 导出内容必须复用当前快照对应的统计结果
- **THEN** 不得因为导出走了另一套查询逻辑而产生与页面不同的数字

### Requirement: 系统必须在经营报表结果交付前执行关键统计一致性校验
系统 MUST 在经营报表结果正式交付前执行关键统计一致性校验。校验至少 MUST 覆盖：摘要指标与分项求和、分组拆分与总计、时间口径一致性、范围口径一致性和派生指标分母一致性。若校验失败，系统 MUST 阻断正式交付或将对应专题标记为不可用并显式说明原因。

#### Scenario: 分组拆分与总计不一致时阻断
- **WHEN** 某个经营报表专题中的分组结果求和不等于总计
- **THEN** 系统必须阻断该批次结果的正式交付或阻断该专题交付
- **THEN** 不得继续向用户展示未经校验的统计结果

#### Scenario: 派生指标分母与专题对象池不一致时阻断
- **WHEN** 某个派生指标使用的分母与当前专题对象池不一致
- **THEN** 系统必须判定该专题结果不可信
- **THEN** 系统必须返回明确的统计校验失败说明
