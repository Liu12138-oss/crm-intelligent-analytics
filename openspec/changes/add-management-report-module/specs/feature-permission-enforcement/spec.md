## ADDED Requirements

### Requirement: 系统必须为经营报表提供独立菜单与动作权限
系统 MUST 为经营报表提供独立权限点，并将其纳入统一权限矩阵与真实执行点映射。经营报表至少 MUST 拥有 `management-report` 菜单可见性、`management.report.view` 查看动作和 `management.report.export` 导出动作。前端导航、路由守卫、后端经营报表接口和导出接口 MUST 使用同一权限决策结果，不得只控制菜单显隐而放行真实接口。

#### Scenario: 缺少经营报表菜单权限的用户看不到入口且无法访问页面
- **WHEN** 某个已登录用户未被授予 `management-report` 菜单权限
- **THEN** 前端不得展示经营报表菜单入口
- **THEN** 该用户直接访问经营报表页面路由或页面初始化接口时，系统必须明确拒绝

#### Scenario: 缺少经营报表查看动作的用户不能请求快照
- **WHEN** 某个用户具备 Web 通道资格，但未被授予 `management.report.view`
- **THEN** 系统必须拒绝其请求经营报表快照接口
- **THEN** 不得因为该用户仍可访问智能分析工作台而默认放行经营报表

#### Scenario: 缺少经营报表导出动作的用户不能导出
- **WHEN** 某个用户可以查看经营报表，但未被授予 `management.report.export`
- **THEN** 前端不得展示可执行的导出入口，或必须明确置灰并说明原因
- **THEN** 后端导出接口仍必须拒绝该用户的导出请求

### Requirement: 系统必须让经营报表权限点映射到真实执行点
系统 MUST 将经营报表菜单和动作权限映射到真实页面、真实接口和真实导出执行点。权限中心中的经营报表权限配置不得成为只写配置；菜单显隐、页面路由守卫、快照接口、导出接口和审计事件必须都能说明它们对应的权限点。

#### Scenario: 权限中心预览经营报表权限与真实执行一致
- **WHEN** 管理员在权限中心预览某个用户的经营报表菜单和动作权限
- **THEN** 预览结果必须与该用户实际访问页面、请求快照和请求导出的放行结果一致
- **THEN** 不得出现“预览可见但接口仍拒绝”或“预览拒绝但接口仍放行”的情况

#### Scenario: 菜单权限撤销后真实路由也被阻断
- **WHEN** 某个角色被撤销 `management-report` 菜单权限
- **THEN** 前端不得继续显示经营报表入口
- **THEN** 该角色成员直接访问经营报表页面路由时也必须被重定向或明确拒绝

### Requirement: 系统必须为经营报表权限拒绝写入独立审计
系统 MUST 为经营报表页面访问拒绝、快照拒绝和导出拒绝写入可排障审计。审计 MUST 能区分是菜单不可见、查看动作不足还是导出动作不足，并记录对应资源、时间范围和目标部门范围。

#### Scenario: 缺少查看动作导致快照接口拒绝时写入审计
- **WHEN** 用户调用经营报表快照接口时因缺少 `management.report.view` 被拒绝
- **THEN** 系统必须写入一条权限拒绝审计事件
- **THEN** 审计事件必须明确记录被拒绝的动作键和目标接口

#### Scenario: 缺少导出动作导致导出接口拒绝时写入审计
- **WHEN** 用户调用经营报表导出接口时因缺少 `management.report.export` 被拒绝
- **THEN** 系统必须写入一条权限拒绝审计事件
- **THEN** 审计事件必须包含本次导出尝试的时间范围和部门范围摘要
