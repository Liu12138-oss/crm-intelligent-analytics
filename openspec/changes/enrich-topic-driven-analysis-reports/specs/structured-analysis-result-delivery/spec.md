## ADDED Requirements

### Requirement: 统一结果交付必须支持主题化区块报告
系统 MUST 在统一结果包中支持主题化区块报告。每份主题报告 MUST 至少由执行摘要、关键指标和明细结果三类区块组成；对于 richer report 主题，系统 SHOULD 继续补充趋势变化、结构观察、风险观察、重点对象和行动建议等区块。系统不得继续把 richer report 结果退化为只有一句摘要和一张表的交付方式。

#### Scenario: 负责人经营报告包含 richer report 区块
- **WHEN** 用户发起负责人经营类问数并命中 richer report 主题包
- **THEN** 统一结果包必须至少包含执行摘要、关键指标、风险观察和行动建议区块
- **THEN** Web 展示层不得只渲染单表结果

#### Scenario: 轻量报告仍保留最小区块结构
- **WHEN** 用户发起 snapshot 级别问数
- **THEN** 系统可以只返回最小区块集合
- **THEN** 结果包仍必须保持统一区块结构，而不是回退到无区块的纯文本结果

### Requirement: richer report 必须输出主题派生指标和经营解释
当系统交付 richer report 时，结果包 MUST 补充主题派生指标和经营解释。负责人经营报告 MUST 至少输出 `TOP1贡献占比`、`平均单笔商机金额`、`第一名领先第二名差距`、`命中商机数` 和 `分组数量` 等指标，并在执行摘要或关键发现中解释这些指标的经营含义。

#### Scenario: 负责人经营报告输出头部贡献指标
- **WHEN** 系统成功交付负责人经营主题报告
- **THEN** 报告中必须包含 `TOP1贡献占比`
- **THEN** 摘要或关键发现必须解释头部集中度带来的经营含义

#### Scenario: 负责人经营报告输出排名差距与平均单笔金额
- **WHEN** 系统成功交付负责人经营主题报告
- **THEN** 报告中必须包含第一名领先第二名差距
- **THEN** 报告中必须包含平均单笔商机金额

### Requirement: 主题报告中的不同区块必须按各自口径生成结论
系统 MUST 让不同区块按自身数据切片的真实口径生成结论。趋势区块 MUST 输出趋势性解释，分布区块 MUST 输出结构观察，排名区块 MUST 输出头部领先结论；系统不得把趋势或分布结果套用成“某人领先”的排名文案。

#### Scenario: 趋势区块不得输出负责人领先文案
- **WHEN** 某个区块的数据切片属于趋势口径
- **THEN** 系统必须输出趋势性结论
- **THEN** 系统不得将该区块误写为“某月份或某标签领先”的负责人排名结论

#### Scenario: 分布区块不得输出趋势涨跌文案
- **WHEN** 某个区块的数据切片属于结构分布口径
- **THEN** 系统必须输出结构观察结论
- **THEN** 系统不得套用趋势涨跌描述

### Requirement: Web 与企业微信必须按主题报告深度裁剪同源结果
系统 MUST 基于同一个主题结果包为 Web 和企业微信生成不同深度的同源结果。Web MUST 能展示 richer report 的完整或折叠区块；企业微信 MUST 只展示适合会话阅读的摘要、关键指标、风险提示和少量重点对象，但不得改变主题口径、派生指标事实或结论方向。

#### Scenario: Web 展示完整 richer report 区块
- **WHEN** Web 用户查看 richer report 结果
- **THEN** 页面必须可以查看完整的主题区块集合
- **THEN** 用户不得只能看到与企业微信相同的裁剪摘要

#### Scenario: 企业微信只裁剪长度不改写主题结论
- **WHEN** richer report 结果通过企业微信分发
- **THEN** 企业微信结果必须只裁剪区块数量和文本长度
- **THEN** 企业微信不得重新生成另一套与 Web 不同的结论口径
