## ADDED Requirements

### Requirement: 系统必须为自由问数定义主题驱动报告包档案
系统 MUST 为自由问数建立受控的主题驱动报告包档案。每个档案 MUST 至少定义主题标识、默认区块集合、推荐统计维度、默认查询任务包、最小可交付结果和渠道展示层级。系统不得继续把所有问数都压成单一 `owner-ranking / time-trend / distribution` 结果页，而必须能够根据主题决定应该补充哪些维度和区块。

#### Scenario: 负责人排名问数命中负责人经营报告档案
- **WHEN** 用户询问“最近一年各销售负责人新增商机金额排名”或语义等价问题
- **THEN** 系统必须命中 `owner-performance-ranking` 或等价负责人经营报告档案
- **THEN** 该档案必须至少定义排名、趋势、结构、风险、重点对象和行动建议等区块

#### Scenario: 商机风险问数命中商机风险报告档案
- **WHEN** 用户询问“最近三个月高风险商机有哪些”或语义等价问题
- **THEN** 系统必须命中 `opportunity-risk` 或等价商机风险报告档案
- **THEN** 系统不得继续返回只有单张表的轻结果，而必须补充风险口径和行动建议区块

### Requirement: 系统必须按主题报告包生成固定查询任务组合
系统 MUST 基于主题报告包生成固定的查询任务组合，而不是每次由模型自由决定补哪些统计维度。每个主题报告包 MUST 区分必选任务和可选任务，并声明执行顺序、结果用途和任务上限。Web `standard` 或 `deep-dive` 报告 SHOULD 支持 3 到 8 个任务；企业微信裁剪结果 MAY 只展示其中的关键区块，但必须消费同一主结果包。

#### Scenario: 负责人经营报告包生成多任务组合
- **WHEN** 系统命中负责人经营报告档案
- **THEN** 系统必须至少生成负责人排名、月度趋势、阶段结构、风险摘要这几类任务中的一个固定组合
- **THEN** 系统不得只执行一条负责人排名 SQL 后直接结束报告生成

#### Scenario: 可选任务失败时报告可降级但不能伪造区块
- **WHEN** 主题报告包的可选任务执行失败，但必选任务已成功
- **THEN** 系统可以继续交付降级后的主题报告
- **THEN** 系统必须显式标记缺失区块，而不得伪造未成功生成的维度结果

### Requirement: 主题报告包必须提供派生经营指标与区块化结果
主题报告包 MUST 在基础聚合结果之外，计算与主题相关的派生经营指标，并按区块组织结果。负责人经营报告档案 MUST 至少支持 `TOP1贡献占比`、`平均单笔商机金额`、`第一名领先第二名差距`、`命中商机数` 和 `分组数量` 这类派生指标；其它主题档案 MUST 具备与自身主题匹配的派生指标集合。

#### Scenario: 负责人排名报告生成头部集中度指标
- **WHEN** 负责人经营报告包成功返回负责人排名结果
- **THEN** 系统必须计算 `TOP1贡献占比`
- **THEN** 系统必须将该指标写入统一结果包和主题报告区块

#### Scenario: 负责人排名报告生成平均单笔金额
- **WHEN** 负责人经营报告包成功返回负责人金额与底层商机数
- **THEN** 系统必须计算平均单笔商机金额
- **THEN** 系统不得只返回原始累计金额和分组数量而缺少派生经营指标

### Requirement: 主题报告包必须支持区块化组织与渠道裁剪
系统 MUST 将主题报告结果组织成区块化结构，至少支持执行摘要、关键指标、趋势变化、结构观察、风险观察、重点对象、明细结果和行动建议等区块类型。Web MUST 能展示完整或折叠后的区块化结果；企业微信 MUST 仅展示裁剪后的摘要区块，但不得改写主题口径或统计事实。

#### Scenario: Web 展示负责人经营报告的多区块内容
- **WHEN** Web 用户查看负责人经营主题报告
- **THEN** 页面必须可展示执行摘要、关键指标、图表、风险观察、重点对象和明细结果
- **THEN** 系统不得继续将 richer report 压缩为单图加单表的固定布局

#### Scenario: 企业微信裁剪负责人经营报告但保持同源口径
- **WHEN** 同一份负责人经营主题报告通过企业微信交付
- **THEN** 企业微信结果必须只裁剪展示长度和区块数量
- **THEN** 其摘要和关键数字必须仍然来自同一份主题报告包
