## MODIFIED Requirements

### Requirement: 系统必须支持受控 AI 直查执行模式

系统 MUST 在受控分析编排中支持“结构化计划执行”和“受控 AI 直查执行”两类只读执行模式，并将不固定的自然语言问数默认收敛到受控 AI 直查主链。自由问数文本 MUST 先经过统一 AI 理解层，再由服务端结合当前场景、知识层命中情况和风险边界决定进入 `GUARDED_DIRECT_QUERY`、`PLAN_EXECUTION`、解释复用、补问或阻断路径；固定功能主题入口一旦被统一 AI 理解层明确命中，系统 MUST 继续走固定程序流，不得误降级为自由问数直查。受控 AI 直查执行中，模型 MUST 先生成 1 到 3 个受控查询任务，任务 MAY 表达只读 SQL、已注册只读 API / 查询工具或受控内部读取意图；系统 MUST 在实际执行前完成语句类型校验、权限范围注入、表字段白名单、结果规模限制、超时限制和审计快照记录。系统 MUST 阻止 `INSERT`、`UPDATE`、`DELETE`、`DROP`、`ALTER`、`TRUNCATE`、schema 探测、未声明写入工具和未声明外部请求。

#### Scenario: 自由自然语言问数默认进入受控直查主链
- **WHEN** Web 或企业微信用户提交一条不属于固定功能主题入口的自然语言问数
- **THEN** 系统必须优先尝试进入受控 AI 直查执行模式
- **THEN** 系统不得默认先把该问题压缩成窄枚举结构化计划后再决定是否允许直查

#### Scenario: 固定功能主题入口不得误降级为自由问数直查
- **WHEN** 企业微信或 Web 输入已经被统一 AI 理解层明确识别为跟进、创建、日报、项目查询或其它固定功能主题入口
- **THEN** 系统必须继续进入对应固定程序流
- **THEN** 系统不得因为该输入同时包含查询词或数据词就把它当作自由问数直查执行

#### Scenario: AI 直接生成受控查询任务后仍需通过安全栈执行
- **WHEN** 用户提交一个允许通过只读 SQL 或已注册只读工具回答的经营分析问题
- **THEN** 系统可以接受模型生成的受控查询任务并从中提取单条或多条只读读取步骤
- **THEN** 系统必须在执行前完成语句类型校验、权限范围注入、白名单校验和行数 / 超时限制
- **THEN** 系统只可将受控结果交给后续 grounded 总结与交付链路

#### Scenario: AI 尝试执行危险语句时被阻断
- **WHEN** 模型生成包含删除、更新、建表、改表或其它未声明危险动作的 SQL 或工具调用
- **THEN** 系统必须立即阻断执行
- **THEN** 系统必须记录阻断原因与对应执行快照

## ADDED Requirements

### Requirement: 系统必须通过统一问数知识层生成受控查询任务

系统 MUST 在不固定自然语言问数场景下引入统一问数知识层，作为统一 AI 理解层和受控查询任务生成器的共享上下文。问数知识层 MUST 至少覆盖字段别名、业务口径别名、组织 / 团队别名、码表翻译、高频查询模板、已验证查询示例和高风险字段边界。AI 在生成受控查询任务时 MUST 优先消费这些已批准知识资产，而不是每次从零假设字段语义或自由探测 schema；系统 MAY 在模板命中时直接复用已验证模板，但模板复用后的执行仍必须经过 API-first 路由、权限注入、白名单、AST 和预检链路。

#### Scenario: 自由问法通过知识层别名稳定命中受控查询任务
- **WHEN** 用户使用口语化、错别字、中英混写或业务别名描述同一类经营分析问题
- **THEN** 系统必须优先通过统一问数知识层中的别名和已验证样例稳定生成受控查询任务
- **THEN** 系统不得仅因未命中固定中文关键词就直接阻断或退回旧的关键词主链

#### Scenario: 高频问题优先复用已验证模板
- **WHEN** 用户问题命中已经批准的高频查询模板或已验证查询示例
- **THEN** 系统必须优先复用对应模板或等价模板资产生成受控查询任务
- **THEN** 系统不得在可复用模板已存在时仍强制每次从零生成新 SQL

#### Scenario: 知识层缺失时系统进入受控澄清或兜底
- **WHEN** 当前问数涉及的字段语义、口径或组织映射不在统一问数知识层中，且无法安全推断
- **THEN** 系统必须进入补问、计划执行兜底或显式阻断中的受控路径
- **THEN** 系统不得让模型继续以自由 schema 探测或自由扩大读取范围的方式补齐缺失知识
