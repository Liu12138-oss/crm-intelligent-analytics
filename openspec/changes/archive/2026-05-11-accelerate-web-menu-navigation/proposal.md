## Why

当前 Web 工作台的一级菜单切换被放在“实时会话续期 + 实时能力快照 + 页面初始化请求”这条同步链路上。用户每次点击受保护菜单时，前端都会重复请求 `session`、`capabilities`，部分页面还会继续串行或并行请求多个重量接口；即使网络面板返回 `304`，后端仍需要先执行会话解析、数据库身份装载和权限快照构建，最终让菜单切换耗时达到数秒。

这类延迟已经不是“页面数据慢”这么简单，而是直接破坏了工作台的基础交互预期。对于已经登录、刚刚完成一次权限校验的同页签用户，菜单切换应当是毫秒级的 UI 行为，而不是一次完整的鉴权往返。与此同时，本系统又不能为了追求快感而牺牲权限真源、实时阻断和审计留痕，因此需要把“热切换性能”与“敏感动作实时校验”明确拆开建模。

本次变更的目标，是在不放松统一权限矩阵、组织范围和后端执行校验的前提下，把工作台一级菜单热切换改造成“默认不阻塞路由、不重复拉会话、不让重页面首屏拖慢导航”的模型，并为后续实现提供明确的缓存、失效、懒加载和验收指标。

## What Changes

### 新增性能能力边界

- 新增 `web-navigation-performance` 能力，定义：
  - 受保护菜单热切换不得把 `session` / `capabilities` 请求放在同步导航热路径上
  - 重页面必须采用“路由即时生效 + 骨架先出 + 分区懒加载”的交付方式
  - 系统必须记录导航性能指标并建立回归验收口径

### 前端导航与页面加载模型调整

- 调整前端路由守卫：
  - 首次进入、扫码回流、显式失效时允许实时拉取会话
  - 已水合页签内的一级菜单切换默认复用当前会话与能力快照
  - 不再把所有 `requiresAuth` 路由切换都视作强制刷新
- 新增前端会话与能力快照缓存模型：
  - `session` 页签级复用
  - `capabilities` 短 TTL 复用
  - 登录、退出、401、权限治理保存后主动失效
- 新增重页面状态复用与分层加载要求：
  - `analysis`、`management-report`、`contract-review`、`audit`、`governance` 等一级页面返回时优先复用最近状态
  - 权限中心、经营报表等重页面首屏只加载摘要或必要区块，其余分区按需加载

### 后端会话解析与权限快照优化

- 新增短时会话用户快照复用机制：
  - 同一页签频繁命中的受保护接口不再每次都触发完整数据库身份解析
- 新增能力快照短时缓存与版本化失效机制：
  - 能力快照可以在受控 TTL 内复用
  - 登录、登出、权限矩阵变化、组织范围变化、401 失效时必须立刻作废
- 保留后端敏感动作与真实数据接口的最终权限校验，不把前端缓存当作放行依据

### 观测与验收

- 新增菜单点击、路由确认、首屏可交互和页面数据就绪四段性能埋点
- 新增后端会话解析、能力快照构建和页面首屏接口耗时日志
- 建立“热切换毫秒级”的回归验收标准

## Capabilities

### New Capabilities

- `web-navigation-performance`：定义工作台一级菜单热切换、页面状态复用、分区懒加载、前后端快照缓存和性能观测要求。

### Modified Capabilities

- `feature-permission-enforcement`：补充“短时权限快照可复用，但必须受统一权限决策版本和失效事件控制”的约束，确保缓存不破坏权限真源。
- `web-ui-ux-redesign`：补充“导航切换必须即时响应、重页面必须先展示骨架并复用最近状态”的体验要求，避免再次把重请求放回菜单切换热路径。

## Impact

### 前端影响

- 需要修改 `router`、`auth store`、应用壳层和各一级页面初始化策略。
- 需要为重页面补充 `KeepAlive`、状态复用、区块懒加载和显式刷新入口。
- 需要补充前端导航性能埋点与端到端性能回归。

### 后端影响

- 需要修改会话解析与能力快照构建链路，引入短时缓存、版本化失效和耗时日志。
- 需要确保缓存命中时不放松真实数据接口与敏感动作接口的统一权限校验。

### 权限与治理影响

- 不新增业务权限点，但会改变权限快照的运行时消费方式。
- 权限治理、组织范围治理、登录态失效与 401 回退将承担更多缓存失效职责。

### 测试与验收影响

- 需要新增“菜单热切换不重复发起 `session` / `capabilities` 请求”的前端回归。
- 需要新增“治理修改后下一次导航立即使用新权限结果”的失效回归。
- 需要新增会话快照、能力快照和重页面懒加载的性能基线测试。
