## Why

当前系统默认依赖本地联调身份注入与企业微信发送者映射，缺少正式的登录入口与统一认证闭环，无法满足真实用户从 Web 进入工作台和从企业微信机器人进入问数链路的生产要求。临时需求同时提出账号密码登录、企业微信扫码登录以及机器人消息认证，因此需要把“身份建立、会话落地、渠道校验”补成一期可交付能力。

## What Changes

- 新增 Web 登录页，严格参考 `.pen` 原型与 UI 规范，提供“账号密码登录”和“企业微信扫码登录”两种入口。
- 新增 CRM 认证接入层，统一封装账号密码登录、登录态换取、当前用户信息获取与失败原因映射，避免前后端直接散落调用外部 CRM API。
- 新增 Web 登录后会话建立与能力初始化流程，使工作台、治理页和审计页都基于真实登录态访问，而不是依赖测试头或临时身份注入。
- 新增企业微信扫码登录绑定流程，要求扫码成功后能够建立与 CRM 用户一致的 Web 登录态，并继承现有组织、部门、角色和数据范围。
- 新增企业微信机器人消息入口的认证校验要求，在进入问数编排前完成签名、来源、用户映射与登录态有效性校验。
- 调整现有 `SessionAuthGuard` 和企业微信接入链路，使 Web 登录态与企业微信身份映射复用同一套用户上下文装载逻辑。

## Capabilities

### New Capabilities
- `crm-login-access`: 定义 Web 登录页、账号密码登录、企业微信扫码登录、登录失败反馈、登录成功跳转与会话建立要求。
- `wecom-channel-authentication`: 定义企业微信机器人消息接入前的验签、来源校验、用户映射、登录态检查与失败处理要求。

### Modified Capabilities

无。

## Impact

- 影响前端路由、应用壳和新增登录页，重点包括 `frontend/src/router`、`frontend/src/App.vue`、`frontend/src/layouts/AppShell.vue` 与新的登录相关页面/服务。
- 影响后端认证与会话链路，重点包括 `backend/src/modules/auth/*`、`backend/src/modules/wecom/*`、`backend/src/modules/sessions/*` 与新的 CRM 认证适配服务。
- 影响运行时配置，需要补充 CRM 认证接口、企业微信扫码登录所需参数与回调地址的环境变量约定。
- 影响企业微信问数入口，要求其在消息进入分析链路前完成更严格的认证与审计记录。
