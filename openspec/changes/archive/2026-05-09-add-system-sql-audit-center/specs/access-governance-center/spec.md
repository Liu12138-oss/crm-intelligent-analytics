## ADDED Requirements

### Requirement: 审计中心必须提供独立的 SQL 审计分区

系统 MUST 在现有审计中心内新增独立的 `SQL 审计` 分区，用于检索本系统后端服务访问 CRM 数据库时的 SQL 审计记录。该分区 MUST 与现有 AI 审计、用户行为审计并列展示，而不是要求治理用户跳转到新的一级菜单或旁路排障页面。

#### Scenario: 被授权治理用户在审计中心看到 SQL 审计 Tab

- **WHEN** 具备 SQL 审计查看权限的治理用户进入审计中心
- **THEN** 页面必须展示 `SQL 审计` Tab
- **THEN** 该用户不得需要离开现有审计中心骨架才能查看 SQL 审计

#### Scenario: 未被授权用户看不到 SQL 审计分区

- **WHEN** 用户进入审计中心但未具备 SQL 审计查看权限
- **THEN** 页面不得展示 `SQL 审计` Tab
- **THEN** 该用户不得通过前端切换或直接接口调用读取 SQL 审计数据

### Requirement: SQL 审计分区必须提供摘要、筛选、明细与详情能力

SQL 审计分区 MUST 至少提供顶部摘要卡、筛选栏、分页明细表和详情抽屉。摘要卡 MUST 至少展示 SQL 总量、写 SQL 数、失败数、阻断数、高风险数和平均耗时；筛选栏 MUST 至少支持按用户、模块、数据库角色、执行阶段、操作类型、状态、表名、请求 ID、会话 ID 和时间范围检索。

#### Scenario: 治理用户按表名和时间范围检索 SQL 审计

- **WHEN** 治理用户在 SQL 审计分区输入表名并限定时间范围
- **THEN** 系统必须返回命中条件的 SQL 审计列表
- **THEN** 列表结果必须能够展示每条记录的时间、用户、模块、操作类型、状态和耗时摘要

#### Scenario: 治理用户查看单条 SQL 审计详情

- **WHEN** 治理用户点击某条 SQL 审计记录查看详情
- **THEN** 页面必须展示该 SQL 的业务上下文、技术上下文、执行来源、命中适配器、fallback 原因和脱敏 SQL 摘要
- **THEN** 若当前用户没有敏感明细权限，页面不得自动展示完整 SQL 与完整参数
