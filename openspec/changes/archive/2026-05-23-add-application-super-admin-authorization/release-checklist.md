# 应用超级管理员授权发布核对清单

## 发布前核对

- 确认历史 `analysisScopePolicy.fullAccessUserIds` 中的有效用户已经在权限中心预览中显示为用户级应用超级管理员授权。
- 确认生产环境拟授权的 CRM 用户和 CRM 角色名单已经由业务负责人复核，且每条变更都有中文原因。
- 确认企业微信机器人通道、Web 登录、SQL 白名单、字段白名单、导出次数和文件安全限制仍按全局基线生效。
- 确认权限预览对命中用户展示全部菜单、全部动作、应用超级管理员命中来源和全量范围说明。
- 确认普通用户和未命中角色仍按当前 CRM 组织、部门、负责人和数据范围白名单收口。

## 发布后验证

- 使用一个用户级应用超级管理员账号验证智能分析、经营报表、合同审核、查询模板、导出和治理入口均可访问。
- 使用一个角色级应用超级管理员账号验证同一角色下的用户获得相同能力，移出角色或停用授权后立即失效。
- 使用普通账号验证权限中心、经营报表、合同审核跨看和导出仍按原权限边界拦截。
- 检索审计事件，确认授权变更、用户权限预览和高风险操作能追溯操作者、授权主体、命中来源和变更原因。

## 回滚方式

- 在权限中心停用所有应用超级管理员授权主体，并保存中文原因。
- 若新接口异常，暂时使用旧 `/api/v1/governance/analysis-scope-policy` 兼容接口读取当前策略，但不得继续新增旧全量查询名单。
- 回滚后再次执行用户权限预览，确认相关人员回到普通角色权限和当前 CRM 数据范围。
