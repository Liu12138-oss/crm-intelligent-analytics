## ADDED Requirements

### Requirement: 统一权限决策必须消费应用超级管理员授权

系统 MUST 在统一权限决策服务中消费应用超级管理员授权。命中应用超级管理员授权的用户 MUST 被视为拥有权限目录中登记的全部菜单和全部动作，权限预览、能力快照、前端导航、后端 `ensureAction` 和真实执行点 MUST 使用同一结果。

#### Scenario: 超级管理员权限预览与真实接口一致

- **WHEN** 管理员在权限中心预览某个应用超级管理员用户
- **THEN** 预览结果必须展示全部菜单、全部动作和全部高风险能力
- **THEN** 该用户真实访问对应页面、接口或企业微信工作流时必须得到相同的允许结果

#### Scenario: 新增权限点自动纳入超级管理员

- **WHEN** 系统在统一权限目录中新增一个菜单或动作权限点
- **THEN** 应用超级管理员必须自动获得该权限点
- **THEN** 不得要求新增功能再单独维护超级管理员名单

### Requirement: 超级管理员授权不得替代安全前置检查

系统 MUST 将应用超级管理员授权限定为功能权限和数据范围覆盖，不得用它替代登录、通道、签名、来源、文件安全、SQL 安全和审计等固定前置检查。

#### Scenario: 企业微信通道关闭时超级管理员不能绕过通道关闭

- **WHEN** 全局企业微信机器人通道被关闭
- **AND** 应用超级管理员通过企业微信发起业务工作流
- **THEN** 系统必须按通道关闭规则拒绝请求
- **THEN** 不得因为其拥有全部动作权限而绕过通道关闭

#### Scenario: SQL 敏感 reveal 仍写入审计

- **WHEN** 应用超级管理员 reveal SQL 审计敏感明细
- **THEN** 系统必须允许其执行 reveal
- **THEN** reveal 行为本身必须继续写入审计事件
