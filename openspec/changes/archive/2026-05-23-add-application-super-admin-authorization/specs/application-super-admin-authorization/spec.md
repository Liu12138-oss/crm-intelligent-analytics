## ADDED Requirements

### Requirement: 系统必须提供应用超级管理员授权

系统 MUST 提供独立的应用超级管理员授权能力，允许具备系统级治理权限的管理员按 CRM 用户或 CRM 角色授予、禁用或恢复本系统超级管理员资格。该授权 MUST 记录授权主体、主体类型、状态、更新人、更新时间和中文变更原因，并且不得依赖新增的独立人员体系。

#### Scenario: 管理员按用户授予超级管理员资格

- **WHEN** 具备 `governance.policy.manage` 的管理员为某个 CRM 用户保存应用超级管理员授权
- **THEN** 系统必须将该 CRM 用户记录为应用超级管理员授权主体
- **THEN** 下一次权限快照、权限预览和真实接口校验必须识别该用户的超级管理员资格

#### Scenario: 管理员按角色授予超级管理员资格

- **WHEN** 具备 `governance.policy.manage` 的管理员为某个 CRM 角色保存应用超级管理员授权
- **THEN** 系统必须将该 CRM 角色记录为应用超级管理员授权主体
- **THEN** 命中该角色的所有当前用户必须在下一次权限解析时获得应用超级管理员资格

#### Scenario: 超级管理员授权被禁用后立即失效

- **WHEN** 管理员禁用某个用户或角色的应用超级管理员授权
- **THEN** 系统必须在下一次权限快照、范围解析和真实接口校验时移除该主体的超级管理员资格
- **THEN** 不得因为旧会话、旧能力快照或历史查询继续按超级管理员放行

### Requirement: 应用超级管理员必须拥有系统内所有功能和操作权限

命中应用超级管理员授权的主体 MUST 获得本系统内所有菜单、所有动作、所有治理入口、所有审计入口、所有 AI 配置入口、所有合同审核能力、所有导出能力和所有企业微信业务工作流动作。系统 MUST 通过统一权限决策服务返回这些权限，不得要求业务模块额外维护超级管理员白名单。

#### Scenario: 超级管理员能力快照包含全部菜单和动作

- **WHEN** 应用超级管理员请求当前能力快照
- **THEN** 系统必须返回权限目录中登记的全部菜单键和全部动作键
- **THEN** 合同上传、合同跨看、合同跨下载、审计查看、SQL 敏感明细查看、模板治理、AI 配置治理和经营报表导出必须全部处于允许状态

#### Scenario: 超级管理员直接调用治理接口被统一放行

- **WHEN** 应用超级管理员直接调用权限中心、审计中心、AI 配置或模板治理接口
- **THEN** 后端必须通过统一权限决策结果放行
- **THEN** 放行结果必须与权限中心预览中展示的菜单和动作一致

### Requirement: 应用超级管理员必须拥有全量数据范围

命中应用超级管理员授权的主体 MUST 获得全量数据范围。系统 MUST 在统一范围快照中明确标记该范围来自应用超级管理员授权，并让分析问数、经营报表、合同审批、查询模板、最近查询重跑和导出补查使用同一份范围结果。

#### Scenario: 超级管理员范围快照标记全量范围

- **WHEN** 系统为应用超级管理员解析数据范围
- **THEN** 返回的范围快照必须明确标记为应用超级管理员全量范围
- **THEN** 范围说明必须使用中文说明“当前已开通应用超级管理员授权，可查看全公司数据”

#### Scenario: 超级管理员查询经营报表不被本人 owner 范围收口

- **WHEN** 应用超级管理员打开经营报表并选择全公司范围
- **THEN** 系统必须按全公司范围查询经营数据
- **THEN** 不得继续使用该用户本人 `ownerIds` 或登录部门作为最终收口条件

#### Scenario: 超级管理员查看合同审批列表不被本人 owner 范围收口

- **WHEN** 应用超级管理员查看智能合同审核中的 CRM 待审批合同列表
- **THEN** 系统必须允许其查看授权组织内的待审批合同
- **THEN** 不得因为合同负责人不是当前用户而过滤掉结果

### Requirement: 应用超级管理员不得绕过固定安全边界

应用超级管理员授权 MUST NOT 绕过登录认证、会话有效性、企业微信签名校验、来源校验、通道关闭、文件类型和大小限制、SQL 白名单、字段白名单、敏感字段处理、危险动作确认和审计留痕。超级管理员只能获得系统内业务功能和数据范围，不代表可以跳过固定安全前置检查。

#### Scenario: 未登录请求仍被拒绝

- **WHEN** 未登录请求访问任意需要登录的业务接口
- **THEN** 系统必须拒绝请求
- **THEN** 不得因为目标账号曾被授予应用超级管理员资格而绕过会话校验

#### Scenario: 超级管理员导出仍受全局基线约束

- **WHEN** 应用超级管理员发起导出
- **THEN** 系统必须继续执行字段白名单、敏感字段处理、导出审计和全局阈值策略
- **THEN** 不得因为其具备超级管理员资格而直接输出未治理字段或跳过审计

### Requirement: 应用超级管理员授权必须可审计和可迁移

系统 MUST 为应用超级管理员授权变更、权限预览、范围解析和高风险操作记录审计事件。系统 MUST 将历史 `analysisScopePolicy.fullAccessUserIds` 迁移为用户级应用超级管理员授权，并在迁移完成后停止把旧名单作为独立分析查询局部开关使用。

#### Scenario: 管理员保存超级管理员授权时写入审计

- **WHEN** 管理员新增、禁用、恢复或修改应用超级管理员授权
- **THEN** 系统必须写入授权变更审计事件
- **THEN** 审计事件必须包含操作者、授权主体、主体类型、变更前后摘要和变更原因

#### Scenario: 旧全量查询名单迁移为超级管理员名单

- **WHEN** 系统检测到历史 `analysisScopePolicy.fullAccessUserIds` 中存在有效用户
- **THEN** 系统必须将这些用户迁移为用户级应用超级管理员授权主体
- **THEN** 后续分析问数必须通过统一超级管理员授权和统一范围解析获得全量范围
