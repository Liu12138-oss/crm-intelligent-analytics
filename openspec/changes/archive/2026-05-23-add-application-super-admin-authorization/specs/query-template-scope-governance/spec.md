## MODIFIED Requirements

### Requirement: 简单模板必须按当前用户范围自动收口执行

当模板被识别为可自动收口模板时，系统 MUST 在执行 SQL 时基于当前用户 `scopeSnapshot` 自动注入组织、部门和负责人范围。自动注入后的执行 SQL MUST 与原始 SQL、执行参数、范围快照和审计记录建立关联。命中应用超级管理员授权的用户 MUST 使用统一范围解析返回的全量范围执行，不得继续依赖旧的分析全量查询名单。

#### Scenario: 部门负责人执行默认全量模板时自动收口
- **WHEN** 部门负责人执行一条没有显式部门条件的全量业务模板
- **THEN** 系统必须在执行前按该负责人的最终 `scopeSnapshot` 自动收口
- **THEN** 结果不得返回不属于当前范围且未被白名单授权的部门、负责人或团队数据

#### Scenario: 应用超级管理员执行同一模板时允许全量
- **WHEN** 应用超级管理员执行同一条可自动收口模板
- **THEN** 系统必须按其应用超级管理员全量范围执行
- **THEN** 审计记录必须明确本次为应用超级管理员全量范围

#### Scenario: 历史全量分析名单不再作为独立放行条件
- **WHEN** 用户只存在于历史全量分析名单但未完成迁移或未命中新应用超级管理员授权
- **THEN** 系统不得仅凭旧名单放行模板全量执行
- **THEN** 系统必须要求其命中新应用超级管理员授权或按普通范围自动收口
