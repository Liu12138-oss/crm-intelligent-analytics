## ADDED Requirements

### Requirement: 组织范围解析必须支持应用超级管理员全量范围

系统 MUST 在统一组织范围解析中识别应用超级管理员授权。命中应用超级管理员授权时，系统 MUST 输出全量数据范围快照，并明确标记范围来源为应用超级管理员授权。任何消费 `scopeSnapshot` 的模块不得把该全量范围降级为本人范围、登录部门范围或空结果范围。

#### Scenario: 超级管理员范围解析返回全量标记

- **WHEN** 应用超级管理员触发范围解析
- **THEN** 系统必须返回带有应用超级管理员来源标记的范围快照
- **THEN** 范围说明必须明确该用户可查看全公司数据

#### Scenario: 普通用户范围解析不受超级管理员策略影响

- **WHEN** 未命中应用超级管理员授权的普通用户触发范围解析
- **THEN** 系统必须继续按企业微信组织事实、默认负责人范围和数据范围白名单解析
- **THEN** 不得因为系统中存在其他超级管理员授权而扩大该普通用户范围

### Requirement: 消费范围的业务模块必须识别全量范围语义

系统中消费统一范围快照的经营报表、合同审批、分析问数、模板执行、最近查询重跑和导出补查 MUST 正确识别应用超级管理员全量范围语义。全量范围下的空部门或空负责人集合 MUST 表示不按部门或负责人收口，不得被替换成无结果占位条件。

#### Scenario: 经营报表识别全量范围语义

- **WHEN** 应用超级管理员进入经营报表并生成全公司快照
- **THEN** 经营报表必须基于全量范围构造查询上下文
- **THEN** 不得把空 `ownerIds` 转换为只返回空结果的占位条件

#### Scenario: 合同审批识别全量范围语义

- **WHEN** 应用超级管理员查看 CRM 待审批合同列表或详情
- **THEN** 合同审批模块必须基于全量范围放开非本人合同
- **THEN** 审计快照必须记录该放行来自应用超级管理员授权
