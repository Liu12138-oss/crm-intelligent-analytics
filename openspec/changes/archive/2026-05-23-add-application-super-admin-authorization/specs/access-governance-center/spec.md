## MODIFIED Requirements

### Requirement: 系统必须建立“CRM 权限权威源 + 应用能力治理层”的双层权限模型

系统 MUST 将 CRM 组织、部门、负责人、角色和数据范围权限作为常规数据访问的权威来源，并将应用层的菜单、动作、导出资格、合同审核扩展授权、企业微信入口资格和应用超级管理员授权作为独立的能力治理层。常规应用层能力配置 MUST 只能收紧入口和动作，不得扩大 CRM 已定义的数据范围、对象范围或字段范围；应用超级管理员授权是受审计的显式例外，命中后可以获得本系统内全量功能、全量操作和全量数据范围，但仍不得绕过登录认证、安全前置检查、字段白名单、SQL 白名单、危险动作确认和审计约束。

#### Scenario: 常规应用层能力配置不得扩大 CRM 数据范围
- **WHEN** 平台管理员为某个 CRM 角色开通 `analysis.use`、`analysis.export` 或 `wecom.analysis.use`
- **THEN** 系统必须仍按该角色当前 CRM 组织、部门、负责人和字段权限生成最终 `scopeSnapshot`
- **THEN** 不得因为常规应用层开通了动作就扩大其 CRM 数据范围

#### Scenario: 应用超级管理员授权显式扩大系统内权限和数据范围
- **WHEN** 平台管理员为某个 CRM 用户或 CRM 角色开通应用超级管理员授权
- **THEN** 系统必须通过统一权限决策授予其本系统内全部菜单和全部动作
- **THEN** 系统必须通过统一范围解析授予其全量数据范围，并在范围快照中标明授权来源

#### Scenario: 同一用户的 Web 与企业微信共享相同 CRM 数据范围
- **WHEN** 同一 CRM 用户分别从 Web 工作台和企业微信机器人发起分析请求
- **THEN** 系统必须对两个入口注入同一份当前实时 CRM 数据范围或应用超级管理员全量范围
- **THEN** 企业微信灰度或通道资格只可影响入口是否可用，不得改变最终数据范围

## ADDED Requirements

### Requirement: 权限中心必须提供超级管理员授权治理入口

权限中心 MUST 将历史“全量查询授权”升级为“超级管理员授权”治理入口。该入口 MUST 支持按 CRM 用户和 CRM 角色配置应用超级管理员授权，并以中文明确提示该授权等同本系统超级管理员，会开放全部功能、全部操作和全量数据范围。

#### Scenario: 管理员查看超级管理员授权分区

- **WHEN** 具备系统级治理动作的管理员进入权限中心
- **THEN** 页面必须展示“超级管理员授权”分区
- **THEN** 页面必须展示当前已授权用户、已授权角色、更新时间、更新人和变更原因

#### Scenario: 保存超级管理员授权必须填写原因

- **WHEN** 管理员新增、禁用或恢复应用超级管理员授权
- **THEN** 系统必须要求填写中文变更原因
- **THEN** 未填写原因时不得保存授权变更
