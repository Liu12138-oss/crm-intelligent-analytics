## ADDED Requirements

### Requirement: 系统必须在 Web 端提供智能合同审核入口

系统 MUST 在现有 `AppShell` 左侧导航中新增 `智能合同审核` 一级入口，并提供受登录保护的工作台页面和结果详情页面。未登录用户不得直接访问合同审核页面；页面布局 MUST 继续遵循现有顶部栏、左侧导航和主内容区结构，不得引入新的顶层壳子。

#### Scenario: 登录用户进入智能合同审核工作台

- **WHEN** 已登录用户在左侧导航点击 `智能合同审核`
- **THEN** 系统必须进入合同审核工作台页面，并在现有应用壳子中展示上传区和最近审核任务列表，不得在首版首页强制展示任务状态区、统计卡或复杂筛选区

#### Scenario: 未登录用户访问合同审核页面

- **WHEN** 未登录用户直接访问合同审核工作台或结果详情路由
- **THEN** 系统必须将其重定向到登录页，并在登录成功后允许回跳到原目标路由

### Requirement: 系统必须提供单文件上传与任务创建交互

系统 MUST 允许用户在工作台中上传单个合同文件并发起审核任务。首版仅允许 `.docx` 文件进入审核流程；系统 MUST 在创建任务前完成文件类型、文件大小和基础元数据校验，并向前端返回可跟踪的任务标识与初始状态。对于不支持的文件类型、超限文件或校验失败文件，系统 MUST 明确提示原因且不得创建审核任务。对于创建成功的任务，前端 MUST 自动进入该任务的结果详情页，而不是停留在首页等待用户二次点击。

#### Scenario: 合法 docx 文件成功创建审核任务

- **WHEN** 用户上传一个符合大小限制的 `.docx` 文件并提交审核
- **THEN** 系统必须成功创建审核任务、返回任务 ID 与初始状态，并自动进入该任务的结果详情页

#### Scenario: 非法文件被拦截且不创建任务

- **WHEN** 用户上传非 `.docx` 文件、超出大小限制的文件或基础校验失败的文件
- **THEN** 系统必须返回明确的拦截原因，并且任务列表中不得出现对应的新审核任务

### Requirement: 系统必须为审核过程提供阶段反馈、结果视图与返回首页能力

系统 MUST 在结果详情页中展示审核任务的阶段状态，并在任务完成后展示结构化审核结果。结果详情 MUST 至少包含审核摘要、风险统计、一票否决项提示、问题列表、规则依据、任务元信息和产物下载入口；当任务仍在处理中、失败或被阻断时，页面 MUST 显示对应状态与处理说明，而不是仅返回空白页面。结果详情页 MUST 提供明确的“返回首页”入口，供用户回到工作台查看最近审核任务。

#### Scenario: 处理中任务展示阶段状态

- **WHEN** 用户打开一个仍处于 `PARSING`、`REVIEWING` 或 `GENERATING_REPORT` 状态的任务详情
- **THEN** 页面必须显示当前阶段状态、最近更新时间和等待说明，并保持结果区处于受控的处理中状态

#### Scenario: 已完成任务展示结构化审核结果

- **WHEN** 用户打开一个已完成的合同审核任务
- **THEN** 页面必须展示风险统计、问题列表、规则依据和可下载产物入口，而不是只返回一段原始文本

#### Scenario: 用户从结果详情返回首页

- **WHEN** 用户在合同审核结果详情页点击“返回首页”
- **THEN** 系统必须回到合同审核工作台首页，并继续展示上传区与最近审核任务列表

### Requirement: 系统必须以风险优先的方式呈现审核结果

系统 MUST 将审核结果页设计为“风险优先、信息降噪”的结构。首屏 MUST 优先展示总体结论、一票否决项、高中低风险统计和每条风险的核心信息；规则编码、模板标签、技术状态、内部来源和长路径依据等低价值元数据 MUST 默认折叠，不得与核心风险信息处于同一视觉优先级。结果列表 SHOULD 以单列卡片或单列分组形式呈现，避免多标签堆叠和多列分散阅读。

#### Scenario: 商务用户进入结果页时先看到风险结论

- **WHEN** 商务用户首次打开某个已完成审核任务的结果页
- **THEN** 页面首屏必须先展示总体风险结论、风险数量和待处理风险项，而不是优先展示模板标签、编码或来源字段

#### Scenario: 低价值元数据默认折叠

- **WHEN** 结果页展示单条审核问题
- **THEN** 规则编码、命中来源、模板标签和其他技术性字段必须默认折叠到“查看依据”或“展开详情”中，而不是默认全部展开

### Requirement: 系统必须提供最近审核任务与再次查看能力

系统 MUST 为当前用户提供最近审核任务列表，并支持再次进入已完成任务的结果详情。首版首页不要求提供按状态筛选、按合同名称搜索或复杂检索面板；最近审核任务列表 MUST 至少展示合同名称、创建时间和最近结果摘要，不得在列表级别暴露完整合同正文。

#### Scenario: 最近创建或完成的任务回写到首页顶部

- **WHEN** 用户上传合同并创建新任务，或某个任务在详情页完成审核后返回首页
- **THEN** 最近审核任务列表必须将该条审核记录更新到靠前位置，便于用户再次进入

#### Scenario: 用户从最近审核任务再次进入结果详情

- **WHEN** 用户在最近审核任务列表中点击某个已完成任务
- **THEN** 系统必须打开该任务的结果详情页，并显示该次审核对应的结构化结果与产物状态
