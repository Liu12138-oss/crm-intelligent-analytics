## Why

当前 CRM-Agent 已经具备统一登录、Web 工作台、AI 调用、审计留痕和左侧导航框架，但尚未覆盖商务人员最常见的“上传合同并按公司审核条目进行智能审查”场景。随着合同审核资料、规则清单和 skill 参考已经进入仓库，需要把原本独立工具化的设想收敛为当前系统内的一个受控业务模块，尽快形成“上传合同 -> AI 审核 -> 结构化结果 -> 报告下载 -> 审计留痕”的最小闭环。

## What Changes

- 新增 Web 端一级业务入口 `智能合同审核`，并在现有 `AppShell` 左侧导航中提供合同审核工作台与结果详情访问路径。
- 新增合同审核任务主链路，支持 `.docx` 合同上传、文件校验、任务创建、文本提取、规则快照加载、AI 审核编排、问题清单生成和任务状态跟踪。
- 新增合同审核结果交付能力，支持结构化结果展示、风险分级、一票否决项提示、审核报告下载、带批注文档下载，以及最近审核任务的再次查看。
- 新增合同审核专属权限与审计规则，明确上传者可见范围、产物下载授权、失败重试边界，以及上传、审核、下载等关键动作的审计留痕要求。
- 新增应用库存储对象与接口契约，承载合同审核任务、审核问题、审核产物、规则集快照和相关审计引用。

## Capabilities

### New Capabilities

- `contract-review-workbench`: 定义智能合同审核页签、上传入口、最近审核任务列表、自动进入结果详情、返回首页和下载交互要求。
- `contract-review-orchestration`: 定义合同审核任务生命周期、规则快照、文本提取、AI 审核编排、问题生成和产物输出要求。
- `contract-review-auditability`: 定义合同审核模块的任务可见性、产物下载授权、关键动作审计与追溯要求。

### Modified Capabilities

- 无

## Impact

- 影响前端 `frontend/src/router`、`frontend/src/layouts/AppShell.vue`、新增合同审核页面与状态管理。
- 影响后端模块边界，需新增独立 `contract-review` 模块，并与 `auth`、`audit`、`shared/config`、应用库存储层协作。
- 影响应用库存储模型，需补充合同审核任务、问题、产物和规则快照等对象。
- 影响 API 契约、快速验证文档和 UI 原型，需要在后续设计和实现阶段同步补齐 `openapi`、`.pen` 原型与验收场景。
