## ADDED Requirements

### Requirement: 系统必须基于同一份分析报告按渠道输出结果
系统 MUST 将统一分析报告同时作为 Web 工作台和企业微信结果输出的来源。Web 工作台 MUST 展示完整报告结构；企业微信 MUST 基于同一报告裁剪为适合会话展示的摘要块、关键指标块和必要提示，但不得改变核心结论和数据口径。

#### Scenario: Web 展示完整分析报告
- **WHEN** 用户在 Web 工作台成功获得一条分析结果
- **THEN** 页面必须展示完整的报告摘要、关键指标、图表、表格、关键发现和过程说明

#### Scenario: 企业微信展示裁剪后的同源报告
- **WHEN** 用户在企业微信渠道获得同一条分析结果
- **THEN** 系统必须返回基于同一报告生成的摘要化结果，并在展示受限时提示用户到 Web 查看完整内容

### Requirement: 工作台必须以报告和过程反馈为主视觉中心
Web 工作台 MUST 在一级分区不变的前提下，以查询输入、分析报告和过程反馈作为主视觉中心。页面 MUST 优先突出结果摘要、关键指标、主图表、关键表格和 AI 处理过程，不得让权限卡、状态卡或其他辅助模块持续抢占页面焦点。

#### Scenario: 有结果时报告区优先呈现
- **WHEN** 用户成功提交问题并获得分析结果
- **THEN** 页面必须让报告区和过程反馈区优先进入视线范围，便于用户快速看到核心结论

#### Scenario: 无结果时输入区优先呈现
- **WHEN** 用户尚未提交问题或当前无可展示结果
- **THEN** 页面必须让输入区成为最主要操作入口，并保留必要的辅助信息摘要

### Requirement: 工作台必须提供完整的用户反馈状态
工作台 MUST 对分析链路中的关键状态提供明确反馈，至少覆盖加载中、补问中、成功、失败、空数据、无权限、超时和可重试提示。系统 MUST 让用户在任意状态下都能理解“当前发生了什么”和“下一步该做什么”，不得仅依赖控制台日志或静态占位。

#### Scenario: 查询执行中展示实时反馈
- **WHEN** 用户发起一条新的分析请求且后端仍在处理中
- **THEN** 页面必须展示处理中状态、阶段进度或骨架反馈，并对提交按钮和重复操作给出禁用或节流提示

#### Scenario: 查询失败时提供明确处理建议
- **WHEN** 分析请求因超范围、校验失败、无权限、超时或系统异常而失败
- **THEN** 页面必须展示中文失败原因、必要的重试入口或调整建议，而不是只显示模糊错误

### Requirement: 工作台必须展示可回看的 AI 过程输出
工作台 MUST 以用户可读方式展示 AI 过程输出，至少覆盖问题识别、补问确认、查询执行、结果整理、校验结论和完成状态。查询完成后，过程输出 MUST 继续保留以供回看，并且不能退化为只显示“处理中”与“完成”两条简单消息。

#### Scenario: 查询执行时逐步追加过程输出
- **WHEN** 一条分析请求从提交进入执行流程
- **THEN** 页面必须按时间顺序追加展示问题识别、查询执行和结果整理等过程信息

#### Scenario: 查询完成后保留完整过程记录
- **WHEN** 用户重新查看刚完成的分析结果
- **THEN** 页面必须仍然可以查看该次分析的完整过程输出和校验说明

### Requirement: 查询资产必须靠近输入区并支持快速复用
工作台 MUST 将常用查询和最近查询放在输入区附近，并允许用户直接回填、复用或重新执行问题。系统 MUST 让高频查询资产服务于“快速发问和快速改写”场景，而不得把这些入口放在远离输入区的位置。

#### Scenario: 用户进入页面即可看到查询资产
- **WHEN** 用户首次进入 AI 智能分析工作台
- **THEN** 页面必须在输入区附近展示常用查询和最近查询入口

#### Scenario: 用户可直接复用查询资产
- **WHEN** 用户点击任一常用查询模板或最近查询记录
- **THEN** 系统必须允许其直接执行或回填问题内容，并立即进入新的分析流程

### Requirement: 工作台必须补齐关键交互态与轻量动效
工作台 MUST 为关键操作提供清晰的鼠标悬浮、点击、聚焦、禁用和完成态反馈，并通过轻量动效强化用户对提交、加载、结果切换和提示消息的感知。动效 MUST 服务于操作反馈和信息理解，不得造成拖慢或干扰。

#### Scenario: 可操作元素在悬浮和禁用时有明确反馈
- **WHEN** 用户将鼠标悬浮到按钮、模板项、最近查询项或导出入口上，或遇到按钮禁用状态
- **THEN** 页面必须给予可感知的悬浮态或禁用态反馈，明确当前元素是否可操作

#### Scenario: 结果与提示切换时具有轻量动效
- **WHEN** 页面从提交状态切换到处理中、从处理中切换到结果展示，或弹出成功失败提示
- **THEN** 页面必须通过轻量、不中断操作的动效帮助用户感知状态变化
