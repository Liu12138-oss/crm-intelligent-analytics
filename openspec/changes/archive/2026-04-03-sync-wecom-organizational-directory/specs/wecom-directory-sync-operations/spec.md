## ADDED Requirements

### Requirement: 系统必须提供生产环境一键同步入口
系统必须提供生产环境可直接执行的一键同步入口，用于在发现企业微信组织架构未同步时立即触发目录同步。该入口可以是运维脚本、受控内部 API，或两者组合，但必须能够触发同一套同步服务，而不是复制一套独立逻辑。

#### Scenario: 运维手动触发一键同步
- **WHEN** 生产环境确认企业微信目录未同步或机器人持续提示未绑定 CRM 身份
- **THEN** 运维必须能够通过一键同步入口立即触发目录同步，而不需要手工逐个调用外部分页接口

#### Scenario: 一键同步入口与定时任务复用同一服务
- **WHEN** 运维脚本或内部 API 触发目录同步
- **THEN** 系统必须复用与定时任务相同的同步服务和游标逻辑，而不是维护两套不同实现

### Requirement: 一键同步执行前必须进行基础校验
系统在执行生产环境一键同步前，必须至少校验外部 EMM API 配置、应用凭证、应用存储库可用性以及当前是否已有同步任务在运行。任一校验失败时，系统必须直接阻止本次执行并返回明确原因。

#### Scenario: 配置缺失时阻止执行
- **WHEN** 生产环境一键同步触发时缺少 EMM API 基础地址、应用凭证或必要运行时配置
- **THEN** 系统必须阻止本次执行，并返回明确的配置缺失结果

#### Scenario: 同步任务已在执行中
- **WHEN** 生产环境已有同类目录同步任务正在执行
- **THEN** 系统必须拒绝重复执行，并提示当前已有同步任务在运行

### Requirement: 一键同步完成后必须输出执行摘要
系统在生产环境一键同步完成后，必须返回可供运维立即判断结果的摘要，至少包括触发时间、资源类型、成功条数、失败条数、最新游标和最近失败原因（若存在）。

#### Scenario: 一键同步成功完成
- **WHEN** 运维触发的一键同步成功完成
- **THEN** 系统必须返回成功摘要，并展示本次推进后的游标与同步条数

#### Scenario: 一键同步部分失败
- **WHEN** 一键同步只完成了部分资源或某一分页失败
- **THEN** 系统必须返回部分失败摘要，并说明失败资源和失败原因

### Requirement: 一键同步执行必须形成运维审计记录
系统必须为生产环境一键同步记录独立的运维审计事件，至少包括触发人、触发时间、触发方式、资源范围和执行结果。

#### Scenario: 脚本触发同步形成审计
- **WHEN** 运维通过生产脚本触发企业微信目录同步
- **THEN** 系统必须创建一条运维审计事件，并记录触发方式为脚本

#### Scenario: 内部 API 触发同步形成审计
- **WHEN** 运维或内部后台通过受控 API 触发企业微信目录同步
- **THEN** 系统必须创建一条运维审计事件，并记录触发方式为内部入口
