## ADDED Requirements

### Requirement: 企业微信机器人必须识别受控新建客户意图
系统 MUST 将“新增客户”“新建客户”“帮我建客户”等企业微信消息识别为受控新建客户意图，并进入专用的客户创建链路，而不是继续走问数分析或统一写入拒绝分支。除新建客户外的其它写入类请求仍 MUST 继续按现有规则拒绝。

#### Scenario: 命中新建客户意图后进入客户创建链路
- **WHEN** 已完成企业微信身份映射的用户在机器人会话中发送“帮我新建一个客户”
- **THEN** 系统进入受控新建客户状态并返回客户字段采集提示
- **THEN** 系统不得把该消息当作分析问数或通用写入拒绝处理

#### Scenario: 非白名单写入请求继续拒绝
- **WHEN** 用户在机器人会话中发送“帮我删除这个客户”或“帮我创建提醒”
- **THEN** 系统继续返回当前不支持该类写入的明确提示
- **THEN** 系统不得误进入客户创建状态

### Requirement: 新建客户链路必须补齐必填字段并在执行前展示摘要
系统 MUST 支持在企业微信会话中分步补齐“名称、电话、IT决策权所在地、统一社会信用代码”等客户创建必填字段。系统 MUST 在字段不完整时只追问缺失项，并在字段齐全后生成结构化摘要，等待用户确认、修改或取消。

#### Scenario: 初始消息缺少部分必填字段
- **WHEN** 用户发送“新增客户，名称是苏州某科技，电话 13800000000”
- **THEN** 系统保留已识别的名称和电话
- **THEN** 系统继续追问缺失的 IT决策权所在地 与统一社会信用代码

#### Scenario: 字段齐全后进入摘要确认
- **WHEN** 当前客户草稿已经补齐所有必填字段
- **THEN** 系统返回包含名称、电话、IT决策权所在地、统一社会信用代码、默认负责人和默认所属部门的摘要
- **THEN** 系统提示用户回复“确认”“修改”或“取消”

### Requirement: 新建客户执行必须支持取消、失败保留和重试
系统 MUST 在用户确认后才调用受控客户创建服务。创建成功时 MUST 返回标准化结果与关键标识；取消时 MUST 终止本次创建；失败时 MUST 保留当前草稿和失败原因，并允许用户修改字段后再次确认或直接重试。

#### Scenario: 用户确认后创建成功
- **WHEN** 用户已确认客户摘要且 CRM 官方创建接口返回成功
- **THEN** 系统返回客户 ID、客户名称、负责人、所属部门、创建时间和成功提示
- **THEN** 系统记录客户创建成功审计事件并清理当前待创建状态

#### Scenario: 用户取消创建
- **WHEN** 用户在确认前回复“取消”
- **THEN** 系统结束当前客户创建流程并返回已取消提示
- **THEN** 系统不得向 CRM 发起创建请求

#### Scenario: 创建失败后保留草稿
- **WHEN** 用户已确认客户摘要但 CRM 官方创建接口返回失败
- **THEN** 系统将当前客户草稿标记为失败并保留已采集字段与失败原因
- **THEN** 系统提示用户可以回复“修改”继续编辑，或回复“重试”再次发起创建
