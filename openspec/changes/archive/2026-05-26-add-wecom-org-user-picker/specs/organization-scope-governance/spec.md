## ADDED Requirements

### Requirement: 组织事实层必须提供治理页面可复用的组织对象选项视图

系统 MUST 基于企业微信组织事实层、CRM 身份映射和 CRM 部门关系提供治理页面可复用的组织对象选项视图。该视图 MUST 至少包含企业微信部门层级、部门到 CRM 部门的映射状态、成员所属部门、成员主部门、企业微信 userid、成员姓名、CRM 用户映射状态、可选性说明和最近同步时间。该视图只用于治理页面选择和诊断，不得作为新的权限权威源。

#### Scenario: 组织对象选项包含成员部门归属

- **WHEN** 权限中心请求组织对象选项视图
- **THEN** 系统必须返回每个企业微信成员所属的部门 ID 集合
- **THEN** 前端必须能够基于该关系完成部门下人员全选和部门下人员展示

#### Scenario: 组织对象选项包含部门映射状态

- **WHEN** 权限中心请求组织对象选项视图
- **THEN** 系统必须返回每个企业微信部门是否已映射到有效 CRM 部门
- **THEN** 前端必须能够基于该状态禁用不能保存为 CRM 部门的组织节点

#### Scenario: 组织对象选项标明最近同步时间

- **WHEN** 权限中心展示企业微信组织架构选择器
- **THEN** 系统必须提供可展示的最近同步时间或等价同步状态
- **THEN** 当组织事实为空或过旧时，页面必须能给出中文提示

### Requirement: 组织对象选择不得扩大正式数据范围

通过组织对象选项视图完成的人员或部门选择 MUST 只影响对应治理配置中已有的授权主体或查询主体，不得直接扩大用户正式数据范围。正式数据范围仍 MUST 通过 CRM 权限、企业微信组织范围解析、数据范围白名单和应用超级管理员授权统一计算。

#### Scenario: 选择对象不直接改变权限范围

- **WHEN** 管理员仅在组织架构选择器中选中某个企业微信部门或人员但尚未保存任何治理配置
- **THEN** 系统不得改变这些人员或部门相关用户的权限或数据范围
- **THEN** 后续正式问数仍必须按当前已保存配置计算范围

#### Scenario: 保存后仍由既有范围解析生效

- **WHEN** 管理员通过组织架构选择器保存数据范围白名单或超级管理员授权
- **THEN** 正式权限和数据范围必须继续由既有权限决策与组织范围解析服务计算
- **THEN** 组织架构选择器不得绕过字段白名单、读写边界或审计留痕
