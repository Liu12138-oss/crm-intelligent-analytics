## ADDED Requirements

### Requirement: 企业微信灰度用户名单和部门名单必须支持按组织架构选择

企业微信灰度策略治理界面中的白名单用户、停用用户和白名单部门 MUST 支持通过企业微信组织架构选择器选择。系统 MUST 只允许将已映射到有效 CRM 用户的企业微信成员保存到 `allowUserIds` 或 `denyUserIds`，只允许将已映射到有效 CRM 部门的企业微信部门保存到 `allowDepartmentIds`，不得将企业微信 userid 或企业微信部门 ID 错存为 CRM 字段。

#### Scenario: 管理员按部门批量加入灰度白名单用户

- **WHEN** 管理员在企业微信灰度分区选择某个企业微信部门并确认加入白名单用户
- **THEN** 系统必须把该部门及子部门下已映射 CRM 的成员加入 `allowUserIds`
- **THEN** 未映射成员不得被加入白名单用户

#### Scenario: 管理员按搜索结果加入停用用户

- **WHEN** 管理员在企业微信灰度分区搜索人员并选择一名已映射 CRM 的成员加入停用用户
- **THEN** 系统必须把该成员对应的 CRM 用户 ID 加入 `denyUserIds`
- **THEN** 搜索关键词必须在选择后清空

#### Scenario: 管理员按组织架构选择灰度白名单部门

- **WHEN** 管理员在企业微信灰度分区选择一个已映射 CRM 的企业微信部门加入白名单部门
- **THEN** 系统必须把该部门对应的 CRM 部门 ID 加入 `allowDepartmentIds`
- **THEN** 系统不得把企业微信部门 ID 直接保存为 CRM 部门 ID

### Requirement: 企业微信灰度选择必须清晰展示未映射人员和部门

企业微信灰度选择器 MUST 在组织树和搜索结果中展示未映射 CRM 的企业微信成员和部门，但在保存 CRM 用户名单或 CRM 部门名单的场景中必须禁用这些对象，并给出中文不可选原因。系统不得因为成员或部门存在于企业微信组织树中就默认授予、停用或放量其 CRM 入口。

#### Scenario: 未映射成员不能加入灰度名单

- **WHEN** 管理员在灰度选择器中看到未绑定 CRM 的企业微信成员
- **THEN** 该成员必须不可勾选
- **THEN** 页面必须提示该成员未绑定 CRM，不能保存为灰度人员

#### Scenario: 未映射部门不能加入灰度部门名单

- **WHEN** 管理员在灰度选择器中看到未绑定 CRM 部门的企业微信部门
- **THEN** 该部门必须不可勾选
- **THEN** 页面必须提示该部门未绑定 CRM 部门，不能保存为灰度部门
