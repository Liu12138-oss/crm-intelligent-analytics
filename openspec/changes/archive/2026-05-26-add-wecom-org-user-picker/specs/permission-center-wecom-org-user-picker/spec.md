## ADDED Requirements

### Requirement: 权限中心必须提供统一的企业微信组织架构人员和部门选择器

系统 MUST 在权限中心提供统一的企业微信组织架构人员和部门选择器，用于替代人员类和部门类平铺下拉选择。选择器 MUST 支持以弹窗或抽屉形式展示企业微信部门树、部门下人员、搜索结果和已选对象列表，并支持人员选择、部门选择、人员与部门混合选择、单选与多选模式。选择器展示的用户可见内容 MUST 使用中文业务表达，不得直接暴露内部字段名、枚举值或状态码。

#### Scenario: 管理员打开统一组织架构选择器

- **WHEN** 管理员在权限中心任一人员或部门选择入口点击选择控件
- **THEN** 系统必须打开统一组织架构选择器
- **THEN** 选择器必须展示企业微信组织架构树、人员或部门列表、搜索框和已选对象区域

#### Scenario: 单选模式只允许保留一个对象

- **WHEN** 选择器处于单选模式，且管理员先后选择两个对象
- **THEN** 系统必须仅保留最后选择的对象
- **THEN** 组件提交给调用方的值必须是单个对象标识

#### Scenario: 多选模式支持保留多个对象

- **WHEN** 选择器处于多选模式，且管理员选择多个对象
- **THEN** 系统必须保留全部已选可选对象
- **THEN** 组件提交给调用方的值必须是对象标识数组

#### Scenario: 部门模式选择部门本身

- **WHEN** 选择器处于部门选择模式，且管理员勾选一个企业微信部门
- **THEN** 系统必须选择该部门本身
- **THEN** 系统不得把该操作解释为选择该部门下所有人员

### Requirement: 选择器必须支持按企业微信部门批量选择人员

选择器 MUST 支持在人员选择模式下由管理员通过企业微信组织架构树选择某个部门，并批量选中该部门及其子部门下所有当前场景可选人员。批量选择不得绕过映射状态、禁用状态或业务场景限制；不可选人员 MUST 保持未选中，并以中文说明不可选原因。

#### Scenario: 管理员全选某个部门下人员

- **WHEN** 管理员在人员选择模式中勾选一个企业微信部门
- **THEN** 系统必须选中该部门及子部门下所有当前场景可选人员
- **THEN** 已选对象区域必须展示新增的人员数量和人员名称

#### Scenario: 部门中存在不可选人员

- **WHEN** 管理员全选某个部门，且该部门下存在未绑定 CRM 或映射异常的不可选人员
- **THEN** 系统不得将不可选人员加入已选列表
- **THEN** 系统必须给出中文反馈，说明部分人员因不可保存未加入

### Requirement: 选择器必须支持搜索人员和部门并在选择后清空搜索词

选择器 MUST 支持按姓名、企业微信 userid、CRM 用户名、部门名、企业微信部门 ID、CRM 部门名或可用说明信息搜索。管理员从搜索结果中完成选择后，系统 MUST 自动清空搜索关键词；清空搜索关键词不得清空已选对象。

#### Scenario: 搜索后选择对象自动清空关键词

- **WHEN** 管理员输入关键词搜索人员或部门并选择其中一个结果
- **THEN** 搜索框中的关键词必须被清空
- **THEN** 已选对象必须继续保留在已选区域

#### Scenario: 搜索无结果时展示中文空状态

- **WHEN** 管理员输入关键词后没有匹配人员或部门
- **THEN** 系统必须展示中文空状态
- **THEN** 空状态不得直接展示内部搜索字段名或接口状态码

### Requirement: 选择器必须按业务场景输出正确对象标识

选择器 MUST 支持按调用场景输出 CRM 用户 ID、企业微信 userid、CRM 部门 ID 或企业微信部门 ID。当输出类型为 CRM 用户 ID 时，系统 MUST 禁止选择未映射到有效 CRM 用户的企业微信成员；当输出类型为 CRM 部门 ID 时，系统 MUST 禁止选择未映射到有效 CRM 部门的企业微信部门；当输出类型为企业微信 userid 或企业微信部门 ID 时，系统 MAY 允许选择未映射对象，但必须展示其映射状态。

#### Scenario: CRM 用户 ID 场景禁用未映射成员

- **WHEN** 调用方要求输出 CRM 用户 ID，且组织树中存在未映射 CRM 的企业微信成员
- **THEN** 该成员必须处于不可选状态
- **THEN** 不可选原因必须说明“未绑定 CRM”或等价中文含义

#### Scenario: 企业微信 userid 场景可选择未映射成员

- **WHEN** 调用方要求输出企业微信 userid，且管理员选择一个未映射 CRM 的企业微信成员
- **THEN** 系统必须允许选择该成员的企业微信 userid
- **THEN** 选择器必须展示该成员当前未完成 CRM 映射

#### Scenario: CRM 部门 ID 场景禁用未映射部门

- **WHEN** 调用方要求输出 CRM 部门 ID，且组织树中存在未映射 CRM 的企业微信部门
- **THEN** 该部门必须处于不可选状态
- **THEN** 不可选原因必须说明“未绑定 CRM 部门”或等价中文含义

### Requirement: 组织对象选项接口必须复用企业微信组织事实和 CRM 映射

系统 MUST 提供治理页面可用的组织对象选项接口，返回企业微信部门、企业微信成员、成员所属部门、CRM 用户映射状态、CRM 部门映射状态和最近同步时间。该接口 MUST 复用现有企业微信组织事实层、CRM 身份映射和 CRM 部门关系，不得新增独立人员、部门或权限模型。

#### Scenario: 获取组织对象选项

- **WHEN** 具备权限中心治理权限的管理员打开组织架构选择器
- **THEN** 系统必须返回企业微信部门树、部门映射信息和成员映射信息
- **THEN** 返回数据必须包含足以支撑部门选择、部门下人员全选、人员搜索和部门搜索的关系

#### Scenario: 未授权用户访问组织对象选项被拒绝

- **WHEN** 不具备权限中心治理权限的用户调用组织对象选项接口
- **THEN** 系统必须拒绝访问
- **THEN** 该用户不得读取企业微信组织人员和部门清单

### Requirement: 后续业务选择人员或部门必须默认复用统一组织架构选择器

项目中后续新增或修改任意页面时，只要涉及选择人员或部门，系统 MUST 默认复用统一企业微信组织架构选择器或其项目级封装。若某业务明确不是企业微信/CRM 组织对象，或统一选择器无法表达该业务，设计文档或 OpenSpec MUST 说明不复用原因。

#### Scenario: 新增页面需要选择人员

- **WHEN** 后续新增业务页面需要选择 CRM 用户、企业微信成员或同类人员对象
- **THEN** 页面必须默认复用统一组织架构选择器
- **THEN** 不得另写平铺人员下拉作为首选方案

#### Scenario: 新增页面需要选择部门

- **WHEN** 后续新增业务页面需要选择 CRM 部门、企业微信部门或同类部门对象
- **THEN** 页面必须默认复用统一组织架构选择器
- **THEN** 不得另写平铺部门下拉作为首选方案
