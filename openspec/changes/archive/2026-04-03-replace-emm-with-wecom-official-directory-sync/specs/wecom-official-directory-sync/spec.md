## ADDED Requirements

### Requirement: 系统必须使用企业微信官方 API 获取部门列表
系统必须使用企业微信官方 `access_token` 调用部门列表接口，获取当前授权范围内的部门树，并将其写入应用侧部门镜像表。

#### Scenario: 同步联软科技集团部门树
- **WHEN** 生产环境执行企业微信目录同步
- **THEN** 系统必须只同步“联软科技集团”授权范围内的部门树，而不是整个 ROOT 全量组织树

### Requirement: 系统必须按部门拉取成员详情
系统必须基于目标部门列表逐个调用企业微信官方部门成员详情接口，获取成员列表，并将成员的 `userid`、姓名、手机号、邮箱、主部门等字段写入应用侧用户镜像表。

#### Scenario: 部门成员详情被写入镜像
- **WHEN** 系统调用部门成员详情接口成功返回成员列表
- **THEN** 系统必须将返回成员以企业微信原生 `userid` 为主键写入镜像表

### Requirement: 系统必须支持用单个成员详情接口补齐字段
当部门成员详情接口返回字段不足以支撑机器人身份识别时，系统必须支持对单个成员继续调用成员详情接口做字段补齐。

#### Scenario: 手机号或邮箱缺失时补齐成员详情
- **WHEN** 部门成员详情接口未返回手机号或邮箱等关键字段
- **THEN** 系统必须对该成员追加详情查询，尝试补齐识别所需字段
