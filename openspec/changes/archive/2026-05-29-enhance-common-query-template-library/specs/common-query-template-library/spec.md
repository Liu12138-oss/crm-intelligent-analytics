## ADDED Requirements

### Requirement: 工作台必须提供左侧常驻常用查询模板库

系统 MUST 在 Web 智能分析工作台桌面端提供左侧常驻常用查询模板库，并将最近查询继续保留在抽屉中。左侧模板库 MUST 默认展示“我的模板”，并允许切换到“其它模板”。左侧模板库 MUST 支持默认宽度、点击展开、点击收缩和拖拽调整宽度，且不得导致右侧查询输入、结果表格、图表或操作按钮重叠。

#### Scenario: 用户进入分析页默认看到我的模板
- **WHEN** 已授权用户进入 Web 智能分析工作台
- **THEN** 页面左侧必须展示常用查询模板库
- **THEN** 模板库必须默认选中“我的模板”
- **THEN** 最近查询不得出现在左侧常驻模板库中

#### Scenario: 用户调整左侧模板库宽度
- **WHEN** 用户点击展开、点击收缩或拖拽左侧模板库宽度
- **THEN** 系统必须在最小宽度和最大宽度范围内更新左侧区域
- **THEN** 右侧查询输入、结果表格、图表和按钮文字不得发生重叠

#### Scenario: 窄屏访问分析页
- **WHEN** 屏幕宽度不足以稳定展示左侧常驻模板库
- **THEN** 系统可以将常用查询折叠为按钮或抽屉
- **THEN** 桌面端默认常驻展示规则不得被移除

### Requirement: 常用查询必须区分我的模板和其它模板

系统 MUST 将工作台可见模板分为“我的模板”和“其它模板”。“我的模板” MUST 包含当前用户保存、手动创建或从其它模板复制得到的个人副本；“其它模板” MUST 包含当前用户可见但不属于当前用户的启用模板。普通业务用户保存的模板默认全员可见，但任何用户执行模板时 MUST 按执行人当前实时权限重新收口。

#### Scenario: 用户查看我的模板
- **WHEN** 用户进入“我的模板”标签页
- **THEN** 系统必须返回当前用户拥有的模板副本和自己创建的模板
- **THEN** 不得混入仅可见但未保存到我的模板的其它用户模板

#### Scenario: 用户查看其它模板
- **WHEN** 用户进入“其它模板”标签页
- **THEN** 系统必须返回当前用户可见且不属于当前用户的启用模板
- **THEN** 系统不得返回停用模板或当前用户无权查看的模板

#### Scenario: 普通用户保存模板后其它用户可见
- **WHEN** 普通业务用户成功保存一条模板
- **THEN** 其它已授权用户必须能在“其它模板”中搜索到该模板
- **THEN** 其它用户执行该模板时必须按执行人当前权限范围重新执行

### Requirement: 添加到我的模板必须生成个人副本

系统 MUST 在用户点击“添加到我的模板”时复制模板定义快照并生成一条当前用户自己的模板记录。个人副本 MUST 保存来源模板 ID、来源模板名称、来源 SQL 版本和复制时间。个人副本创建后 MUST 作为独立模板管理，来源模板后续变更、停用或删除不得静默影响已有个人副本。

#### Scenario: 用户从其它模板创建个人副本
- **WHEN** 用户在“其它模板”列表点击“添加到我的模板”
- **THEN** 系统必须创建新的模板记录并归属到当前用户
- **THEN** 新模板必须记录来源模板 ID、来源模板名称、来源 SQL 版本和复制时间
- **THEN** 新模板必须出现在当前用户“我的模板”中

#### Scenario: 来源模板后续发生变更
- **WHEN** 来源模板在用户复制后被修改、停用或删除
- **THEN** 已经复制到我的模板的个人副本不得被静默覆盖、停用或删除
- **THEN** 治理后台和审计必须仍能追溯该副本来自哪条模板

#### Scenario: 添加到我的模板不计入执行点击
- **WHEN** 用户仅点击“添加到我的模板”但未执行模板
- **THEN** 系统不得把该动作计入模板历史执行次数
- **THEN** 系统必须记录模板复制审计事件

### Requirement: 自由问数结果必须可受控保存为模板

系统 MUST 允许用户将成功自由问数结果保存为查询模板。保存模板时，前端只可提交模板名称、说明、标签、可见性和展示配置等元数据；后端 MUST 根据 `queryId` 读取已成功执行的受控 SQL 或执行计划。若无法得到可复现 SQL，且无法从执行计划安全编译参数化 SQL，系统 MUST 拒绝保存并返回中文原因。

#### Scenario: 成功自由问数保存为模板
- **WHEN** 用户对已成功返回的自由问数结果点击“保存为模板”
- **AND** 填写模板名称和标签
- **THEN** 系统必须基于该 `queryId` 的受控执行快照创建模板
- **THEN** 模板必须出现在保存人的“我的模板”中
- **THEN** 模板默认可被其它已授权用户在“其它模板”中搜索到

#### Scenario: 无法复现的自由问数不得保存
- **WHEN** 用户尝试保存一条没有可复现 SQL 且执行计划无法安全编译的分析结果
- **THEN** 系统必须拒绝保存
- **THEN** 系统必须提示当前查询暂不支持保存为模板
- **THEN** 系统不得创建不可执行的半成品模板

#### Scenario: 保存后的模板按执行人权限重跑
- **WHEN** 另一个用户执行由自由问数保存生成的模板
- **THEN** 系统必须按该执行人的当前实时权限重新注入范围
- **THEN** 系统不得复用保存人当时的授权快照放行数据

### Requirement: 模板必须支持标签分类

系统 MUST 为模板提供一组可选标签。标签 MUST 使用中文业务表达。保存、编辑和治理模板时，标签 MUST 支持选择已有标签，也 MUST 支持自由输入新标签。AI MAY 推荐标签，但推荐结果 MUST 经用户确认后才写入模板。

#### Scenario: 用户保存模板时选择已有标签
- **WHEN** 用户打开保存模板弹窗
- **THEN** 系统必须提供已有标签可选
- **THEN** 用户选择的多个标签必须保存到模板记录中

#### Scenario: 用户保存模板时输入新标签
- **WHEN** 用户输入当前不存在的新标签并保存模板
- **THEN** 系统必须将该标签写入模板标签集合
- **THEN** 后续模板搜索和筛选必须能使用该新标签

#### Scenario: AI 推荐标签需要确认
- **WHEN** AI 根据问题、SQL 或结果摘要推荐标签
- **THEN** 页面必须将其作为推荐项展示
- **THEN** 只有用户确认选择后，推荐标签才可以写入模板记录

### Requirement: 模板列表必须支持搜索筛选和历史点击排序

系统 MUST 允许“我的模板”和“其它模板”分别按关键词、标签和创建人筛选。关键词 MUST 覆盖模板名称、说明和默认问题。筛选输入 MUST 支持按 Enter 执行查询。模板默认排序 MUST 按历史累计点击或执行次数倒序，点击次数相同后再按最近使用时间和更新时间排序。

#### Scenario: 用户按关键词搜索模板
- **WHEN** 用户在模板库输入关键词并点击查询或按 Enter
- **THEN** 系统必须按模板名称、说明和默认问题执行模糊搜索
- **THEN** 搜索行为不得清空当前分析结果

#### Scenario: 用户按标签筛选模板
- **WHEN** 用户选择一个或多个标签筛选模板
- **THEN** 系统必须只返回匹配标签条件的模板
- **THEN** 空结果必须展示中文空状态和当前筛选条件摘要

#### Scenario: 模板按历史点击排序
- **WHEN** 用户打开我的模板或其它模板列表
- **THEN** 系统必须优先展示历史累计点击或执行次数更高的模板
- **THEN** 模板被正式执行后必须更新历史使用统计

### Requirement: 模板库必须支持大列表稳定交互

系统 MUST 支持几百到上千条模板的列表展示。后端模板列表 MUST 支持分页或等效游标参数；前端 MUST 使用虚拟滚动或等效方案，并保持单条模板卡片高度稳定。加载、失败、空列表、无权限和操作成功 / 失败状态 MUST 使用中文反馈。

#### Scenario: 模板数量达到 1000 条
- **WHEN** 用户打开包含 1000 条模板的列表
- **THEN** 页面滚动和筛选必须保持可操作
- **THEN** 模板卡片不得因为动态内容导致滚动定位明显跳动

#### Scenario: 模板列表加载失败
- **WHEN** 模板列表接口失败或超时
- **THEN** 页面必须展示中文失败原因和重试入口
- **THEN** 不得只在控制台输出错误

#### Scenario: 用户复制模板成功
- **WHEN** 用户点击“添加到我的模板”且复制成功
- **THEN** 页面必须展示 3 秒自动消失的中文成功提示
- **THEN** 我的模板列表必须能刷新并显示新副本
