## ADDED Requirements

### Requirement: 自由问数结果保存模板必须复用受控执行事实

系统 MUST 支持将成功自由问数结果保存为查询模板。保存时，系统 MUST 通过 `queryId` 读取已验证的受控执行事实，优先复用实际执行的受控 SQL、参数、范围治理结论、结果展示配置和一致性标识；若只有执行计划，系统 MUST 在服务端将其编译为参数化 SQL 模板并重新校验。系统 MUST NOT 接受前端直接提交自由拼接 SQL 作为自由问数保存模板的执行来源。

#### Scenario: 基于 queryId 保存成功分析结果
- **WHEN** 用户对已成功返回的自由问数结果发起保存模板
- **THEN** 系统必须根据 `queryId` 读取该结果的执行快照
- **THEN** 系统必须基于服务端受控 SQL 或可编译执行计划生成模板
- **THEN** 系统不得使用前端提交的 SQL 作为可信执行来源

#### Scenario: 执行计划编译后重新校验
- **WHEN** 分析结果没有直接保存受控 SQL，但存在可复现执行计划
- **THEN** 系统必须在服务端将执行计划编译为参数化 SQL 模板
- **THEN** 编译后的模板必须重新通过只读、白名单、AST、安全范围治理和预检校验

### Requirement: 不可复现分析结果不得保存为模板

当自由问数结果缺少可复现 SQL，且执行计划无法安全编译为参数化 SQL 模板时，系统 MUST 拒绝保存模板，并返回面向业务用户的中文原因。系统 MUST 记录保存失败审计，不得创建不可执行、不可校验或无法追溯执行来源的模板记录。

#### Scenario: 缺少可复现执行事实时拒绝保存
- **WHEN** 用户尝试保存一条缺少受控 SQL 且执行计划不可编译的分析结果
- **THEN** 系统必须拒绝保存模板
- **THEN** 系统必须提示当前查询暂不支持保存为模板
- **THEN** 系统不得创建模板记录

#### Scenario: 保存失败写入审计
- **WHEN** 自由问数保存模板因执行事实不足或安全校验失败被拒绝
- **THEN** 系统必须写入保存失败审计
- **THEN** 审计必须包含 `queryId`、操作者、失败阶段和中文失败原因

### Requirement: 保存生成的模板执行时必须重新装载当前权限

由自由问数保存生成的模板 MUST 只保存查询口径、参数结构和展示配置，不得保存为后续执行放行依据的历史授权快照。任何用户执行该模板时，系统 MUST 基于执行人当前实时权限、当前模板定义和当前范围治理结论重新执行。

#### Scenario: 其它用户执行保存模板
- **WHEN** 用户 A 保存模板后，用户 B 执行该模板
- **THEN** 系统必须按用户 B 当前权限范围重新注入组织、部门和负责人范围
- **THEN** 系统不得复用用户 A 保存模板时的权限快照

#### Scenario: 保存人权限变化后再次执行
- **WHEN** 模板保存人的权限在保存后发生变化
- **THEN** 保存人再次执行该模板时必须使用最新权限范围
- **THEN** 系统不得沿用保存时的历史权限放行
