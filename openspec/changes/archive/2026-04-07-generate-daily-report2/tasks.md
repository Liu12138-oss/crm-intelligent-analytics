## 1. 日报自由口述拆解

- [ ] 1.1 扩展 `WecomDailyReportIntakeService` 的拆句、关键词识别和片段归桶逻辑，把一段口述稳定拆成五类日报字段。
- [ ] 1.2 补齐公司名、项目名、引号内容和后缀规则，避免公司被误识别成项目，或带“项目”后缀的名称被误归到客户。
- [ ] 1.3 生成逐条编号的“客户/商机跟进”确认摘要，并输出缺失字段清单供后续补问使用。

## 2. 主数据确认与补问流程

- [ ] 2.1 在企业微信日报入口增加“主题入口才触发”的判断，未命中主题时不得自动进入日报二期流程。
- [ ] 2.2 实现待确认实体识别、名称修正重试和多候选选择能力，支持用户通过修正名称或“候选1/候选2”完成确认。
- [ ] 2.3 固化日报固定五问顺序，每轮只追问下一项缺失字段，并把“没有”“无需共享”等回复规范化为有效片段内容。
- [ ] 2.4 支持在收集中再次输入主题入口时重开新一轮日报口述，清理上一轮待确认上下文。

## 3. 日报内受控写回桥接

- [ ] 3.1 当用户在日报口述中明确要求“写入 CRM 跟进记录”时，基于确认后的客户/项目生成待写回草稿。
- [ ] 3.2 支持日报场景下的 `Customer`、`Opportunity` 以及多对象批量写回桥接，并在写回完成后回到日报补问链路。
- [ ] 3.3 复用既有写回确认、失败保留、幂等保护和审计机制，不新增日报专属的绕过式写入路径。

## 4. 回归验证

- [ ] 4.1 补充单测，覆盖公司/项目识别、多条跟进摘要输出和负向回答标准化。
- [ ] 4.2 补充集成测试，覆盖主题入口、自由口述确认、名称修正、多候选选择、批量写回与写回后继续补日报。
- [ ] 4.3 验证无主题入口的普通聊天不会误入日报链路，避免影响既有问数和普通会话行为。
