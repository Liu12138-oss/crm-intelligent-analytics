## ADDED Requirements

### Requirement: 受控分析必须消费统一组织范围快照

系统 MUST 在 Web 工作台和企业微信机器人问数进入受控分析执行前，基于当前企业微信组织事实、CRM 身份映射、默认团队范围和数据范围白名单生成统一组织范围快照。受控分析执行链路 MUST 仅消费该快照进行范围注入、审计和结果展示，不得直接读取历史静态权限表、企业微信灰度白名单或前端传入的部门条件来扩大数据范围。

#### Scenario: Web 与企业微信问数使用同一范围解析
- **WHEN** 同一个 CRM 用户分别从 Web 工作台和企业微信机器人发起相同经营分析问题
- **THEN** 系统必须基于同一组织范围解析服务生成权限范围
- **THEN** 两个入口不得因为通道不同返回不同的数据范围

#### Scenario: 普通成员问数只返回本人范围
- **WHEN** 普通成员发起“本月新增商机金额排名”等问数请求
- **THEN** 系统必须将范围限制为该成员本人数据
- **THEN** 不得返回其同部门其他成员或团队管理者的数据

#### Scenario: 团队管理者问数返回其团队范围
- **WHEN** 团队管理者发起团队经营分析请求
- **THEN** 系统必须基于直属上级链和递归下属关系注入其团队成员范围
- **THEN** 不得返回不属于其默认团队且未被白名单授权的部门数据

#### Scenario: 数据白名单扩展问数范围
- **WHEN** 当前用户或其角色命中有效数据范围白名单
- **THEN** 系统必须在默认组织范围之外叠加白名单授权部门
- **THEN** 结果中的范围说明和审计快照必须标明白名单授权来源

#### Scenario: 请求超出组织范围时被阻断
- **WHEN** 用户明确请求查看未被默认范围或白名单覆盖的部门、团队或人员数据
- **THEN** 系统必须阻断或收敛到当前有效范围并明确提示
- **THEN** 不得按用户自然语言中的目标部门直接扩大查询范围
