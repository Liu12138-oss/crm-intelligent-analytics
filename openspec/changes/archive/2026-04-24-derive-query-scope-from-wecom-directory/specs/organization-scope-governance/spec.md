## ADDED Requirements

### Requirement: 系统必须沉淀企业微信组织事实层

系统 MUST 基于企业微信官方 API 同步“联软科技集团”授权范围内的部门树、部门负责人、成员列表、成员主部门、成员所属部门、成员直属上级链和原始 payload，并通过手机号、邮箱或人工绑定结果映射到 CRM 用户。组织事实层 MUST 作为后续数据范围推导、日报团队归属、主管诊断和权限排障的统一事实来源。

#### Scenario: 同步部门负责人和成员直属上级
- **WHEN** 企业微信目录同步任务拉取部门与成员详情
- **THEN** 系统必须保存部门 `department_leader`、成员 `main_department`、成员所属部门和可用的直属上级字段
- **THEN** 系统必须保留原始 payload，便于排查企业微信字段缺失或授权不足

#### Scenario: 企业微信成员映射到 CRM 用户
- **WHEN** 企业微信成员手机号或邮箱唯一命中 CRM 用户
- **THEN** 系统必须建立或更新企业微信用户到 CRM 用户的映射
- **THEN** 后续组织范围解析必须使用映射后的 CRM 用户标识作为数据访问主体

#### Scenario: 组织事实字段缺失时不得扩大权限
- **WHEN** 企业微信 API 未返回直属上级、部门负责人或成员部门等关键字段
- **THEN** 系统必须记录字段缺失原因和最近同步时间
- **THEN** 系统不得因为字段缺失而默认授予团队范围或部门范围

### Requirement: 系统必须按组织事实推导默认数据范围

系统 MUST 基于当前组织事实为每个 CRM 用户推导默认数据范围。普通成员默认仅能查看本人数据；团队管理人员默认能查看自己和直属下级团队成员数据；被识别为上级管理者时，系统 MUST 支持递归包含其下级管理人员及成员。默认范围 MUST 输出可审计的 `scopeSnapshot`，至少包含组织、部门、负责人、成员、范围来源和中文范围说明。

#### Scenario: 普通成员默认只能查看本人数据
- **WHEN** 当前用户没有直属下级成员，也没有有效数据白名单
- **THEN** 系统必须将其默认数据范围限制为本人 CRM 用户
- **THEN** 问数、日报预览和团队汇总不得返回其他成员数据

#### Scenario: 团队管理人员查看自己团队数据
- **WHEN** 当前用户在企业微信直属上级链中是若干成员的直属或递归上级
- **THEN** 系统必须将这些成员纳入该用户默认团队范围
- **THEN** 范围快照必须记录命中的上下级链路

#### Scenario: 大区负责人通过下级管理人员归集团队
- **WHEN** 严刚和官俊东均挂在大北区部门中，且两人的直属上级都是王冬
- **THEN** 系统必须将严刚、官俊东及其递归下属团队归入王冬的大北区管理范围
- **THEN** 系统不得把严刚和官俊东误判为大北区最终并列负责人

### Requirement: 系统必须提供独立的数据范围白名单授权

系统 MUST 提供独立于企业微信机器人灰度策略的数据范围白名单。白名单 MUST 支持按 CRM 用户或 CRM 角色授权可访问的部门集合，并记录是否包含子部门、授权原因、有效期、状态、更新人和更新时间。白名单只能扩大到显式授权部门，不得绕过字段白名单、读写边界和审计约束。

#### Scenario: 用户白名单扩展指定部门范围
- **WHEN** 管理员为某 CRM 用户授予某部门的数据范围白名单且授权仍在有效期内
- **THEN** 系统必须在该用户默认范围之外叠加该部门数据范围
- **THEN** 范围快照必须标明该部门来自用户级白名单授权

#### Scenario: 角色白名单扩展指定部门范围
- **WHEN** 管理员为某 CRM 角色授予某部门的数据范围白名单
- **THEN** 命中该角色的用户必须获得对应部门的数据范围扩展
- **THEN** 范围快照必须标明该部门来自角色级白名单授权

#### Scenario: 白名单过期后立即失效
- **WHEN** 某条数据范围白名单已经超过 `expiresAt`
- **THEN** 系统必须在下一次范围解析时排除该授权
- **THEN** 不得因为历史会话、最近查询或缓存继续放行过期范围

### Requirement: 系统必须提供组织范围诊断与审计

系统 MUST 提供组织范围诊断能力，支持管理员查看某个用户的企业微信组织事实、CRM 映射、默认团队范围、白名单扩展范围、最终有效范围和失败原因。系统 MUST 为白名单变更、范围预览、范围解析失败和越权阻断记录审计事件。

#### Scenario: 管理员预览用户最终数据范围
- **WHEN** 管理员在权限中心选择某个 CRM 用户或企业微信用户执行数据范围预览
- **THEN** 系统必须返回默认范围、白名单扩展范围、最终范围和范围说明
- **THEN** 预览结果必须与正式问数运行时使用的范围解析结果一致

#### Scenario: 数据范围授权变更写入审计
- **WHEN** 管理员新增、修改、禁用或恢复数据范围白名单
- **THEN** 系统必须记录授权变更审计事件
- **THEN** 审计事件必须包含变更前后摘要、授权对象、部门范围、是否包含子部门和变更原因

#### Scenario: 越权范围请求被阻断并留痕
- **WHEN** 用户请求查看不在默认范围和白名单范围内的部门或团队数据
- **THEN** 系统必须阻断本次请求并返回明确无权限提示
- **THEN** 系统必须记录目标范围、当前有效范围和阻断原因
