## ADDED Requirements

### Requirement: CRM 智能分析必须以联软 OpenAPI 作为唯一真实数据来源
正式 CRM 智能分析请求 MUST 通过联软标准 OpenAPI 获取真实业务对象、明细和统计基础数据。SQLite 脱敏快照、MySQL 分析库、Text-to-SQL、受控 SQL 和本地样例数据 MUST NOT 作为正式结果的数据来源。它们仅可用于结构理解、离线模板验证、治理诊断或测试。

#### Scenario: 商机分析返回真实 OpenAPI 明细
- **WHEN** 用户询问“帮我分析一下商机的进展”
- **THEN** 系统必须通过联软 OpenAPI 获取当前授权范围内的商机明细或统计基础数据
- **THEN** 返回明细中的商机名、客户名、渠道商名、负责人和阶段释义必须来自 OpenAPI 或 OpenAPI 字典
- **THEN** 系统不得展示 SQLite 样例名、脱敏占位名或分析库占位行

#### Scenario: OpenAPI 不可用时不展示伪结果
- **WHEN** 用户问题已被识别为 CRM 只读分析，但联软 OpenAPI 未配置、不可达或缺少必要资源
- **THEN** 系统必须返回 OpenAPI 能力缺口或联调失败提示
- **THEN** 系统不得自动切换到 SQLite、MySQL 分析库或受控 SQL 生成正式结果

### Requirement: 自由问数必须只保留统一业务语义主入口
自由自然语言 CRM 分析 MUST 使用统一业务语义解析结果作为主入口。系统 MUST 先识别业务对象、指标、维度、时间、分析主题、明细需求和建议需求，再映射到受控 OpenAPI 数据需求计划。窄业务意图解析和本地 CRM 只读词表 MAY 保留为测试或迁移代码，但 MUST NOT 作为正式分析主链。

#### Scenario: 综合经营问题拆分为业务任务
- **WHEN** 用户询问“公司当前的合作伙伴开拓情况、客户商机报备及订单情况，并给出后续经营建议”
- **THEN** 统一业务语义主入口必须识别合作伙伴、客户报备/商机、订单和经营建议四类需求
- **THEN** 系统必须生成组合业务任务，而不是把问题压成单一合同转化、订单贡献或商机金额报告

#### Scenario: 语义解析失败时安全阻断
- **WHEN** 统一业务语义解析不可用、超时或返回非法结构
- **THEN** 系统必须返回“暂无法完成业务语义解析”的安全提示或缺口说明
- **THEN** 系统不得继续使用窄意图解析或本地词表构造可执行 CRM 分析意图

### Requirement: OpenAPI 数据需求计划必须显式表达业务缺口
当统一业务语义可理解但联软 OpenAPI 缺少对象、字段、筛选、排序、分页、字典或关联能力时，系统 MUST 生成业务可读的能力缺口清单，并在审计快照中记录缺口来源。系统 MUST NOT 用脱敏快照、样例数据或 AI 编造内容补齐缺口。

#### Scenario: 缺少客户报备到商机关联能力
- **WHEN** 用户要求分析客户报备和关联商机情况，但 OpenAPI 未提供可用关联字段或关联查询能力
- **THEN** 系统必须说明缺少“客户报备与商机关联关系”能力
- **THEN** 系统必须建议联软补充对应字段或接口
- **THEN** 系统不得用 SQLite 表关系推断正式结果

#### Scenario: 缺少真实名称字段
- **WHEN** OpenAPI 返回记录缺少渠道商名称、客户名称、商机名称或负责人名称
- **THEN** 系统必须在结果中标记该明细字段缺失或返回能力缺口
- **THEN** 系统不得用“渠道商001”“客户001”或样例名替代真实名称

### Requirement: 四类默认主题必须按 OpenAPI 业务链执行
系统 MUST 为商机、渠道商、客户报备和订单四类默认问法建立稳定 OpenAPI 数据需求模板。模板 MUST 表达对应对象、关联对象、指标、维度、明细字段和经营建议输入，不得依赖 SQLite 固定模板或 MySQL 分析库模板。

#### Scenario: 商机默认主题
- **WHEN** 用户只问“商机”“商机情况”或“商机进展”
- **THEN** 系统必须默认查询商机整体情况，并带上渠道商维度
- **THEN** 指标至少覆盖商机数量、商机金额、阶段分布、对应渠道商和重点商机明细

#### Scenario: 渠道商默认主题
- **WHEN** 用户只问“渠道商”“合作伙伴”或“服务商经营情况”
- **THEN** 系统必须围绕渠道商经营贡献分析其关联客户报备、商机和订单情况
- **THEN** 明细和聚合基础数据必须来自 OpenAPI

#### Scenario: 客户报备默认主题
- **WHEN** 用户只问“客户报备”或“报备情况”
- **THEN** 系统必须查询客户报备整体情况并带上渠道商维度
- **THEN** 指标至少覆盖报备数、关联商机情况和未关联商机报备

#### Scenario: 订单默认主题
- **WHEN** 用户只问“订单”或“下单情况”
- **THEN** 系统必须查询订单整体情况并带上渠道商维度
- **THEN** 指标至少覆盖订单数、订单金额、渠道商贡献和重点订单明细
