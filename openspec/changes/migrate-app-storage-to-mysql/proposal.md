## Why

线上当前仍以 `/srv/crm-intelligent-analytics/shared/.runtime/app-storage.json` 承接本系统自有状态，文件已增长到约 58 MB，并持续保存审计、问数结果、最近查询、会话、企业微信上下文、AI 配置、合同审核元数据等多类生产数据；合同审核原文、审核报告、结构化结果、AI 调试上下文和 SQL 审计归档仍散落在 `.runtime/` 子目录。该形态缺少事务、索引、并发写保护、统一备份恢复和完整数据盘点边界，随着审计、合同审核和分析结果增长，会继续放大数据丢失、写入覆盖、接口变慢、遗漏迁移和回滚困难风险，因此必须尽早、彻底切换到独立 MySQL 应用库。

## What Changes

- 新增独立 MySQL 应用库存储能力，承接本系统全部自有持久化数据，不写入 CRM 业务库，不改变 CRM 原始客户、商机、合同、组织、用户等数据来源。
- 引入可灰度的应用存储驱动：支持 `json`、`dual`、`mysql` 三种模式，按“JSON 主读写 -> 历史导入 -> 双写校验 -> MySQL 主读 -> MySQL 主写”的顺序平顺切换。
- 新增历史数据幂等导入能力：从线上 `app-storage.json`、`contract-review/`、`sql-audit-archive/` 和其它 `.runtime/` 持久化文件读取历史数据，按集合和文件对象导入独立应用库，支持批次水位、记录哈希、文件哈希、冲突报告、重跑和校验，不以一次性停机导入作为切换前提。
- 新增双写与对账能力：双写期间继续保障现有功能以稳定路径运行，同时将新增与更新同步写入 MySQL，并输出集合数量、最近记录、关键字段哈希和失败差异。
- 新增读切换与回退能力：低风险读接口先切到 MySQL，保留 JSON / 历史文件 fallback；确认稳定后再切 MySQL 主写，JSON 和历史文件目录保留为只读回滚快照。
- 新增受保护的一键迁移与回滚编排命令：提供 `storage:switch-to-mysql` 和 `storage:rollback-to-json`，一键触发但必须内置前置检查、备份校验、导入、对账、分阶段切换、失败中断、脱敏日志和回滚差异处理，禁止无校验强切。
- 调整 SQL 审计能力：SQL 审计不得继续依赖 `app-storage.json` 作为长期主存储；必须支持应用库表级检索、分页、详情、reveal 权限校验和历史数据校验。
- 调整合同审核和文件产物能力：合同原始文件、审核报告、结构化结果、批注版合同、AI 调试上下文等所有本系统生成或接收的持久化文件必须进入数据库文件对象表；文件系统只允许作为迁移来源或短时临时区，不得作为长期权威存储。
- 新增运行态锁、迁移状态、对账报告、失败补偿、归档记录和数据库文件对象的统一存储要求，避免 `.runtime/` 中继续残留业务恢复所需数据。
- 不改变登录、企业微信机器人、Web 智能分析、治理后台、合同审核、最近查询、导出和审计中心的用户可见接口语义。

## Capabilities

### New Capabilities

- `app-storage-mysql-persistence`: 定义本系统全部自有持久化数据从 JSON / 文件运行态平顺切换到独立 MySQL 应用库的存储模式、数据导入、文件对象入库、双写、对账、切读、回滚、备份和验收要求。

### Modified Capabilities

- `sql-execution-auditability`: 增加 SQL 审计使用独立应用库作为长期主存储的要求，并约束分页检索、详情、reveal、归档文件对象、双写校验和旧 JSON / 文件兼容回退行为。

## Impact

- 后端数据库层：新增应用库连接、迁移执行器、存储驱动抽象、集合级仓储适配、JSON 导入脚本、双写与对账服务，以及受保护的迁移 / 回滚编排命令。
- 后端业务仓储：涉及 `AppStorageService` 及当前依赖 `appStorage.state` / `appStorage.persist()` 的分析、查询资产、治理、审计、会话、企业微信、日报、通知、合同审核、AI Profile 等仓储。
- 数据库：新增独立 MySQL 应用库表、索引、迁移批次表、对账报告表、文件对象表、运行态锁表和必要的 JSON payload 字段；不得复用 CRM 业务库承接本系统状态。
- 运行态数据：线上 `shared/.runtime/app-storage.json`、`contract-review/`、`sql-audit-archive/`、运行态锁和其它持久化文件必须先完整备份，再作为导入来源和回滚快照保留；切换完成后不得继续作为主写或权威读取存储。
- 运维配置：新增 `APP_STORAGE_DB_*`、`APP_STORAGE_DRIVER`、双写、读主库、fallback、迁移批次、校验、编排命令和回滚相关配置；敏感值不得写入文档、日志或测试夹具。
- 测试与验证：必须覆盖历史导入幂等、文件对象入库与下载、双写失败降级、读 MySQL + JSON / 文件 fallback、MySQL 主写、回滚、审计检索、登录、问数、最近查询、常用查询、合同审核、企业微信上下文恢复、通知投递记录和权限变更缓存失效。
